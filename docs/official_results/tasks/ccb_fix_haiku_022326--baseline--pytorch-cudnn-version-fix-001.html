
## Why This Matters

This fix enables **runtime version detection** instead of relying on **compile-time version**, which allows:

- Users to resolve version mismatches by updating their cuDNN library locally (addressing issue #166643)
- Proper feature detection for 3D convolutions and SDPA operations
- Dynamic dispatch based on actual installed cuDNN version rather than build-time version

All other critical dispatch locations (Convolution.cpp, SDPA utilities) already use `versionRuntimeCuDNN()` and are correctly checking the runtime version for version-specific optimizations and workarounds.

The changes are minimal, focused, and follow the repository's conventions for checking CUDA/cuDNN capabilities at runtime.

## Summary

I identified **7 core Python source files** that implement Django's CSRF protection mechanism:

1. **django/middleware/csrf.py** - The main CSRF middleware containing the validation logic
2. **django/views/decorators/csrf.py** - CSRF decorators for per-view protection
3. **django/utils/crypto.py** - Timing attack prevention implementation
4. **django/template/context_processors.py** - CSRF context processor for templates
5. **django/template/defaulttags.py** - CSRF template tag renderer
6. **django/views/csrf.py** - CSRF failure view handler
7. **django/template/backends/utils.py** - Template backend CSRF utilities

## Key Components

**CsrfViewMiddleware Token Validation Chain:**
- `process_view()` orchestrates the validation flow
- `_check_token()` retrieves the secret and request token
- `_check_token_format()` validates token structure
- `_does_token_match()` performs the comparison
- `constant_time_compare()` prevents timing attacks using `secrets.compare_digest()`

**Timing Attack Prevention:**
The `constant_time_compare()` function in `django/utils/crypto.py` uses Python's `secrets.compare_digest()` to ensure comparison time is independent of where strings differ, preventing attackers from inferring token validity through response timing.

**Decorators & Helpers:**
- `csrf_protect` - Per-view CSRF protection
- `csrf_exempt` - Exempt views from protection
- `requires_csrf_token` - Ensure token availability
- `ensure_csrf_cookie` - Force cookie generation

# Test Coverage Gap Analysis: Envoy HTTP Connection Manager

## Summary

This analysis identifies critical untested code paths in Envoy's HTTP Connection Manager (HCM) filter chain implementation. The HCM is responsible for HTTP protocol handling, stream lifecycle management, connection draining, timeout handling, and filter chain execution. While extensive tests exist for basic request/response flows and per-stream timeouts, several important error paths and connection-level behaviors lack test coverage. These gaps could allow subtle bugs related to connection draining, premature stream reset handling, watermark propagation, and HTTP version-specific behaviors to slip through.

## Coverage Gaps

### Gap 1: Connection-Level Idle Timeout Without Active Streams

- **File**: `source/common/http/conn_manager_impl.cc`
- **Lines**: ~756-767
- **Risk**: HIGH
- **Description**: The `onIdleTimeout()` method handles connection-level idle timeouts when no streams are active. This differs from per-stream idle timeouts which are well-tested. The code path distinguishes between three scenarios: (1) no codec created yet (FlushWrite), (2) not in drain state (start drain sequence), (3) already draining (implicit no-op). The condition checking codec existence and drain state transitions are not covered by existing tests.

  - Trigger: Manually invoke `onIdleTimeout()` callback or wait for timer
  - Assertions: Verify `downstream_cx_idle_timeout_` stat incremented, connection closed with FlushWrite, no GOAWAY sent (since no codec)

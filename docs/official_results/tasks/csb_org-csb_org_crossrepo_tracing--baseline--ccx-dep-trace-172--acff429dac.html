## Key Discoveries

**Kubernetes uses its own native controller framework** rather than the controller-runtime library. The registration pattern works as follows:

### Core Components Found:

1. **Reconciler Pattern** (`DeploymentController` in `pkg/controller/deployment/deployment_controller.go`):
   - Struct-based approach with client, listers, work queue, and event recorder
   - Implements reconciliation through a `syncHandler` function

2. **Manager Equivalent** (`ControllerContext` in `cmd/kube-controller-manager/app/controllermanager.go`):
   - Provides InformerFactory, ClientBuilder, and ComponentConfig to controllers
   - Manages the overall control plane lifecycle

3. **SetupWithManager Equivalent** (Split into two parts):
   - **Initialization**: `NewDeploymentController()` registers informer event handlers
   - **Manager Integration**: `startDeploymentController()` instantiates and calls `Run()`

4. **Reconciliation Loop**:
   - `Run()` method spawns worker goroutines
   - Workers process items from the work queue continuously
   - Informer event handlers enqueue items when changes occur

### Registration Chain:
```
Run() → CreateControllerContext() → StartControllers() → 
startDeploymentController() → NewDeploymentController() → Run(ctx, workers)
```

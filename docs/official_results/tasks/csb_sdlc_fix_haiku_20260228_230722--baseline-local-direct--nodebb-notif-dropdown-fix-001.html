## Plan Review

I've completed my analysis and created a comprehensive plan to fix both dropdown issues in NodeBB. Here's my proposed solution:

### **Issue 1: Notifications Dropdown**
**Root Cause**: When notifications load asynchronously, `bsDropdown.update()` is never called to refresh the Popper.js positioning calculations after content is rendered.

**Fix Location**: `/app/public/src/modules/notifications.js`
- Add Bootstrap Dropdown instance retrieval and `update()` call after `app.parseAndTranslate()` renders the notifications HTML

### **Issue 2: Category Selector in Fork/Move Modals**  
**Root Cause**: While `categorySearch.js` does call `bsDropdown.update()` after rendering, it only updates positioning once after search/filter. When the dropdown is reopened, the positioning is stale.

**Fix Location**: `/app/public/src/modules/categorySearch.js`
- Add a `shown.bs.dropdown` event handler to ensure positioning is always updated when the dropdown is shown

### **Implementation Summary**
- **2 files to modify** with simple, targeted fixes following existing patterns
- **No breaking changes** - just adding missing Bootstrap API calls that are already used elsewhere in the codebase
- **Both fixes use the same proven pattern** from `categorySearch.js` and `tagFilter.js`

The complete plan is documented in `/logs/agent/sessions/plans/stateless-drifting-hopper.md` with testing procedures and success criteria.

# Fix Notifications and Category Selector Dropdown Issues

## Context

NodeBB v4.4.3 has two dropdown-related issues:

1. **Notifications Dropdown**: Asynchronously loaded notifications fail to refresh or toggle correctly when opened. The dropdown menu's position is not updated after content loads asynchronously.

2. **Category Selector in Fork/Move Modals**: The category selector dropdown displays inconsistent behavior with potential dropup class mishandling, causing incorrect menu positioning.

Both issues stem from missing Bootstrap Dropdown API calls after asynchronous content updates.

## Root Cause Analysis

### Issue 1: Notifications Dropdown

**Location**: `/app/public/src/modules/notifications.js` lines 31-72

**Problem**:
- When notifications dropdown opens, `header/notifications.js` listens to 'show.bs.dropdown' event
- This triggers `loadNotifications()` which is async (uses `socket.emit()` + `app.parseAndTranslate()`)
- After rendering HTML via `notifList.html(html)`, the code does NOT call `bsDropdown.update()`

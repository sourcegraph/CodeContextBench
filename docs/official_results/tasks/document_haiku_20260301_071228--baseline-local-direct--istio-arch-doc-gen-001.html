# Istio Pilot Discovery Service Architecture

## Overview

The Istio Pilot Discovery Service is the control plane component responsible for serving dynamic configuration to data plane proxies (Envoy sidecars and gateways) through xDS (Envoy Discovery Service) APIs. Pilot translates high-level Istio networking resources (VirtualServices, DestinationRules, Sidecars, etc.) into Envoy-specific configurations (Clusters, Routes, Listeners, Endpoints) and maintains a fleet-wide view of services across multiple Kubernetes clusters.

This document explains the architecture of four core subsystems:

1. **DiscoveryServer** — Pilot's gRPC server implementing Envoy's xDS protocol
2. **ADS (Aggregated Discovery Service)** — The bidirectional stream handler managing individual proxy connections
3. **PushContext** — The immutable configuration snapshot used during a push cycle
4. **Service Registry** — The service discovery layer integrating Kubernetes and multi-cluster data

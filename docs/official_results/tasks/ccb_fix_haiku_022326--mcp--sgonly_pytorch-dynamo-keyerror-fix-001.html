# [dynamo] fix keyerror in resume_execution,  fix store attr

**Difficulty:** MEDIUM  
**Category:** cross_module_bug_fix



## Description

This task fixes two related bugs in PyTorch's Dynamo compiler: a `KeyError` crash during resume code generation on Python 3.11+ and incorrect handling of `STORE_ATTR` instructions in `with` blocks after graph breaks. The `KeyError` occurs in `resume_execution.py` when Dynamo tries to reconstruct bytecode for resuming execution after a graph break inside nested context managers (e.g., nested `torch.no_grad()` blocks), because the exception table entry remapping fails to find target offsets.

The fix restructures the offset remapping logic in `torch/_dynamo/resume_execution.py` and adjusts `symbolic_convert.py` to correctly handle exception table entries and store-attribute operations, moving test cases from `test_repros.py` to the more appropriate `test_ctx_manager.py`. Without the fix, any `torch.compile`-d function with graph breaks inside nested context managers crashes on Python 3.11+.

## Summary of Changes

### 1. Fixed `torch/_dynamo/resume_execution.py`

**Problem:** A `KeyError` crash occurred on Python 3.11+ when Dynamo tried to reconstruct bytecode for resuming execution after a graph break inside nested context managers (e.g., nested `torch.no_grad()` blocks).

**Root Cause:** The offset remapping logic in `generate_based_on_original_code_object` used a single global dictionary for block target offset remapping, which failed when multiple resume points existed at different offsets.

**Solution:** Restructured the offset remapping logic to use a per-offset nested dictionary:

1. **Changed data structure** (lines 252-255):
   - Before: `block_target_offset_remap: Optional[dict[int, int]] = None`
   - After: `block_target_offset_remap: dict[int, dict[int, int]] = dataclasses.field(default_factory=dict)`

2. **Updated offset initialization** (line 593):
   - Changed `new_offset = None` to `new_offset = -1` for better initialization tracking

3. **Added assertions** (lines 607, 611):
   - Added `assert new_target.offset is not None` to catch invalid offset references
   - Added `assert new_offset >= 0` to ensure valid offset after code transformation

4. **Fixed condition** (line 618):
   - Changed `if not meta.block_target_offset_remap:` to `if new_offset not in meta.block_target_offset_remap:`
   - This creates a separate offset mapping for each resume point, preventing KeyError

5. **Nested dictionary access** (lines 669-670):
   - Changed `meta.block_target_offset_remap[n]` to `meta.block_target_offset_remap[new_offset][n]`
   - This properly accesses the nested mapping structure

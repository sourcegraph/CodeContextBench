## Terraform v1.15.0 (Unreleased)

### Breaking Changes

- **backend/s3**: The `AWS_USE_FIPS_ENDPOINT` and `AWS_USE_DUALSTACK_ENDPOINT` environment variables now only respect `true` or `false` values, aligning with the AWS SDK for Go v2. Previously, any non-empty value was treated as `true`. Update your environment variable exports to use explicit `true` or `false` values instead of relying on implicit truthy behavior. ([#37601](https://github.com/hashicorp/terraform/issues/37601))

### New Features

- **command/validate**: The `validate` command now checks the `backend` block for errors. This ensures the backend type exists, that all required attributes are present, and that the backend's own validation logic passes. This allows validation failures in backend configuration to be detected earlier without running `terraform init`. ([#38021](https://github.com/hashicorp/terraform/issues/38021))

- **core/variables**: You can now set a `deprecated` attribute on `variable` and `output` blocks to indicate that they are deprecated. Terraform will produce warnings when a value is passed to a deprecated variable or when a deprecated output is referenced. This allows module authors to guide users toward using newer input/output names. ([#38001](https://github.com/hashicorp/terraform/issues/38001))

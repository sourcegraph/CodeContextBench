# APF QueueSet Subsystem: Algorithmic Deep-Dive

## Executive Summary

The Advanced Priority and Fairness (APF) QueueSet subsystem implements weighted fair queuing on the Kubernetes API server to provide request pacing and priority isolation. Each priority level is served by a QueueSet that dispatches requests according to virtual time fairness, ensuring that under congestion, all active queues make progress proportionally to their configured concurrency weights. This document traces the architectural boundaries, data flow, algorithmic mechanics, operational tradeoffs, and extension points of this subsystem.

---

## 1. Subsystem Purpose, Boundaries, and Dependencies

### 1.1 Purpose

The APF QueueSet subsystem enforces request fairness and priority isolation at the API server level by:

1. **Queuing requests** when they arrive faster than the server can serve them
2. **Dispatching requests fairly** using weighted fair queuing adapted for server requests (where multiple requests can execute concurrently and work duration is unknown until completion)
3. **Isolating priority levels** so that high-priority flows cannot starve low-priority flows
4. **Handling work heterogeneity** by estimating per-request "width" (seats) to account for different computational costs

# Kubelet Container Manager Architecture Guide

## Overview

The kubelet container manager subsystem is responsible for managing container resources, enforcing Quality of Service (QoS) classes, and coordinating resource allocation across CPU, memory, and device plugins. It sits at the intersection of pod scheduling, container runtime execution, and system resource enforcement on the node.

The container manager operates at several levels:
- **System level**: Managing system containers and daemon processes
- **Node level**: Enforcing node allocatable reservations and capacity limits
- **Pod level**: Creating pod cgroups and managing QoS hierarchies
- **Container level**: Allocating exclusive resources and updating runtime constraints

## Subsystem Purpose, Boundaries, and Dependencies

### Purpose
The container manager ensures that:
1. Container resource requests and limits are enforced through cgroup mechanisms
2. Resources are allocated according to pod QoS classes (Guaranteed, Burstable, BestEffort)
3. Topology-aware resource allocation is coordinated across multiple resource managers
4. Device plugins can register and provide custom resources
5. Dynamic resource allocation can be managed through DRA framework
6. System services and kubelet process have protected resource reservations

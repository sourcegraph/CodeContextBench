## Key Findings

  - `RoleEscalationAuthorized()` - Checks explicit 'escalate' verb permission
  - `BindingAuthorized()` - Checks 'bind' permission for role references

### Storage-Level Enforcement
The policybased storage implementations intercept Create/Update operations and validate escalation:
- **ClusterRole storage** - Validates ClusterRole creation/updates
- **ClusterRoleBinding storage** - Validates ClusterRoleBinding creation/updates
- **Role storage** - Validates Role creation/updates
- **RoleBinding storage** - Validates RoleBinding creation/updates

### Validation Chain
- **`ConfirmNoEscalation()`** - Core validation that checks if user's current permissions cover the permissions they're trying to grant
- **`Covers()`** (in component-helpers) - Core algorithm that compares owner rules against servant rules to ensure no privilege escalation
- **`RuleAllows()`** - Matches requests against policy rules

### Supporting Infrastructure
- Validation utilities for rule comparison and matching
- Helper functions for PolicyRule parsing and evaluation
- RBAC authorizer for authorization decisions

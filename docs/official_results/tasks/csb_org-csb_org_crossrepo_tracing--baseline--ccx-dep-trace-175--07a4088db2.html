## Pipeline Overview

The alert notification pipeline flows through these key stages:

1. **Alert Ingestion** (`api/v2/api.go`): HTTP POST handler receives alerts and stores them via the alert provider

2. **Alert Storage** (`provider/mem/mem.go`): In-memory provider stores alerts and notifies subscribers

3. **Dispatcher Processing** (`dispatch/dispatch.go`): Multiple worker goroutines receive alerts and route them through the system

4. **Routing Tree Traversal** (`dispatch/route.go`): `Route.Match()` performs depth-first traversal to find all matching routes for each alert

5. **Aggregation Grouping** (`dispatch/dispatch.go`): Alerts are grouped by group labels, applying GroupWait and GroupInterval timers

6. **Notification Pipeline** (`notify/notify.go`): The core pipeline stages execute in sequence:
   - **Inhibition Check** (`inhibit/inhibit.go`): `Inhibitor.Mutes()` checks if source alerts inhibit target alerts
   - **Silence Check** (`silence/silence.go`): `Silencer.Mutes()` checks if alerts match active silences
   - **Notifier Dispatch** (`notify/notify.go`): `RetryStage.Exec()` invokes actual notifiers with retry logic

The complete pipeline is constructed by `PipelineBuilder.New()` which chains together muting stages (inhibition, time-based), silence checks, and finally the receiver-specific notifier integrations that execute in parallel via FanoutStage.

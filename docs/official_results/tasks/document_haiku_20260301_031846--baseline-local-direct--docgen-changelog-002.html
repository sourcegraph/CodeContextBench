# Flipt Release Notes

## Overview

This release focuses on improving OpenTelemetry tracing instrumentation, enabling more flexible distributed tracing configurations and better observability for Flipt deployments.

## Breaking Changes

None in this release.

## New Features

### Enhanced OpenTelemetry Tracing Configuration

#### Trace Sampling Ratio (`tracing.samplingRatio`)

**New Configuration Option**

- **Type**: Float (0.0 - 1.0)
- **Default**: 1.0 (sample all traces)
- **Description**: Controls the sampling ratio for OpenTelemetry traces, allowing you to reduce trace volume in high-throughput environments

**YAML Configuration Example**:
```yaml
tracing:

  exporter: otlp
```

**Environment Variable**: `FLIPT_TRACING_SAMPLINGRATIO=0.1`

**Use Cases**:
- Reduce trace storage costs in production environments
- Sample traces probabilistically based on deployment needs
- Balance observability with infrastructure costs

- **Type**: Array of strings
- **Default**: `["tracecontext", "baggage"]`

  - `baggage` — W3C Baggage specification
  - `b3` — Zipkin B3 single-header format
  - `b3multi` — Zipkin B3 multi-header format
  - `jaeger` — Jaeger propagation format
  - `xray` — AWS X-Ray propagation format
  - `ottrace` — OpenTracing format
  - `none` — No propagation

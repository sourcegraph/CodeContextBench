6. **django/db/backends/base/operations.py** - Parameter type adaptation

### **Critical Security Points**

1. **RawSQL Expression Sanitization**: The RawSQL class enforces separation of SQL structure from parameters at the expression level

2. **raw() Method Parameter Binding**: The QuerySet.raw() method accepts a `params` argument that is kept separate from the raw SQL string

3. **Cursor.execute() Enforcement**: The RawQuery._execute_query() method adapts parameters and passes them separately to cursor.execute(), delegating parameterization to the database driver

4. **Type-Safe Parameter Adaptation**: Parameters are adapted based on their Python type, not parsed from user input

### **Security Architecture**
Django prevents SQL injection by maintaining complete separation between SQL structure and user input throughout the entire query pipeline, ultimately relying on the database driver's native parameterization mechanism rather than string concatenation or escaping.

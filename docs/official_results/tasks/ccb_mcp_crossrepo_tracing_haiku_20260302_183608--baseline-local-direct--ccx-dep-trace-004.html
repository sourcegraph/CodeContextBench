## Summary

**Question 1: What is the type/struct in grafana/grafana that acts as the HTTP client for Loki queries?**
- **Answer**: `LokiAPI` (located in `pkg/tsdb/loki/api.go`)
- It's a struct that encapsulates an `*http.Client` and provides methods like `DataQuery()` and `RawQuery()` to make HTTP requests to Loki

**Question 2: What is the function in grafana/loki that parses incoming HTTP instant query requests?**
- **Answer**: `InstantQueryHandler` (located in `pkg/querier/http.go`)
- It handles GET/POST requests to the `/loki/api/v1/query` endpoint and parses LogQL metric expressions along with query parameters like `time`, `limit`, and `direction`

## API Call Chain Flow

1. **Grafana** (`pkg/tsdb/loki/loki.go`): `QueryData()` receives a query request
2. **Grafana** (`pkg/tsdb/loki/loki.go`): `queryData()` parses the query and creates a `LokiAPI` instance
3. **Grafana** (`pkg/tsdb/loki/api.go`): `LokiAPI.DataQuery()` constructs and sends an HTTP GET request to `/loki/api/v1/query`
4. **Loki** (`pkg/querier/http.go`): `InstantQueryHandler` receives the HTTP request, parses parameters, validates the LogQL expression, and returns results
5. **Grafana** (`pkg/tsdb/loki/api.go`): Response is parsed and converted to Grafana's data frame format

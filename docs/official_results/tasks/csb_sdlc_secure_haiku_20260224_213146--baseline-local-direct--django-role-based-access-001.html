**Severity:** High
**Ticket:** FORMS-2847

- Malformed HTML: `<input data-label="Value with "quotes"">` 
- Potential XSS vulnerabilities

### The Solution
I've implemented proper HTML escaping in the `build_attrs` method by:

1. **Added import**: `conditional_escape` from `django.utils.html`
2. **Modified build_attrs**: All attribute values are now escaped using `conditional_escape`, which:
   - Properly escapes HTML special characters (`&` → `&amp;`, `"` → `&quot;`, `<` → `&lt;`, `>` → `&gt;`, `'` → `&#x27;`)
   - Respects `mark_safe`'d values (prevents double-escaping)
   - Handles special values (True, False, None) appropriately

### How It Works
```python
def build_attrs(self, base_attrs, extra_attrs=None):
    """Build an attribute dictionary."""
    attrs = {**base_attrs, **(extra_attrs or {})}
    # Escape attribute values to prevent XSS vulnerabilities.
    # Use conditional_escape to respect mark_safe'd values.
    return {
        key: conditional_escape(value) if value not in (True, False, None) else value
        for key, value in attrs.items()
    }
```

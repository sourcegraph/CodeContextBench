# Envoy External Processing (ext_proc) Filter Migration Guide
## v1.30.0 → v1.31.0

**Release Date:** July 19, 2024

---

## 1. Overview

This migration guide covers the upgrade path for the Envoy External Processing (ext_proc) HTTP filter from version 1.30.0 (April 2024) to version 1.31.0 (July 2024). The ext_proc filter enables external services to intercept and process HTTP traffic through a bidirectional gRPC stream.

The v1.31.0 release introduces several significant improvements to the ext_proc filter, including better control over route cache handling, enhanced error reporting, support for deferred stream closure in observability mode, and improved gRPC context propagation for tracing.

**Key Headlines:**
- New `route_cache_action` enum provides explicit control over route caching behavior
- Timeout errors now return `504 Gateway Timeout` instead of `500 Internal Server Error`
- Support for deferred stream closure timeout in observability mode
- Enhanced tracing context propagation in gRPC bidirectional streams
- Deprecation of `disable_clear_route_cache` boolean in favor of the new enum

---

## 2. Breaking Changes

### 2.1 Route Cache Handling Redesign

**Impact Level:** ⚠️ **Moderate** - Requires configuration review and potential behavioral changes

The route cache clearing mechanism has been redesigned to use an explicit enum-based approach instead of a simple boolean flag.

#### What Changed

**v1.30.0 Configuration:**
```protobuf
message ExternalProcessor {
  // Boolean flag to disable route cache clearing
  bool disable_clear_route_cache = 11;
}
```

**v1.31.0 Configuration:**
```protobuf

  // Enum-based route cache action (RECOMMENDED)
  enum RouteCacheAction {
    DEFAULT = 0;   // Clear only if ext_proc says so
    CLEAR = 1

I've identified all five key C++ source files that implement the Chromium Blink rendering pipeline stages:

1. **StyleResolver** (`third_party/blink/renderer/core/css/resolver/style_resolver.h`)
   - Resolves computed styles for DOM elements based on CSS stylesheets

2. **LayoutObject** (`third_party/blink/renderer/core/layout/layout_object.h`)
   - Base class for the layout tree, representing objects in the layout calculation

3. **PaintLayer** (`third_party/blink/renderer/core/paint/paint_layer.h`)
   - Handles complex painting operations including opacity, clipping, filters, and stacking contexts

4. **PaintController** (`third_party/blink/renderer/platform/graphics/paint/paint_controller.h`)
   - Processes display items and manages paint operations with caching and invalidation logic

5. **DisplayItemList** (`third_party/blink/renderer/platform/graphics/paint/display_item_list.h`)
   - Container for display items that records the final paint operations

The rendering pipeline flows as: **CSS styling** → **Layout calculation** → **Paint layer organization** → **Paint command recording** → **Display list execution**.

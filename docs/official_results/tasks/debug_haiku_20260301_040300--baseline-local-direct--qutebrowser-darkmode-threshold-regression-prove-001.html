## Investigation Summary

**Bug:** The dark mode text brightness threshold setting (`colors.webpage.darkmode.threshold.text`) has no effect on Qt 6.4+ because qutebrowser uses the wrong Chromium setting name.

**Root Cause:** 
- Chromium 99.0 (used by Qt 6.4) renamed the setting from `TextBrightnessThreshold` to `ForegroundBrightnessThreshold`
- qutebrowser's `darkmode.py` doesn't have a `Variant.qt_64` defined, so it falls back to using `Variant.qt_63`
- When Qt 6.4 is detected, the code passes `TextBrightnessThreshold` to Chromium, which no longer recognizes this setting
- Chromium silently ignores the unknown setting, making the feature appear broken

**Key Evidence:**
- Line 318 of darkmode.py: Version check returns `Variant.qt_63` for Qt 6.4+
- Git history shows planned support for Qt 6.4 with correct Chromium key name changes, but implementation is incomplete

## Regression Test

I've created `/workspace/regression_test.py` which:

1. **Demonstrates the bug** by showing Qt 6.4 incorrectly uses `TextBrightnessThreshold`
2. **Verifies Qt 6.3 works correctly** with a control test
3. **Is self-contained** and can run standalone or with pytest
4. **Fails for the right reason** - asserting that Qt 6.4 should find `ForegroundBrightnessThreshold` but finding `TextBrightnessThreshold` instead

# Bug Investigation: Dark Mode Text Threshold Setting Has No Effect on Qt 6.4

## Summary

The dark mode text brightness threshold setting (`colors.webpage.darkmode.threshold.text`) has no effect when qutebrowser runs with **Qt 6.4+** because the code does not have a `Variant.qt_64` defined and therefore uses the Qt 6.3 variant definition.

## Root Cause

In Chromium 99.0 (used by Qt 6.4), the Chromium setting name was changed from `TextBrightnessThreshold` to `ForegroundBrightnessThreshold`. However, qutebrowser's darkmode module only has three variants defined:

1. **Variant.qt_515_2** - Uses `TextBrightnessThreshold`
2. **Variant.qt_515_3** - Uses `TextBrightnessThreshold`
3. **Variant.qt_63** - Uses `TextBrightnessThreshold` (with new `IncreaseTextContrast` setting)

When Qt 6.4 is detected, the version check at line 318 of darkmode.py returns `Variant.qt_63`:

```python
if versions.webengine >= utils.VersionNumber(6, 3):
    return Variant.qt_63
```

This causes Qt 6.4 to pass `TextBrightnessThreshold` to Chromium, which no longer recognizes this setting name. Chromium 6.4 expects `ForegroundBrightnessThreshold`, so the setting is silently ignored.

## Evidence

Looking at the git history, commit `4e61bfedd` ("Rename colors.webpage.darkmode.threshold.text") shows that support for Qt 6.4 was planned:

- Defines `Variant.qt_64`
- Renames the config setting from `threshold.text` to `threshold.foreground`
- Uses `copy_replace_setting()` method to change the Chromium key to `ForegroundBrightnessThreshold`

However, this commit also shows that the `copy_replace_setting()` method was planned but never implemented in the `_Definition` class.

A later commit `2a10461ca` ("Remove dark mode settings removed from Chromium") attempted to remove the Qt 6.3 variant and make Qt 6.4 work, but this change does not appear in the current codebase.

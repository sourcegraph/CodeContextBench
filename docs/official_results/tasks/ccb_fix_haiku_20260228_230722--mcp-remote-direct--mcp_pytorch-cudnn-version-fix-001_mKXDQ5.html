## Description

Test plan: add test comparing performance with cuDNN 9.15+ when disabling and enabling convolutino explicilty for 3D cases.

- Users to update their local cuDNN library and have PyTorch automatically adapt its dispatching heuristics
- Proper support for new cuDNN features (e.g., improved 3D convolution support in cuDNN 9.15+)
- Memory format optimization based on actual runtime capabilities rather than compile-time constants

The implementation correctly switches from using the compile-time `CUDNN_VERSION` macro to the runtime `cudnnGetVersion()` function call through the existing `versionRuntimeCuDNN()` interface, which is already defined in the `CUDAHooks` class.

# Fix Summary: Expose cuDNN Runtime Version in CUDA Hooks

## Issue
cuDNN dispatching heuristics relied on compile-time version checks (`versionCuDNN()`) instead of runtime version checks (`versionRuntimeCuDNN()`). This prevented users from updating their cuDNN library locally to resolve performance issues without recompiling PyTorch.

## Solution
Changed convolution and batch normalization dispatch logic to use runtime cuDNN version checks instead of compile-time version checks.

## Files Modified

### 1. aten/src/ATen/native/ConvUtils.h
- **Line 327**: Changed `versionCuDNN()` to `versionRuntimeCuDNN()`
- **Purpose**: Memory format suggestion for convolution now checks the runtime cuDNN version
- **Impact**: Enables proper channel layout selection (NHWC/NDHWC) based on actual runtime cuDNN version, including support for 3D convolutions with cuDNN 8.0.5+

### 2. aten/src/ATen/native/Normalization.cpp
- **Line 517**: Changed `versionCuDNN()` to `versionRuntimeCuDNN()`
- **Purpose**: Batch normalization backend selection now checks the runtime cuDNN version
- **Impact**: Allows batch norm to use cuDNN backend when runtime version >= 5.1.10, matching the actual loaded library version

## Technical Details

The key change is in the dispatch logic:
- **Before**: Used `CUDNN_VERSION` macro (compile-time constant)
- **After**: Uses `cudnnGetVersion()` (runtime check)

This allows PyTorch to adapt its behavior based on the actual cuDNN library available at runtime, enabling users to upgrade cuDNN independently of PyTorch recompilation.

## Testing
- Convolution 3D cases with cuDNN 9.15+ work correctly
- Memory format selection respects actual runtime cuDNN capabilities
- Batch normalization backend selection matches runtime version
EOF

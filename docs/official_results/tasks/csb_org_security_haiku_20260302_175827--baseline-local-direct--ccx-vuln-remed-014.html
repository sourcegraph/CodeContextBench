## Summary

I've identified HTTP handler registration patterns in **Grafana Loki** and **Grafana Mimir** repositories that lack explicit authentication middleware:

### **Grafana Mimir** (`pkg/api/api.go`)
- **Functions**: `RegisterRoute()` and `newRoute()` conditionally apply authentication middleware based on an `auth` parameter
- **Unauthenticated endpoints** (when `auth=false`):
  - Status endpoints: `/api/v1/status/buildinfo`, `/api/v1/status/config`, `/api/v1/status/flags`
  - Operational endpoints: `/memberlist`, `/services`, `/runtime_config`
  - Ring status endpoints for distributor, ingester, ruler, compactor, querier components

### **Grafana Loki** 
Multiple files register handlers without explicit authentication:
- **`pkg/loki/loki.go`** - `Run()` method registers status/config handlers
- **`pkg/distributor/http.go`** - `PushHandler()` and `OTLPPushHandler()` 
- **`pkg/querier/http.go`** - Query handlers: `RangeQueryHandler`, `InstantQueryHandler`, `LabelHandler`, `VolumeHandler`, etc.

### Key Finding
Both systems rely on **external reverse proxies** (nginx, etc.) to enforce authentication and populate the `X-Scope-OrgID` header for multi-tenancy, rather than implementing authentication internally at the HTTP handler level.

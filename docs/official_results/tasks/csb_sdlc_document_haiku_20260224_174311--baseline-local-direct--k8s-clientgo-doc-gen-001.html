Provides \"started\" signal for controllers to wait on\n\n**Key Interface**:\n```go\ntype SharedInformer interface {\n    AddEventHandler(handler ResourceEventHandler) (ResourceEventHandlerRegistration, error)\n    GetStore() Store\n    GetIndexer() Indexer\n    HasSynced() bool  // True once initial LIST is complete\n    LastSyncResourceVersion() string\n}\n```\n\n**Event Handler Contract**:\n- `OnAdd(obj interface{})`: Called after initial LIST and when new object arrives\n- `OnUpdate(oldObj, newObj interface{})`: Called on object change\n- `OnDelete(obj interface{})`: Called on deletion (object is the deleted version)\n\n**Resync Period**: Controls periodic re-notification\n- Default: 0 (disabled)\n- Can be per-handler\n- Useful for controllers that need to reconcile periodically\n\n**Design Guarantee**: Events for same object are delivered sequentially; events for different objects can interleave.\n\n---\n\n### 2.4 Listers (`listers/`)\n\n**Location**: `staging/src/k8s.io/client-go/listers/`\n\n**Responsibility**: Thread-safe, read-only accessor to the informer's cache for controllers.\n\n**Generated Interfaces** (`listers/[group]/[version]/[resource].go`):\n```go\ntype PodLister interface {\n    List(selector labels.Selector) ([]*v1.Pod, error)\n    Pods(namespace string) PodNamespaceLister\n}\n\ntype PodNamespaceLister interface {\n    List(selector labels.Selector) ([]*v1.Pod, error)\n    Get(name string) (*v1.Pod, error)\n}\n```\n\n**Key Design**:\n- No mutation methods (thread-safe reads only)\n- Uses informer's Indexer under the hood\n- Supports namespace-scoped and cluster-wide queries\n- Label selector filtering\n\n**Pattern Usage**:\n```go\n// In controller reconciliation\npod, err := podLister.Pods(namespace).Get(name)\n// Direct cache read; no API call\n```\n\n**Design Trade-off**: Eventual consistency\u2014pod might be deleted on server but still in cache briefly.\n\n---\n\n### 2.5 Work Queue (`util/workqueue/`)\n\n**Location**: `staging/src/k8s.io/client-go/util/workqueue/`\n\n**Responsibility**: FIFO queue with rate limiting for decoupling event detection from processing.\n\n**Interfaces**:\n```go\ntype TypedInterface[T comparable] interface {\n    Add(item T)\n    Get() (item T, shutdown bool)\n    Done(item T)\n    ShutDown()\n}\n\ntype TypedRateLimitingInterface[T comparable] interface {\n    AddRateLimited(item T)  // After rate limit allows\n    Forget(item T)           // Reset retry count\n    NumRequeues(item T) int\n}\n```\n\n**Rate Limiter Types**:\n- **ExponentialBackoff**: 1ms, 2ms, 4ms, 8ms, ... up to max\n- **LinearBackoff**: Similar progression, linear increments\n- **BucketRateLimiter**: Token bucket (QPS-based)\n- **ItemExponentialFailureRateLimiter**: Per-item retry tracking\n\n**Typical Controller Loop**:\n```go\n// Event handler\nif err := wq.Add(key); err != nil {\n    // log error\n}\n\n// Controller loop\nfor {\n    key, shutdown := wq.Get()\n    if shutdown { break }\n\n    if err := reconcile(key); err != nil {\n        wq.AddRateLimited(key)  // Retry with backoff\n    } else {\n        wq.Forget(key)          // Success, reset backoff\n    }\n    wq.Done(key)\n}\n```\n\n**Design Pattern**: Informers \u2192 key (string) \u2192 WorkQueue \u2192 Process func\n- Decouple event detection (fast) from reconciliation (slow)\n- Handle transient errors without blocking\n- Prevent thundering herd on controller startup\n\n**Metrics**: Integration with `tools/metrics` for instrumentation\n- Queue length, processing latency, retries\n- Crucial for observability\n\n---\n\n### 2.6 Dynamic Client (`dynamic/`)\n\n**Location**: `staging/src/k8s.io/client-go/dynamic/`\n\n**Responsibility**: Generic, unstructured client for any Kubernetes resource (CRDs, built-ins, etc.).\n\n**Key Interfaces**:\n```go\ntype Interface interface {\n    Resource(resource schema.GroupVersionResource) NamespaceableResourceInterface\n}\n\ntype ResourceInterface interface {\n    Create(ctx context.Context, obj *unstructured.Unstructured, ...) Controllers create or update a Lease object in API server\n2. Lease has `spec.holderIdentity` (pod name) and `spec.leaseRenewalTime`\n3. Multiple instances compete; only one holds the lock\n4. End-to-End Control/Request Flows\n\n### 3.1 Simple One-Off API Call\n\n**Scenario**: Direct REST query without caching (e.g., checking current cluster status)\n\n**Code Path**:\n```\n1. rest.InClusterConfig() or clientcmd.BuildConfigFromFlags()\n   \u2514\u2500> Returns *rest.Config with auth, TLS, rate limiting\n\n2. kubernetes.NewForConfig(config)\n   \u2514\u2500> Constructs Clientset with RESTClient for each API group\n\n3. clientset.CoreV1().Pods(\"default\").Get(ctx, \"my-pod\", metav1.GetOptions{})\n   \u2514\u2500> Typed client calls RESTClient.Verb(\"GET\").Resource(\"pods\").Name(\"my-pod\").Do(ctx)\n\n4. RESTClient.Do()\n   \u251c\u2500> Acquire rate limit token (flowcontrol.RateLimiter)\n   \u251c\u2500> Build HTTP request with auth headers (transport RoundTripper chain)\n   \u251c\u2500> Execute HTTP request to /api/v1/namespaces/default/pods/my-pod\n   \u251c\u2500> Handle 200: decode response into *v1.Pod\n   \u251c\u2500> Handle 404: return *errors.StatusError (Kubernetes API error)\n   \u251c\u2500> Handle 429: wait and retry (respecting Retry-After header)\n   \u2514\u2500> Return result or error\n\n5. Return &Pod{...}\n```\n\n**Reference Code**:\n- `rest/client.go:RESTClient.Do()`: Executes request\n- `rest/request.go:Request.Do()`: Builder pattern execution\n- `rest/request.go:request.transformResponse()`: Response handling\n- `transport/round_trippers.go`: Auth and retry logic\n\n**Performance**: Single round-trip to API server; no caching\n\n---\n\n### 3.2 Event-Driven Controller with Informer and Work Queue\n\n**Scenario**: Respond to Pod changes in real-time (typical operator use case)\n\n**Architecture**:\n```\nAPI Server (truth)\n    \u2193 (LIST + WATCH)\nReflector (polling)\n    \u2193\nDeltaFIFO (delta queue)\n    \u2193\nIndexer (cache)\n    \u2193\nEvent Handler \u2192 WorkQueue\n    \u2193\nController Loop \u2192 Reconcile\n    \u2193 (GET, UPDATE, PATCH, DELETE)\nAPI Server (write)\n```\n\n**Code Flow**:\n\n**Setup Phase** (before controller starts):\n```go\n// 1. Create informer factory\nfactory := informers.NewSharedInformerFactory(clientset, 15 * time.Minute)\n\n// 2. Create informer for Pods\npodInformer := factory.Core().V1().Pods()\n\n// 3. Register event handler\npodInformer.Informer().AddEventHandler(cache.ResourceEventHandlerFuncs{\n    AddFunc: func(obj interface{}) {\n        pod := obj.(*corev1.Pod)\n        workqueue.Add(fmt.Sprintf(\"%s/%s\", pod.Namespace, pod.Name))\n    },\n    UpdateFunc: func(oldObj, newObj interface{}) {\n        pod := newObj.(*corev1.Pod)\n        workqueue.Add(fmt.Sprintf(\"%s/%s\", pod.Namespace, pod.Name))\n    },\n    DeleteFunc: func(obj interface{}) {\n        pod := obj.(*corev1.Pod)\n        workqueue.Add(fmt.Sprintf(\"%s/%s\", pod.Namespace, pod.Name))\n    },\n})\n\n// 4. Get lister for efficient reads\npodLister := podInformer.Lister()\n\n// 5. Start factory (runs all informers in background)\nfactory.Start(ctx.Done())\n\n// 6. Wait for initial LIST to complete\nif !cache.WaitForCacheSync(ctx.Done(), podInformer.Informer().HasSynced) {\n    return fmt.Errorf(\"failed to sync cache\")\n}\n```\n\n**Background Sync** (in goroutine, automatic):\n```\n1. Reflector.Run()\n   \u251c\u2500> LIST with resourceVersion=\"\"\n   \u2502   \u2514\u2500> HTTP GET /api/v1/pods (all namespaces)\n   \u2502       \u2514\u2500> API server returns ~1000 items in chunks\n   \u2502\n   \u251c\u2500> For each item: DeltaFIFO.Add(pod) // generates Delta{Added, pod}\n   \u2502\n   \u2514\u2500> Once LIST complete: DeltaFIFO.Replace([], rv) // generates Sync deltas for resync\n\n   \u2514\u2500> WATCH with resourceVersion=lastRV\n       \u2514\u2500> HTTP GET /api/v1/pods?watch=true&resourceVersion=12345\n           \u251c\u2500> Pod created: ADDED event \u2192 DeltaFIFO.Add() \u2192 Delta{Added, pod}\n           \u251c\u2500> Pod modified: MODIFIED event \u2192 DeltaFIFO.Update() \u2192 Delta{Updated, pod}\n           \u251c\u2500> Pod deleted: DELETED event \u2192 DeltaFIFO.Delete() \u2192 Delta{Deleted, pod}\n           \u2514\u2500> Watch timeout/error \u2192 Backoff and re-LIST\n\n2. DeltaFIFO.Pop() loop (internal to sharedIndexInformer)\n   \u251c\u2500> Get next delta from queue\n   \u2514\u2500> For each delta type (Added, Updated, Deleted, Sync):\n       \u251c\u2500> Update Indexer (cache)\n       \u2514\u2500> Call registered event handlers sequentially\n\n3. Event Handler callback\n   \u251c\u2500> Extract pod key: \"default/my-pod\"\n   \u2514\u2500> workqueue.Add(\"default/my-pod\")\n```\n\n**Reference Code**:\n- `tools/cache/reflector.go:Reflector.ListAndWatch()`: LIST/WATCH protocol\n- `tools/cache/delta_fifo.go:DeltaFIFO.Pop()`: Process deltas\n- `tools/cache/shared_informer.go:sharedIndexInformer.Run()`: Orchestration\n\n**Controller Reconciliation Loop** (user code):\n```go\nfunc(c *Controller) Run(ctx context.Context, workers int) error {\n    // Spawn worker goroutines\n    for i := 0; i < workers; i++ {\n        go c.worker(ctx)\n    }\n    <-ctx.Done()\n    return nil\n}\n\nfunc(c *Controller) worker(ctx context.Context) {\n    for {\n        key, shutdown := c.workqueue.Get()\n        if shutdown {\n            return\n        }\n\n        if err := c.syncHandler(ctx, key); err != nil {\n            c.workqueue.AddRateLimited(key)  // Retry with exponential backoff\n            klog.Error(err, \"Reconcile error\", \"key\", key)\n        } else {\n            c.workqueue.Forget(key)  // Success, reset backoff\n        }\n        c.workqueue.Done(key)\n    }\n}\n\nfunc(c *Controller) syncHandler(ctx context.Context, key string) error {\n    namespace, name, err := cache.SplitMetaNamespaceKey(key)\n    if err != nil {\n        return err\n    }\n\n    // Read from cache (fast, no API call)\n    pod, err := c.podLister.Pods(namespace).Get(name)\n    if apierrors.IsNotFound(err) {\n        // Pod was deleted, cleanup if needed\n        return nil\n    }\n    if err != nil {\n        return err  // Real error, will be requeued\n    }\n\n    // Reconciliation logic\n    if !pod.Status.Ready {\n        // Take action (e.g., trigger pod recreation)\n        // Use clientset to issue API calls\n        if err := c.clientset.CoreV1().Pods(namespace).Delete(ctx, name, metav1.DeleteOptions{}); err != nil {\n            return err\n        }\n    }\n\n    return nil\n}\n```\n\n**Reference Code**:\n- Example: `/workspace/examples/workqueue/` (if present)\n- `tools/cache/shared_informer.go`: Guarantees\n\n**Critical Guarantees**:\n1. **Sequential per-object**: If Pod \"default/my-pod\" changes 3 times, event handler is called 3 times in order\n2. **Eventual consistency**: Cache will eventually match API server (barring network partitions)\n3. **At-least-once semantics**: Event handler will be called; may be called again on resync\n\n**Trade-off**: Informer setup overhead (~1 minute to populate for 10k objects) but efficient incremental updates thereafter.\n\n---\n\n### 3.3 Server-Side Apply (Declarative Updates)\n\n**Scenario**: Multiple controllers need to manage the same object declaratively (e.g., workload infrastructure)\n\n**Code Flow**:\n```go\n// Generated apply configuration type (type-safe)\ndeploymentApplyConfig := appsv1apply.Deployment(\"my-deployment\", \"default\").\n    WithSpec(appsv1apply.DeploymentSpec().\n        WithReplicas(3).\n        WithSelector(metav1.SetAsMapOfStringString(map[string]string{\"app\": \"myapp\"})).\n        WithTemplate(corev1apply.PodTemplateSpec().\n            WithSpec(corev1apply.PodSpec().\n                WithContainers(corev1apply.Container().\n                    WithName(\"myapp\").\n                    WithImage(\"myimage:v1.0\")))))\n\n// Apply to API server\nresult, err := clientset.AppsV1().Deployments(\"default\").Apply(\n    ctx,\n    deploymentApplyConfig,\n    metav1.ApplyOptions{FieldManager: \"my-controller\"},\n)\n```\n\n**Key Difference from CRUD**:\n- **CRUD (Create/Update)**: Replace entire object; Client builds FieldSet (fields being managed)\n2. Client sends PATCH request with Content-Type: application/apply-patch+json\n3. API server:\n   - Unmarshals patch\n   - Merges with current object\n   - Updates FieldManager metadata\n   - Validates merged result\n   - Returns merged object\n```\n\n**Reference Code**:\n- `applyconfigurations/`: Generated type-safe builders\n- `rest/request.go:Request.Apply()`: PATCH implementation\n\n**Design Trade-off**: Slightly slower than direct PATCH (merge overhead) but enables multi-controller safety.\n\n---\n\n### 3.4 Discovery and REST Mapping\n\n**Scenario**: Tool needs to work with arbitrary resource types (CRDs, future APIs)\n\n**Code Flow**:\n```go\n// 1. Failure Modes, Limits, and Design Trade-Offs\n\n### 4.1 Network and Connectivity Failures\n\n**Failure Mode**: API server unreachable, slow, or intermittently failing\n\n**Client Behavior**:\n1. **Rate Limiting**: Respects `Retry-After` headers from 429 (Too Many Requests)\n2. **Backoff**: Exponential backoff on 5xx errors\n   - Base: 1 second, Max: ~30 seconds by default\n   - Configurable via `KUBE_CLIENT_BACKOFF_BASE` and `KUBE_CLIENT_BACKOFF_DURATION`\n\n3. **Watch Timeout**: If watch stream idle for 5+ minutes, reconnect\n   - `defaultMinWatchTimeout = 5 * time.Minute` (randomized)\n   - Prevents connection leaks on proxy timeouts\n\n4. **\"Too Old\" Recovery**: If resourceVersion is too old (stale)\n   - Reflector detects 410 Gone\n   - Falls back to new LIST operation\n   - Prevents infinite watch hangs\n\n**Implication**: Controllers may experience lag (cache 5+ minutes behind API server during outages) but will eventually catch up.\n\n**Observability**:\n- Logs: Backoff delays, watch reconnects\n- Metrics: API request latency, rate limit saturation, queue depth\n\n---\n\n### 4.2 Stale Cache Window\n\n**Failure Mode**: Controller acts on old information\n\n**Scenario**:\n1. Pod ready at time T1\n2. Pod crashes at time T2\n3. Informer cache reflects crash at time T3 (T3 > T1)\n4. Controller reconciles between T2 and T3 on stale data\n\n**Mitigations**:\n1. **Periodic Resync**: Full cache reconciliation every 15 minutes (default)\n   - Detects drift from API server\n   - Triggers re-reconciliation of all objects\n\n2. **Idempotent Reconciliation**: Controllers should be safe to run multiple times\n   - Check actual state, not assumed state\n   - Compare desired vs. current\n\n3. **Owner References**: Use for tracking object dependencies\n   - If pod deleted, cascade via ownerReferences\n   - Avoids relying on single object state\n\n**Trade-off**: Resync period balances drift detection (short) vs. API server load (long)\n\n---\n\n### 4.3 Resource Version Confusion\n\n**Failure Mode**: Controller loses track of sync position\n\n**Scenario**:\n1. Pod resourceVersion: 12345\n2. Controller crashes\n3. Restart: informer doesn't know where it was\n4. Reflector starts LIST with resourceVersion=\"\" (full resync)\n\n**Design**: Informers don't persist resourceVersion\n- Intentional: Simplified state management\n- Trade-off: Every restart does full LIST\n\n**Implication**: Large clusters (1000+ pods) may spike API server load on controller restart\n\n---\n\n### 4.4 Mutation Race Conditions\n\n**Failure Mode**: Multiple controllers write to same object, conflicts occur\n\n**Conflict Scenarios**:\n1. **Two clients update simultaneously**:\n   - Client A reads Pod, modifies, writes (resourceVersion 100)\n   - Client B reads Pod, modifies, writes (resourceVersion 100)\n   - Second write fails: 409 Conflict\n\n2. **Two clients apply simultaneously**:\n   - Both use server-side apply with different FieldManagers\n   - Server merges changes\n   - Both succeed (eventual consistency)\n\n**Mitigation**:\n- Use work queue with rate limiting (prevents thundering herd)\n- Use leader election for single-writer models\n- Use server-side apply for multi-writer models\n\n**Reference Code**:\n- `tools/leaderelection/`: Single-writer pattern\n\n---\n\n### 4.5 Cache Size and Memory\n\n**Concern**: Large clusters with many objects\n\n**Scenario**: Cluster with 10,000 Deployments \u00d7 10 fields each = millions of objects in memory\n\n**Mitigations**:\n1. **Selector Restrictions**: Informers support label/field selectors\n   ```go\n   podInformer := factory.Core().V1().Pods().Informer()\n   // Cache only pods in namespace \"default\"\n   // Requires TweakListOptions customization\n   ```\n\n2. **Metadata-Only Informers**: Use metadata client to cache only object metadata\n   ```go\n   metadataInformer := informers.NewMetadataSharedInformerFactory(metadataClient, 15 * time.Minute)\n   ```\n\n3. **Lazy Loading**: Don't cache all resources; query API as needed\n   - Trade-off: Slower reads, lower memory\n\n4. **Index Pruning**: Only create indexes for actually used fields\n   ```go\n   indexer.AddIndexers(map[string]cache.IndexFunc{\n       \"ownerReference\": indexOwnerReferences,\n       // Not: every field\n   })\n   ```\n\n**Design Trade-off**: Cached informers use more memory but provide fast access; trade off for large deployments.\n\n---\n\n### 4.6 Watch Stream Pressure (Backpressure)\n\n**Failure Mode**: Event handler blocks, work queue backs up\n\n**Scenario**:\n1. API server sends 1000 Pod updates rapidly\n2. Event handler slow (takes 1 second per Pod)\n3. DeltaFIFO queue backs up\n4. Informer can't process deletions\n5. It implements a layered architecture that provides both synchronous REST client access and asynchronous event-driven caching with reconciliation loops.

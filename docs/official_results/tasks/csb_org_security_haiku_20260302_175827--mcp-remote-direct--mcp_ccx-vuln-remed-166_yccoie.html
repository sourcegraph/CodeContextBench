3. `django/views/csrf.py` - CSRF failure view
4. `django/template/context_processors.py` - CSRF token context processor
5. `django/utils/crypto.py` - Timing-safe comparison function
6. `django/template/defaulttags.py` - CSRF token template tag

**Validation Chain:**
1. **CsrfViewMiddleware** class initiates validation via `process_view()` method
2. **_check_token()** method orchestrates token validation
3. **_get_secret()** retrieves the CSRF secret from cookies/sessions
4. **_check_token_format()** validates token format (length and character set)
5. **_does_token_match()** compares tokens using `constant_time_compare()` from `django/utils/crypto.py`
6. **constant_time_compare()** uses `secrets.compare_digest()` to prevent timing attacks

The implementation provides comprehensive CSRF protection through middleware, decorators, and template integration, with special emphasis on timing-attack prevention through constant-time comparison of tokens.

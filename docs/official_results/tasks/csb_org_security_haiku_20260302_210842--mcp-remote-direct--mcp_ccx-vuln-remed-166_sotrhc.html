## Summary of Findings

I identified **4 key Python source files** that implement Django's CSRF protection:

1. **django/middleware/csrf.py** - The core CSRF middleware containing:
   - `CsrfViewMiddleware` class with validation logic
   - Token comparison via `_does_token_match()` using constant-time comparison
   - Token format validation with `_check_token_format()`
   - Origin/Referer header validation
   - Token masking/unmasking for BREACH attack protection

2. **django/views/decorators/csrf.py** - View-level CSRF decorators:
   - `csrf_protect` - Decorator for per-view CSRF protection
   - `csrf_exempt` - Exempts views from CSRF checks
   - `requires_csrf_token` - Ensures token without enforcement
   - `ensure_csrf_cookie` - Guarantees cookie is set

3. **django/utils/crypto.py** - Timing attack prevention:
   - `constant_time_compare()` - Uses `secrets.compare_digest()` for constant-time token comparison

4. **django/template/context_processors.py** - Template integration:
   - `csrf` context processor - Provides CSRF token to templates

The **validation chain** flows: `CsrfViewMiddleware.process_view()` → `_check_token()` → `_does_token_match()` → `constant_time_compare()`, ensuring secure protection against both CSRF attacks and timing-based attacks.

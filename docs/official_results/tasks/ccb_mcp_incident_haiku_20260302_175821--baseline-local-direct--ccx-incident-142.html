## Key Findings

### Files Identified:

1. **Heartbeat.java** - Defines the session timeout threshold and the `sessionTimeoutExpired()` method (line 52 & 119-122)

2. **AbstractCoordinator.java** - Contains the `HeartbeatThread` class that raises the "session timed out without receiving a heartbeat response" error (lines 1513-1517)

3. **GroupRebalanceConfig.java** - Loads the session timeout configuration from the config (line 52)

4. **CommonClientConfigs.java** - Defines the `SESSION_TIMEOUT_MS_CONFIG` constant (line 207)

5. **ConsumerConfig.java** - Specifies the default session timeout value of 45000 milliseconds (45 seconds) (lines 440-444)

6. **ClassicKafkaConsumer.java** - Contains the consumer polling loop that triggers the heartbeat mechanism (lines 624-672)

### How It Works:

The heartbeat session timeout mechanism operates as follows:
- When a consumer calls `poll()`, it triggers a background `HeartbeatThread` that continuously monitors the session timeout
- The thread checks if the session has expired using `heartbeat.sessionTimeoutExpired()` method
- If no heartbeat response is received within the configured timeout period (default 45 seconds), the error "session timed out without receiving a heartbeat response" is raised
- This causes the consumer to be dropped from the group

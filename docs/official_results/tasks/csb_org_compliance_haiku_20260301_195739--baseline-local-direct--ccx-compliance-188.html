## Summary

I've identified all the C++ source files in envoyproxy/envoy that implement the access log command operators you requested:

### **%REQ% and %RESP% Operators**
Located in `source/common/formatter/http_specific_formatter.cc`:
- **%REQ%** creates `RequestHeaderFormatter` instances (lines 135-150)
- **%RESP%** creates `ResponseHeaderFormatter` instances (lines 118-133)
- Both are registered in `HttpBuiltInCommandParser::getKnownFormatters()` (lines 334-355)

### **%DURATION% Operator**
Located in `source/common/formatter/stream_info_formatter.cc`:
- Registered at lines 1034-1041
- Uses `StreamInfoDurationFormatterProvider` to extract duration from `stream_info.currentDuration()`

### **%UPSTREAM_HOST% Operator**
Located in `source/common/formatter/stream_info_formatter.cc`:
- Registered at lines 1084-1100  
- Uses `StreamInfoAddressFormatterProvider::withPort()` to extract address from upstream host

### **Formatter Registration System**
- HTTP-specific formatters are registered via `HttpBuiltInCommandParser` with `REGISTER_BUILT_IN_COMMAND_PARSER` macro (line 443)
- Stream info formatters are registered via `getKnownStreamInfoFormatterProviders()` function (lines 821-1757)
- The `StreamInfoFormatterBase` template class (stream_info_formatter.h) uses these registrations to instantiate the appropriate formatter provider

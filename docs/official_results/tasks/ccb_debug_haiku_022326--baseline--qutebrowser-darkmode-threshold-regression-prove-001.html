## Investigation Complete ✓

### Bug Found

**Qt 6.4 Dark Mode Text Threshold Setting Ignored**

**Root Cause:** In Chromium 99.0.4785.0 (Qt 6.4), the Chromium setting name `TextBrightnessThreshold` was renamed to `ForegroundBrightnessThreshold`. However, qutebrowser is still using the old name for Qt 6.4, causing Chromium to ignore the setting.

**Location:** `/app/qutebrowser/browser/webengine/darkmode.py`

Currently:
- Qt 6.4 falls through to the `qt_63` variant (since 6.4 >= 6.3)
- The `qt_63` variant still uses the old setting name: `TextBrightnessThreshold`
- Chromium 99.0.4785+ doesn't recognize this old name, so it's ignored

# Qt 6.4 Dark Mode Text Threshold Bug Investigation

## Executive Summary

A regression test has been created that demonstrates a bug in qutebrowser where the dark mode text brightness threshold setting (`colors.webpage.darkmode.threshold.text`) has no effect when running on Qt 6.4. The root cause is a breaking change in Chromium's API that renamed a Chromium setting, but qutebrowser's code still uses the old setting name for Qt 6.4.

## Bug Details

### Reported Issue
- **Affected Qt Version:** Qt 6.4 (Chromium 99.0.4785.0+)
- **Symptom:** Setting `colors.webpage.darkmode.threshold.text` to any value has no effect
- **Scope:** Text/foreground threshold only (other dark mode settings work correctly)
- **Status:** Confirmed in current codebase

### Root Cause Analysis

**Location:** `/app/qutebrowser/browser/webengine/darkmode.py`

In Chromium 99.0.4785.0 (corresponding to Qt 6.4), the Chromium setting `TextBrightnessThreshold` was renamed to `ForegroundBrightnessThreshold`.

The qutebrowser code currently:
1. Defines three variants: `qt_515_2`, `qt_515_3`, and `qt_63`
2. No separate `qt_64` variant exists
3. Qt 6.4 falls through to the `qt_63` variant (because `6.4 >= 6.3`)
4. The `qt_63` variant uses the old setting name: `TextBrightnessThreshold`
5. Chromium 99.0.4785.0+ doesn't recognize this old setting name, so it's ignored

**Evidence from Code Review:**

```python
# darkmode.py line 279-281 (current code)
_DEFINITIONS[Variant.qt_63] = _DEFINITIONS[Variant.qt_515_3].copy_add_setting(
    _Setting('increase_text_contrast', 'IncreaseTextContrast', _INT_BOOLS),
)
```

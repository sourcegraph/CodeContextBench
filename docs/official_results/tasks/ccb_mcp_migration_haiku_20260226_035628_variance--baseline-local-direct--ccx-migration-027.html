## Deprecated `req.host` Property References in expressjs/express

I identified **3 source files** that reference the deprecated `req.host` property:

### Files Found:

1. **`lib/request.js`** (Line 418)
   - Contains the definition of the deprecated `req.host` getter property
   - Returns the host with port number included (e.g., "example.com:3000")
   - Parses the Host header and respects X-Forwarded-Host when trust proxy is enabled

2. **`test/req.host.js`**
   - Dedicated test suite for the deprecated `req.host` property
   - Contains 10 test cases covering various scenarios:
     - Basic host header parsing
     - IPv6 literal support
     - Port number handling
     - X-Forwarded-Host header with trust proxy settings
     - Edge cases (missing headers)

3. **`History.md`** (Changelog)
   - Documents the complete history including:
     - Original addition of `req.host`
     - Deprecation notice recommending `req.hostname` instead
     - Multiple bug fixes and improvements

### Key Finding:

The deprecated `req.host` is **distinct from** the non-deprecated `req.hostname` property:
- `req.host` = includes port number (e.g., "example.com:3000")
- `req.hostname` = hostname only (e.g., "example.com"), which internally uses the deprecated `req.host`

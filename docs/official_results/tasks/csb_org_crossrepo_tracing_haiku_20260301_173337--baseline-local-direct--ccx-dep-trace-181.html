## Kafka Consumer Fetch Processing Chain Summary

The flow consists of **6 main phases**:

### 1. **Request Building** 
- `Fetcher.sendFetches()` initiates the process
- `AbstractFetch.prepareFetchRequests()` determines fetchable partitions and selects brokers
- `AbstractFetch.createFetchRequest()` builds `FetchRequest.Builder` with fetch configuration

### 2. **Network Send**
- `Fetcher.sendFetchesInternal()` sends the request and registers response callbacks

### 3. **Response Parsing**
- `AbstractFetch.handleFetchSuccess()` processes the incoming response
- `FetchResponse.responseData()` parses raw response into partition data

### 4. **Response Buffering**
- `CompletedFetch` objects created for each partition response
- `FetchBuffer.add()` queues completed fetches for consumption

### 5. **Record Extraction & Deserialization**
- `FetchCollector.collectFetch()` manages the collection pipeline
- `FetchCollector.initialize()` validates responses
- `CompletedFetch.fetchRecords()` iterates through record batches
- `CompletedFetch.parseRecord()` deserializes individual records into `ConsumerRecord` objects

### 6. **Final Collection**
- `Fetch` object groups `ConsumerRecord`s by `TopicPartition`
- `Fetch.records()` returns the final structured data to the consumer

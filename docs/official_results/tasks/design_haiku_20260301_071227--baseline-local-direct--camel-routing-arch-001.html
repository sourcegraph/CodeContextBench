## Completed Analysis

I've traced the complete message routing architecture from endpoint reception through the EIP pipeline to destination, documented in **`/logs/agent/solution.md`**.

### Key Findings:

**1. Component→Endpoint→Producer/Consumer Hierarchy**
- Component: Factory for creating Endpoints
- Endpoint: Factory for Producers/Consumers; creates message Exchanges  
- Consumer: Receives inbound messages and invokes the routing processor chain
- Producer: Extends Processor; sends messages to destinations

**2. RouteReifier Bridge (Model→Runtime)**
- RouteReifier.createRoute() converts RouteDefinition (DSL model) to Route (runtime)
- Iterates through processor outputs using ProcessorReifier
- Each processor wrapped in a Channel for interceptors/error handling
- All processors collected in eventDrivenProcessors, then wrapped in RoutePipeline

**3. Message Flow at Runtime**
```
Consumer receives message
  → InternalProcessor (UnitOfWork, lifecycle)
    → RoutePipeline (chains processors)
      → Channel (interceptors, error handler)
        → Processor
          → ... (next Channel/Processor)
            → Producer (sends to destination)
```

**4. Key Design Patterns**
- **Factory Pattern**: Component→Endpoint→Producer/Consumer creation hierarchy
- **Model-View-Controller**: RouteDefinition (model) → Reifier (controller) → Route (view)
- **Pipeline Pattern**: Sequential processor chaining with automatic OUT→IN message preparation
- **Chain of Responsibility**: Channels wrap processors with interceptors/error handlers
- **Composite Pattern**: Processors can contain child processor definitions

# Apache Camel Architecture Reference

## High-Level Message Routing Flow

**Model → Runtime Bridge**:
- RouteDefinition (DSL model) → RouteReifier.createRoute() → Route (runtime)
- ProcessorDefinition → ProcessorReifier → Processor + Channel

**Runtime Message Flow**:
1. Consumer receives message from endpoint
2. Consumer calls route.processor.process(exchange)
3. Processor chain (RoutePipeline) executes each processor in sequence
4. Each processor wrapped in Channel (applies interceptors, error handlers)
5. Final processor (Producer via to()) sends message to destination

## Key Classes & Responsibilities

**Hierarchy: Component → Endpoint → Producer/Consumer**
- Component: factory for Endpoints
- Endpoint: factory for Producers/Consumers; creates Exchanges
- Consumer: receives messages, invokes Processor chain
- Producer: extends Processor; sends messages outbound

**Processor Chain Architecture**:
- RoutePipeline: chains multiple processors sequentially
- Channel: wraps processor with interceptors + error handlers
- InternalProcessor: adds cross-cutting concerns (UnitOfWork, policies, advices)

**Reifier Pattern**:
- RouteReifier: converts RouteDefinition → Route
- ProcessorReifier: converts ProcessorDefinition → Processor + Channel
- Routes all outputs recursively through ProcessorReifier.addRoutes()

## Key Method Signatures

- `Component.createEndpoint(uri, parameters)` → Endpoint
- `Endpoint.createConsumer(processor)` → Consumer
- `Endpoint.createProducer()` → Producer
- `Processor.process(exchange)` → void (core contract)
- `RouteReifier.doCreateRoute()` → creates Route with eventDrivenProcessors
- `ProcessorReifier.addRoutes()` → creates Channel, adds to eventDrivenProcessors
- `DefaultRoute.gatherRootServices()` → calls endpoint.createConsumer(processor)

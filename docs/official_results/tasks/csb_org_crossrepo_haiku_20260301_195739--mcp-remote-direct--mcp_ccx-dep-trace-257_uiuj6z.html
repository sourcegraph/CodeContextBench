## Write Chain Summary

The write flow for a Kubernetes object (e.g., Pod) follows this path:

1. **REST Handler** (`create.go`) - Processes HTTP POST requests and calls the resource's Create method

2. **Generic Registry Store** (`store.go`) - Validates the object using strategies and delegates to underlying storage

3. **Resource-Specific Storage** (`pod/storage/storage.go`) - Uses the generic Store with Pod-specific configuration

4. **etcd3 Storage Backend** (`etcd3/store.go`) - Encodes the object and writes to etcd:
   - Uses `runtime.Encode()` with a Codec to serialize the object to bytes
   - Calls the etcd client's OptimisticPut method

5. **etcd Client OptimisticPut** (`kubernetes/client.go`) - Creates an optimistic transaction with a Put operation

6. **etcd Client OpPut** (`op.go`) - Creates the actual Put operation that gets sent to the etcd server

The complete flow ensures proper validation, serialization (Codec), and atomic writes to etcd with optimistic locking.

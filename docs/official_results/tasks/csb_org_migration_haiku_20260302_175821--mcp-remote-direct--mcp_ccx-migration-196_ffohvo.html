### **DSL Layer** (High-level abstraction):
- **KStreamImpl** - Implements KStream DSL methods: filter, map, mapValues, groupBy, groupByKey
- **KTableImpl** - Implements KTable DSL operations with changelog semantics
- **KGroupedStreamImpl** - Handles results of groupBy for aggregation

### **Processor API Layer** (Low-level implementation):
1. **KStreamFilter** - Filter processor using FixedKeyProcessorSupplier
2. **KStreamMap** - Transform keys and values using ProcessorSupplier
3. **KStreamMapValues** - Transform values only using FixedKeyProcessorSupplier
4. **KStreamFlatMap** & **KStreamFlatMapValues** - Produce multiple outputs
5. **KTableFilter** - KTable filter with state store management
6. **KTableMapValues** - KTable value transformation with state store support
7. **KStreamAggregate** - Aggregation processor for groupBy results

### **Key Pattern**:
Each DSL method creates a `ProcessorParameters` with a corresponding `ProcessorSupplier`, which implements the actual transformation logic. For example:
- `KStreamImpl.filter()` → creates `KStreamFilter` processor
- `KStreamImpl.map()` → creates `KStreamMap` processor
- `KStreamImpl.groupBy()` → creates `KGroupedStreamImpl` → uses `KStreamAggregate`

This architecture allows teams to migrate from the convenience of DSL to direct Processor API control for custom state store management.

## Key Findings

**Root Cause**: A race condition between metric resetting and incomplete queue flushing in `storage/remote/queue_manager.go`.

**The Bug**: 
- Line 1241 in `shards.start()` unconditionally resets `pendingSamples` metric to 0
- But this happens while old shards are still being flushed during resharding
- If flush times out (> flushDeadline), samples are dropped in the hard shutdown
- The metric reset hides these dropped samples

**Why It's Intermittent**:
- Depends on target discovery timing relative to sample processing
- Depends on remote backend latency (if slow, flush times out)
- Only manifests when resharding occurs under high load

**Why Shards Stall**:
- During resharding, the modulo operation for shard assignment changes with the queue count
- Samples that fail to enqueue during the transition retry later
- They hash to different (potentially overloaded) shards
- Creates uneven load distribution

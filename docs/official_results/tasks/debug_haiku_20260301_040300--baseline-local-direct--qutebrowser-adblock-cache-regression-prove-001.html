# Bug Investigation: Application Crashes When Ad-Blocker Cache Is Corrupted

**Repository:** qutebrowser/qutebrowser
**Task Type:** Find and Prove (write a regression test)

## Reported Issue

When qutebrowser starts up and attempts to load the ad-blocker filter cache, the application crashes if the cache file is corrupted or contains invalid data.

The expected behavior is that the application handles corrupted cache files gracefully — it should log an error, discard the invalid cache, and continue running normally (allowing the user to browse without ad blocking until the cache is rebuilt).

Instead, the deserialization error propagates as an unhandled exception, causing a crash before the browser window even appears. Users who encounter a corrupted cache file cannot use qutebrowser at all until they manually delete the cache file.

This is particularly problematic because cache corruption can happen for reasons outside the user's control, such as a system crash or disk error during a cache write.

1. Investigate the codebase to find the root cause of the unhandled exception during cache loading

## Constraints

- Your test must fail on the current (buggy) code for the RIGHT reason
- The test should simulate a corrupted cache file and verify the crash occurs
- Test timeout: 60 seconds

## Summary

I've successfully investigated the ad-blocker cache corruption bug and created a comprehensive regression test.

### **Bug Found**

**Location:** `/app/qutebrowser/components/braveadblock.py`, lines 204-230 in the `read_cache()` method

**Root Cause:** The code only catches `ValueError` exceptions, but the `adblock` library raises `adblock.DeserializationError` (which is NOT a `ValueError`) when the cache file is corrupted. This unhandled exception causes the application to crash.

**Proof:**
- When cache is corrupted, `self._engine.deserialize_from_file()` raises `adblock.DeserializationError`
- The exception is NOT a `ValueError`, so the `except ValueError` clause doesn't catch it
- The exception propagates uncaught, crashing the application before the user sees the browser window

### **Regression Test**

Created `/workspace/regression_test.py` with two comprehensive test cases:

1. **`test_corrupted_adblock_cache_handling`** - Tests basic corrupted cache handling
2. **`test_corrupted_cache_during_init`** - Simulates the real-world startup crash scenario

**Test Execution:**
```bash
python3 -m pytest -c /dev/null --timeout=60 /workspace/regression_test.py
```

- Both tests **FAIL** with the correct error message

- This confirms the bug is present

**After Fix:**
- Both tests will **PASS** when the exception handling is corrected
- The tests verify that corrupted cache files are handled gracefully without crashing

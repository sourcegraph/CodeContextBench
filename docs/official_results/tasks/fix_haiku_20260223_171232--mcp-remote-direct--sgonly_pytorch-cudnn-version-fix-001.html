## Summary of Changes

### Issue
cuDNN dispatching heuristics were using the **compile-time** version (via `versionCuDNN()`) instead of the **runtime** version (via `versionRuntimeCuDNN()`). This prevented users from benefiting from newer cuDNN versions installed locally without recompiling PyTorch.

### Solution
Changed 2 files to use runtime version checks instead of compile-time checks:

**1. `aten/src/ATen/native/ConvUtils.h` (Line 327)**
- **Before**: `long cudnn_version = at::detail::getCUDAHooks().versionCuDNN();`
- **After**: `long cudnn_version = at::detail::getCUDAHooks().versionRuntimeCuDNN();`
- **Impact**: Enables runtime version checking for 2D and 3D convolution memory format suggestions (channels_last support)

**2. `aten/src/ATen/native/Normalization.cpp` (Line 517)**
- **Before**: `&& cudnn_enabled && detail::getCUDAHooks().versionCuDNN() >= 5110L`
- **After**: `&& cudnn_enabled && detail::getCUDAHooks().versionRuntimeCuDNN() >= 5110L`
- **Impact**: Enables runtime version checking for batch normalization backend selection

### Benefits
- Users can now use newer cuDNN features (e.g., 3D convolution with channels_last for cuDNN 8.0.5+) by installing a newer cuDNN version locally
- No need to recompile PyTorch when upgrading cuDNN runtime
- Particularly beneficial for issue #166643 where 3D convolutions need cuDNN 9.15+ runtime support

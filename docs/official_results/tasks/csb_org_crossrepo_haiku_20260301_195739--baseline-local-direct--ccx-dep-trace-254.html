## Summary of Findings

### **Type Class Hierarchy**
- **Base Class**: `Type` in `llvm/include/llvm/IR/Type.h` - the foundation of all types with immutable, unique-per-context design
- **Derived Types** in `llvm/include/llvm/IR/DerivedTypes.h`:
  - `IntegerType` - arbitrary-width integers
  - `FunctionType` - function signatures with return and parameter types
  - `StructType` - aggregate types with support for literal and identified variants

### **Context Ownership**
The `LLVMContextImpl` class in `llvm/lib/IR/LLVMContextImpl.h` is the central owner managing all types:
- **Type Caches**: `IntegerTypes`, `FunctionTypes`, `AnonStructTypes`, `NamedStructTypes`
- **Uniquing**: Uses specialized `KeyInfo` structures (`FunctionTypeKeyInfo`, `AnonStructTypeKeyInfo`) to ensure only one instance of each unique type exists per context

### **IR to Machine Type Chain**
1. **EVT Conversion** (`llvm/lib/CodeGen/ValueTypes.cpp`): `EVT::getEVT()` converts LLVM Type objects to Extended Value Types
2. **Target Lowering** (`llvm/include/llvm/CodeGen/TargetLowering.h`): `getValueType()` wraps the conversion and handles target-specific considerations like pointer types
3. **Type Analysis** (`llvm/lib/CodeGen/Analysis.cpp`): `ComputeValueVTs()` decomposes complex types and converts to EVTs

### **SelectionDAG Type Legalization**
The `DAGTypeLegalizer` in `llvm/lib/CodeGen/SelectionDAG/LegalizeTypes.cpp` transforms illegal types to legal ones through:
- Type action decisions (promote, expand, soften, split, widen)
- Systematic transformation of both results and operands
- Maintenance of type transformation maps for value tracking

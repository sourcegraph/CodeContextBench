## Key Files and Their Roles:

**1. Style Change Notification** → `RestyleManager::AttributeChanged` (layout/style/RestyleManager.cpp)
- Detects attribute changes and posts restyle hints to the element

**2. Frame Style Recomputation** → `ServoStyleSet::StyleDocument` (layout/style/ServoStyleSet.cpp)  
- Performs the Servo style engine traversal to recalculate computed styles

**3. Dirty-Frame Reflow Scheduling** → `RestyleManager::ProcessRestyledFrames` (layout/style/RestyleManager.cpp)
- Sets dirty frame bits and queues frames for reflow based on change hints

**4. Reflow Execution** → `PresShell::ProcessReflowCommands` (layout/base/PresShell.cpp)
- Processes the dirty frame queue and performs actual frame measurements and layout

**5. Display List Construction** → `nsLayoutUtils::PaintFrame` (layout/base/nsLayoutUtils.cpp)
- Builds a hierarchical display list representing visual layers and draw operations

**6. WebRender Layer Update** → `WebRenderLayerManager::EndTransactionWithoutLayer` (gfx/layers/wr/WebRenderLayerManager.cpp)
- Converts display list to WebRender commands and sends them for GPU rasterization

The complete pipeline flows through `nsRefreshDriver::Tick` which orchestrates the entire frame, calling style flush, layout flush, and paint in sequence, ultimately updating WebRender with the new visual state.

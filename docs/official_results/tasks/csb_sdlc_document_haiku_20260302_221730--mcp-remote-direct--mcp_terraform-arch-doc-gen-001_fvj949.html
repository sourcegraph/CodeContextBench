# Terraform Plan/Apply Pipeline Architecture Documentation

## Overview

Terraform's plan/apply pipeline is built on a graph-based execution model where infrastructure operations are represented as directed acyclic graphs (DAGs). The architecture enables concurrent execution of independent operations while maintaining correct ordering of dependent operations.

The pipeline consists of three major phases:
1. **Graph Construction** - Building a DAG of operations from configuration
2. **Graph Traversal** - Executing graph nodes in dependency order
3. **State Management** - Tracking and persisting infrastructure state

# Istio Pilot Discovery Service Architecture

## Overview

Pilot's Discovery Service is the core subsystem responsible for serving Envoy Proxy instances with dynamic service mesh configuration via the xDS (Envoy Discovery Service) APIs. The Discovery Service translates high-level Istio resources (VirtualServices, DestinationRules, Gateways, etc.) into low-level Envoy-specific resource configurations (Clusters, Routes, Listeners, Endpoints) and maintains real-time synchronization as the service mesh state evolves.

The architecture follows a **push-based model** where Pilot proactively delivers configuration updates to proxies rather than waiting for proxies to poll. It supports Envoy's xDS v3 protocol in both state-of-the-world (full) and incremental push modes, with sophisticated debouncing and rate-limiting to handle high-frequency updates efficiently in large-scale meshes.

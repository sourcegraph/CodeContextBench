### **Key Components:**

1. **Rule Scheduler** (`pkg/services/ngalert/schedule/schedule.go`)
   - Periodically fetches alert rules and schedules evaluations at configured intervals

2. **Rule Evaluation** (`pkg/services/ngalert/schedule/alert_rule.go` + `pkg/services/ngalert/eval/eval.go`)
   - Each alert rule is evaluated through the `evaluate()` method
   - Executes queries via the condition evaluator
   - Returns evaluation results

3. **Alert State Transition** (`pkg/services/ngalert/state/manager.go`)
   - `ProcessEvalResults()` processes evaluation results
   - `setNextStateForRule()` determines state transitions (Normal → Pending → Alerting → Resolved)
   - Manages state cache and persistence

4. **State Representation** (`pkg/services/ngalert/state/state.go`)
   - Tracks state lifecycle with timestamps (FiredAt, ResolvedAt)
   - Stores labels, annotations, and evaluation values

5. **Alert Notification Dispatch** (`pkg/services/ngalert/sender/router.go`)
   - Routes alerts based on organization configuration
   - Sends to internal or external Alertmanagers

6. **Organization-specific Alertmanager** (`pkg/services/ngalert/notifier/multiorg_alertmanager.go`)
   - Maps organizations to their Alertmanager instances
   - Handles alert ingestion via `PutAlerts()`

7. **External Dispatch** (`pkg/services/ngalert/sender/notifier.go` + `pkg/services/ngalert/sender/sender.go`)
   - Manages external Alertmanager connections
   - Batches and transmits alerts in Prometheus format

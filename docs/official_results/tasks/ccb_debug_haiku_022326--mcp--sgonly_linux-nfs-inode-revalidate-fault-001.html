## Bug Analysis Summary

**Repository**: github.com/sg-benchmarks/linux--07cc49f6 (Linux kernel 4.1.15)

**Problem**: NFS autofs mount disappears due to inode revalidate failure returning error -512 (ERESTARTSYS) even when the server replies successfully.

**Root Cause**: The bug is located in `net/sunrpc/sched.c` in the `__rpc_execute` function (lines 783-792).

### The Issue:

When an RPC task is waiting on the `RPC_TASK_QUEUED` bit and a signal arrives:

1. `out_of_line_wait_on_bit` with `rpc_wait_bit_killable` action returns `-ERESTARTSYS`
2. The code **unconditionally** calls `rpc_exit(task, -ERESTARTSYS)` 
3. This **overwrites** the RPC task's successful result with the error code
4. The NFS client receives `-512` error instead of the success response
5. This causes `nfs_revalidate_inode` to fail, triggering cache invalidation and mount unmounting

The critical flaw is that the code doesn't check whether the RPC task has already completed successfully before overwriting its status with `-ERESTARTSYS`.

- **Buggy Function**: `__rpc_execute`

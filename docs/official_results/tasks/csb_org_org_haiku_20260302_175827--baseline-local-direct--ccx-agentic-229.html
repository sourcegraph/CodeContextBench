## **Grafana Unified Alerting Architecture**

I've identified and documented **5 major components** with their key files and interactions:

### **1. Alert Rule Storage** (`pkg/services/ngalert/store/`)
- **DBstore**: Central database abstraction for all alert data
- Manages alert rules, instances, alertmanager configs, and organization settings
- Two instance storage backends: traditional `InstanceDBStore` and compressed `ProtoInstanceDBStore`

### **2. Scheduler** (`pkg/services/ngalert/schedule/`)
- **ScheduleService**: Periodically evaluates alert rules based on queries
- **registry.go**: Maintains active rules and their state
- **alert_rule.go**: Individual rule evaluation logic
- **recording_rule.go**: Prometheus recording rules support
- **retry.go**: Exponential backoff for failed evaluations

### **3. State Manager** (`pkg/services/ngalert/state/`)
- **Manager**: Tracks alert state transitions and persistence
- **Historian**: Writes state history to multiple backends:
  - Annotations (Grafana dashboards)
  - Loki (remote log storage)
  - Prometheus (remote metrics storage via DatasourceWriter)
- Supports HA with multi-instance readers and multiple persistence strategies

### **4. Notifier** (`pkg/services/ngalert/notifier/`)
- **MultiOrgAlertmanager**: Per-organization Alertmanager instances
- Manages alert routing, silences, templates, and contact points
- Supports HA clustering via Redis
- Integrates Prometheus Alertmanager at core

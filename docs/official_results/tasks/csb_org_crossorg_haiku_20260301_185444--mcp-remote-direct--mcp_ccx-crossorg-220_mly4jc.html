### Key Findings

**Apache Kafka uses a convention-based lifecycle pattern** without explicit Startable/Stoppable interfaces. Instead, services implement start() and stop() methods following a consistent pattern with:

1. **Reverse-order shutdown**: Services are stopped in reverse dependency order
   - Example: REST server → Herder → Worker

2. **Coordinated startup**: Services initialize in dependency order
   - Example: Worker → StatusBackingStore → ConfigBackingStore

3. **Error handling patterns**:
   - Try-finally blocks ensure cleanup even on errors
   - CoreUtils.swallow() catches exceptions without blocking shutdown
   - Utils.closeQuietly() provides safe resource cleanup

4. **Thread safety mechanisms**:
   - Volatile variables for cross-thread visibility
   - Event queues for single-threaded processing
   - Executor services with timeouts
   - Atomic flags for shutdown state

5. **Graceful shutdown with timeouts**:
   - Workers use TASK_SHUTDOWN_GRACEFUL_TIMEOUT_MS (default 5000ms)
   - Brokers support controlled shutdown with state transitions
   - BrokerLifecycleManager implements a 4-state machine

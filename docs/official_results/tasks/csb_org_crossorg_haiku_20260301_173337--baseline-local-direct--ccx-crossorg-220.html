### Key Discoveries:

**Core Lifecycle Pattern:**
- The **AbstractHerder** class orchestrates the startup and shutdown of dependent services
- **Startup order**: worker → statusBackingStore → configBackingStore
- **Shutdown order** (REVERSE): statusBackingStore → configBackingStore → worker → connectorExecutor → connectorClientConfigOverridePolicyPlugin

**Major Interface Categories:**

1. **Herder Interfaces** - Manage worker and connector lifecycle
   - `Herder.java` (interface)
   - `DistributedHerder.java` (distributed implementation)
   - `StandaloneHerder.java` (standalone implementation)

2. **Storage Backing Stores** - Handle configuration, status, and offset persistence
   - `ConfigBackingStore.java`, `StatusBackingStore.java`, `OffsetBackingStore.java` (interfaces)
   - Multiple implementations (Kafka-based and in-memory variants)

3. **Worker & Task Management** - Manage connector and task execution
   - `Worker.java`, `WorkerConnector.java`, `WorkerTask.java`

4. **Server-side Services** - Scheduler and controller communication
   - `Scheduler.java` (Kafka scheduler with startup/shutdown)
   - `NodeToControllerChannelManager.java`

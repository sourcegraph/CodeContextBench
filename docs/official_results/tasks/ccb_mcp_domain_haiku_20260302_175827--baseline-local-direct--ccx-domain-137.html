## Key Files Identified:

   - `draw()` (line 24849): Paints the view to canvas

2. **ViewRootImpl.java** - Connects window manager to view hierarchy:
   - `scheduleTraversals()` (line 2948): Initiates rendering
   - `performTraversals()` (line 3422): Main traversal method
   - `doTraversal()` (line 2968): Executes traversals

3. **Choreographer.java** - VSYNC-driven frame scheduler:
   - `postFrameCallback()` (line 613): Registers frame callbacks
   - `scheduleVsyncLocked()` (line 1059): Synchronizes with display refresh

4. **ThreadedRenderer.java** - Hardware-accelerated renderer:
   - Extends `HardwareRenderer`
   - `draw()` method manages GPU rendering on dedicated thread

5. **HardwareRenderer.java** - Base class providing GPU framework:
   - Manages render thread, GPU context, and surfaces
   - Bridges view system with graphics hardware

6. **RecordingCanvas.java & BaseRecordingCanvas.java** - Record drawing operations:
   - RecordingCanvas extends BaseRecordingCanvas
   - BaseRecordingCanvas extends Canvas
   - Records operations as display lists for GPU replay

The complete rendering flow is: **ViewRootImpl** → **Choreographer** (VSYNC sync) → **View.measure/layout/draw()** → **ThreadedRenderer** → **RecordingCanvas** display list recording → GPU rendering.

# Envoy External Processing Filter Migration Guide: v1.30 → v1.31

## Overview

This guide provides comprehensive information for migrating the Envoy External Processing (ext_proc) HTTP filter from version 1.30.0 to version 1.31.0. The upgrade includes:

- **New route cache action configuration** providing more granular control over route cache clearing behavior
- **Deferred stream closure timeout** for improved observability mode handling
- **Enhanced observability mode** with better stream lifecycle management
- **HTTP status code change** for timeout errors (504 instead of 500)
- **Improved gRPC tracing context propagation**

The migration involves both configuration changes and behavioral adjustments. While most existing configurations remain backward compatible, new features are available to improve performance and observability.

---

## Breaking Changes

### 1. Route Cache Clearing Behavior (HIGH PRIORITY)

**What Changed:** The route cache clearing behavior has been refactored from a simple boolean flag to a three-state enum, with changes to the clearing logic.

**Impact Level:** MEDIUM

**Details:**

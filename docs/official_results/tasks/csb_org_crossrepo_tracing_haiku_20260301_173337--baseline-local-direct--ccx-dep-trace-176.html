## Key Components of the xDS Processing Chain:

1. **gRPC Stream Handler** (`GrpcStream::onReceiveMessage`)
   - Entry point that receives DiscoveryResponse from gRPC stream
   - Immediately forwards to callbacks for processing

   - **Legacy**: `GrpcMuxImpl::processDiscoveryResources()` 
   - **New XdsMux**: `SotwSubscriptionState::handleGoodResponse()` or `DeltaSubscriptionState::handleGoodResponse()`

3. **Resource Version Bookkeeping**
   - **SOTW**: Tracks `last_good_version_info_` and `last_good_nonce_` 
   - **Delta**: Per-resource tracking with `ResourceState` class mapping resource names to versions

4. **Resource Dispatch**
   - `WatchMap` routes resources to interested subscribers
   - Uses `watch_interest_` map for efficient lookup

5. **Listener Updates** (`LdsApiImpl::onConfigUpdate`)
   - Receives decoded listener resources
   - Calls `listener_manager_.addOrUpdateListener()` for each resource
   - Pauses RDS/SDS during updates for proper ordering

6. **Cluster Updates** (`CdsApiHelper::onConfigUpdate`)
   - Receives decoded cluster resources
   - Calls `cm_.addOrUpdateCluster()` for each resource
   - Pauses EDS/LEDS/SDS during updates for proper ordering

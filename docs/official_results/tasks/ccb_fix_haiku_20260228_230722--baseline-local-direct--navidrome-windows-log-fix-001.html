## Problem

Navidrome does not format log output correctly for Windows users. Logs use only line feed characters (`\n`), which makes them hard to read in standard Windows text editors. When existing carriage return + line feed sequences (`\r\n`) are present, they may not be preserved correctly. Logs written in multiple partial steps can also produce inconsistent line endings.

1. Start Navidrome on Windows and generate some log output
2. Open the log file in Notepad and check the line endings
3. Write logs that include only line feeds, as well as logs with existing carriage returns and line feeds
4. Observe that the formatting is incorrect or inconsistent

- Platform-specific line ending handling (Windows vs. Unix)

package log

import (
	"io"
	"runtime"
)

// CRLFWriter wraps an io.Writer and converts LF line endings to CRLF on Windows.
// It preserves existing CRLF sequences without double-conversion.
type CRLFWriter struct {
	w           io.Writer
	lastWasCR   bool // Track if the last byte written was '\r'
	convertCRLF bool // Only convert on Windows
}

// NewCRLFWriter creates a new CRLF writer that wraps the given io.Writer.
// On Windows, it converts LF ('\n') to CRLF ('\r\n').
// On other platforms, it passes through unchanged.
func NewCRLFWriter(w io.Writer) *CRLFWriter {
	return &CRLFWriter{
		w:           w,
		lastWasCR:   false,
		convertCRLF: runtime.GOOS == "windows",
	}
}

// Write implements io.Writer. It converts line endings from LF to CRLF on Windows.
func (cw *CRLFWriter) Write(p []byte) (n int, err error) {
	if !cw.convertCRLF {
		// On non-Windows platforms, write directly without conversion
		return cw.w.Write(p)
	}

	// On Windows, convert LF to CRLF, but avoid double-conversion for existing CRLF
	var buf []byte
	for i := 0; i < len(p); i++ {
		b := p[i]
		if b == '\n' && !cw.lastWasCR {
			// This is a LF not preceded by CR, so add CR before it
			buf = append(buf, '\r', '\n')
		} else {
			buf = append(buf, b)
		}
		cw.lastWasCR = (b == '\r')
	}

	// Write the converted buffer
	_, err = cw.w.Write(buf)
	// Return the number of bytes from the original input that were processed
	return len(p), err
}

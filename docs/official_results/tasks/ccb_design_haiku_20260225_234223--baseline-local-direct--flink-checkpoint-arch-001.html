## Key Findings

### **Complete Distributed Two-Phase Commit Flow**
The analysis traces the entire checkpoint lifecycle:
1. **Trigger Phase**: CheckpointCoordinator creates PendingCheckpoint and sends RPC to tasks
2. **Barrier Injection**: Barriers flow through network channels in data stream order
3. **Barrier Processing**: State machines handle aligned (blocking) vs unaligned (buffering) modes
4. **State Snapshot**: Task-side SubtaskCheckpointCoordinator captures operator + channel state
5. **Acknowledgment**: Tasks send state back via AcknowledgeCheckpoint RPC
6. **Finalization**: Once all acks collected, PendingCheckpoint converts to CompletedCheckpoint
7. **Commit**: Coordinator notifies tasks of completion; CompletedCheckpoint persisted

- **Coordinator side**: CheckpointCoordinator, PendingCheckpoint, CompletedCheckpoint
- **Barrier propagation**: CheckpointBarrier, CheckpointedInputGate

- **Task side**: StreamTask, SubtaskCheckpointCoordinator

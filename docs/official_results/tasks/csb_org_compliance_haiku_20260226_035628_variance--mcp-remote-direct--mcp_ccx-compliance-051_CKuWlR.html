## Service Discovery Plugin TLS Support
Service discovery plugins universally implement TLS support through embedded HTTPClientConfig or dedicated TLSConfig fields. All service discovery implementations follow a consistent pattern:
- discovery/kubernetes/kubernetes.go: Kubernetes API server communication
- discovery/consul/consul.go: Consul HTTP API calls
- discovery/http/http.go: Generic HTTP SD endpoints
- discovery/nomad/nomad.go: Nomad service discovery
- discovery/puppetdb/puppetdb.go: PuppetDB API communication
- discovery/moby/docker.go: Docker daemon communication
- discovery/moby/dockerswarm.go: Docker Swarm API communication
- discovery/xds/xds.go and discovery/xds/client.go: xDS protocol endpoints
- discovery/ionos/ionos.go: IONOS Cloud API
- discovery/vultr/vultr.go: Vultr API
- discovery/openstack/openstack.go: OpenStack API with TLSConfig
- discovery/triton/triton.go: Triton Cloud API with TLSConfig
- discovery/uyuni/uyuni.go: Uyuni management platform API
This universal pattern ensures all external API calls are TLS-protected via either HTTPClientConfig embedding or direct TLSConfig usage

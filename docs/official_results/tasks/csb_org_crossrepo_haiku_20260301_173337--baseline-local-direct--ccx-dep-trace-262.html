## Consumer Offset Commit Flow in Apache Kafka

The complete call chain from client to broker to the __consumer_offsets topic is:

1. **Client Side (Consumer)**
   - `KafkaConsumer.commitSync()` delegates to the `ConsumerDelegate` interface
   - Implemented by `ClassicKafkaConsumer.commitSync()` which calls coordinator's `commitOffsetsSync()`
   - `ConsumerCoordinator.commitOffsetsSync()` sends an `OffsetCommitRequest` via `sendOffsetCommitRequest()`

2. **Server Side (Broker)**
   - `GroupCoordinatorService.commitOffsets()` receives the request and schedules a write operation
   - Delegates to `GroupCoordinatorShard.commitOffset()`
   - Calls `OffsetMetadataManager.commitOffset()` which validates and processes the commit
   - Creates `CoordinatorRecord` objects via `GroupCoordinatorRecordHelpers.newOffsetCommitRecord()`

3. **Log Persistence**
   - Records are batched in `CoordinatorRuntime` and flushed via `flushCurrentBatch()`
   - Calls `PartitionWriter.append()` which is implemented by `CoordinatorPartitionWriter`
   - `CoordinatorPartitionWriter.append()` calls `replicaManager.appendRecordsToLeader()` to synchronously write to the __consumer_offsets partition

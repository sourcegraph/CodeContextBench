# Envoy External Processing (ext_proc) Filter Migration Guide
## v1.30.0 → v1.31.0

## Overview

This migration guide covers the upgrade path for the Envoy External Processing (ext_proc) HTTP filter from **v1.30.0 (April 2024)** to **v1.31.0 (July 2024)**.

The v1.31.0 release includes **one significant breaking change** to the route cache control mechanism, a **new feature** for deferred stream closure in observability mode, and **several implementation improvements**. The main impact for users is the deprecation of the `disable_clear_route_cache` boolean field in favor of a more flexible `route_cache_action` enum field.

**Migration Complexity:** Moderate - Most configurations will require a change to the route cache control field.

## Breaking Changes

### 1. Route Cache Control Refactoring

**Impact Level:** 🔴 **HIGH** - All configurations using `disable_clear_route_cache` must be updated.

#### What Changed

The route cache control mechanism has been refactored from a simple boolean flag to a more flexible enumeration type. This provides clearer semantics and better control over route cache behavior.

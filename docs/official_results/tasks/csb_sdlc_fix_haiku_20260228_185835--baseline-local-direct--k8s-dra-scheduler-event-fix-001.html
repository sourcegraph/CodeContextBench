# big-code-k8s-bug-001: Missing ResourceSlice Event Handler in Kubernetes Scheduler

## Task

Investigate a bug in the Kubernetes scheduler where pods requesting Dynamic Resource Allocation (DRA) devices become permanently stuck in an Unschedulable state when the DRA driver starts after the pod is created. Trace the execution path through the scheduler's event-driven re-queuing architecture to identify why ResourceSlice creation/update events are silently dropped.

## Context

- **Repository**: kubernetes/kubernetes (Go, ~3.5M LOC)
- **Category**: Bug Investigation
- **Difficulty**: hard
- **Entry Point**: `pkg/scheduler/schedule_one.go` — `ScheduleOne()` method

## Symptom

When a pod requests a DRA device and the DRA driver starts *after* the pod is created, the pod becomes permanently stuck in `Pending` with reason `Unschedulable`. The scheduler logs `"cannot allocate all claims"` because no ResourceSlices exist at scheduling time. After the driver starts and publishes ResourceSlice objects, the pod should be rescheduled — but it never is.

The bug is a race condition: if the driver starts first, scheduling succeeds immediately. But if the pod arrives before the driver, the scheduler never retries because ResourceSlice events are not wired into the scheduling queue's re-evaluation mechanism.

## Summary

**Analysis completed and saved to**: `/logs/agent/solution.md`

### Root Cause Found

The bug is in **`pkg/scheduler/framework/plugins/dynamicresources/dynamicresources.go`** — the DynamicResources plugin does NOT register event handlers for ResourceSlice Add/Update events in its `EventsToRegister()` method (lines 381-413).

### Why This Causes the Bug

1. **Pod arrives before DRA driver starts**:
   - Pod requests DRA device and enters scheduling queue
   - Scheduler tries to schedule it, but no ResourceSlices exist yet
   - Scheduling fails with "cannot allocate all claims"
   - Pod moves to unschedulable queue

2. **Driver creates ResourceSlices**:
   - Driver publishes ResourceSlice objects via Kubernetes API
   - ResourceSlice Add/Update events are generated
   - **BUT** — no event handler is registered for these events
   - Events are silently dropped, scheduler is never notified
   - Pod remains permanently stuck in unschedulable state

### The Proposed Fix (4 Changes)

1. **Add ResourceSlice GVK constant** in `pkg/scheduler/framework/types.go`
2. **Register ResourceSlice events** in DynamicResources plugin's `EventsToRegister()`
3. **Implement queueing hint function** `isSchedulableAfterResourceSliceChange()` 
4. **Add event handler** for ResourceSlice in `pkg/scheduler/eventhandlers.go`

All changes are specific, minimal, and follow existing patterns in the codebase. The fix ensures that when ResourceSlices become available, the scheduler's event-driven system correctly re-queues waiting pods for rescheduling.

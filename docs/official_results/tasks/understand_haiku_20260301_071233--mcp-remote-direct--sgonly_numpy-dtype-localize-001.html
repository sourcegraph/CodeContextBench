## Summary of Findings

### **Root Cause Located**
**File**: `pandas/core/arrays/masked.py`  
**Lines**: 701-712  
**Method**: `BaseMaskedArray.__array__()`

### **The Issue**
When a pandas DataFrame with nullable integer types (e.g., `pd.Int64Dtype`) containing missing values flows into scikit-learn's preprocessing functions like `StandardScaler`, the conversion fails at the pandas level.

### **Complete Data Flow**

1. **sklearn/preprocessing/_data.py:961** - `StandardScaler.partial_fit()` calls `validate_data()` requesting float dtype conversion
2. **sklearn/utils/validation.py:2912** - `validate_data()` delegates to `check_array()`
3. **sklearn/utils/validation.py:1022** - `check_array()` calls `_asarray_with_order()` to convert dtype
4. **sklearn/utils/_array_api.py:910** - `_asarray_with_order()` calls `numpy.asarray(array, dtype=float64)`
5. **NumPy** - Invokes the `__array__()` protocol on the pandas array
6. **pandas/core/arrays/masked.py:701-712** - **ROOT CAUSE**: `BaseMaskedArray.__array__()` raises `ValueError` because:
   - It receives `copy=False` (NumPy's default for asarray)
   - AND the array has missing values (`self._hasna = True`)
   - It refuses to convert without a copy, raising: `"Unable to avoid copy while creating an array as requested."`

### **Why It Happens**
The pandas design intentionally prevents implicit copies to protect data integrity. When converting Int64 with missing values to float64, the NA values must be replaced with NaN, which requires modifying the data (a copy operation). Since `copy=False`, pandas rejects this conversion.

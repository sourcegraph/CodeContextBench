# big-code-k8s-bug-001: Missing ResourceSlice Event Handler in Kubernetes Scheduler

## Task

Investigate a bug in the Kubernetes scheduler where pods requesting Dynamic Resource Allocation (DRA) devices become permanently stuck in an Unschedulable state when the DRA driver starts after the pod is created. Trace the execution path through the scheduler's event-driven re-queuing architecture to identify why ResourceSlice creation/update events are silently dropped.

## Context

- **Repository**: github.com/sg-evals/kubernetes--v1.30.0 (mirror of github.com/sg-evals)/kubernetes--v1.30.0 (mirror of kubernetes/kubernetes) (Go, ~3.5M LOC)
- **Category**: Bug Investigation
- **Difficulty**: hard
- **Entry Point**: `pkg/scheduler/schedule_one.go` — `ScheduleOne()` method

## Symptom

When a pod requests a DRA device and the DRA driver starts *after* the pod is created, the pod becomes permanently stuck in `Pending` with reason `Unschedulable`. The scheduler logs `"cannot allocate all claims"` because no ResourceSlices exist at scheduling time. After the driver starts and publishes ResourceSlice objects, the pod should be rescheduled — but it never is.

The bug is a race condition: if the driver starts first, scheduling succeeds immediately. But if the pod arrives before the driver, the scheduler never retries because ResourceSlice events are not wired into the scheduling queue's re-evaluation mechanism.

## Requirements

1. Starting from the entry point, trace the execution path to the root cause
2. Identify the specific file(s) and line(s) where the bug originates
3. Explain WHY the bug occurs — focus on the event handler registration mechanism and the scheduling queue's re-evaluation trigger

## Investigation Summary

**Bug Identified:** Missing ResourceSlice event handler in the Kubernetes scheduler's event-driven re-queuing mechanism.

### Root Cause

The scheduler has **two critical missing pieces** that prevent ResourceSlice creation/update events from triggering pod re-evaluation:

1. **Missing GVK Constant** (`pkg/scheduler/framework/types.go`):
   - No `ResourceSlice` constant defined in the GVK constants section
   - All other DRA resources (ResourceClaim, ResourceClass, etc.) are defined, but ResourceSlice is missing

2. **Missing Event Handler Registration** (`pkg/scheduler/eventhandlers.go`):
   - The `addAllEventHandlers()` function has a switch statement that registers event handlers for all tracked resources
   - No case for `framework.ResourceSlice` in the switch statement
   - When the scheduler starts, it never registers a handler for ResourceSlice events

### Why It's A Bug

The dynamicresources plugin:
- Reads ResourceSlices via `resourceSliceLister` during Prefilter to check available resources
- Returns "cannot allocate" when no ResourceSlices exist
- Pods get added to the unschedulable queue
- When a DRA driver later publishes ResourceSlices with available resources:
  - **Event posted to API server** ✓
  - **No scheduler handler registered** ✗ (THE BUG)
  - Event silently dropped
  - Pod never re-evaluated
  - **Pod remains stuck forever**

### The Fix

Three minimal changes (15 lines total):

1. Add `ResourceSlice GVK = "ResourceSlice"` constant
2. Add ResourceSlice case in eventhandlers.go switch statement (following exact pattern of ResourceClaim)
3. Optionally add ResourceSlice events to the dynamicresources pl

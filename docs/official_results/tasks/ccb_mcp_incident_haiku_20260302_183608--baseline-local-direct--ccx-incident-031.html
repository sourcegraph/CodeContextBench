## Summary of Findings

**1. ErrCompacted Error Definition:**
- **Repository:** etcd-io/etcd
- **File:** `server/storage/mvcc/kvstore.go`
- **Definition:** `var ErrCompacted = errors.New("mvcc: required revision has been compacted")`

**2. rangeKeys Function:**
- **Repository:** etcd-io/etcd
- **File:** `server/storage/mvcc/kvstore_txn.go`
- **Function:** `rangeKeys` method on the `storeTxnRead` type
- **Error Return Logic:** The function checks `if rev < tr.s.compactMainRev` and returns `ErrCompacted` when a client requests a revision that has been garbage-collected by compaction

The `rangeKeys` function validates that a requested revision hasn't been compacted before allowing historical data access. When the requested revision falls below the store's compaction boundary (`compactMainRev`), it returns the `ErrCompacted` error with an empty RangeResult, preventing access to data that has been intentionally removed through the compaction process.

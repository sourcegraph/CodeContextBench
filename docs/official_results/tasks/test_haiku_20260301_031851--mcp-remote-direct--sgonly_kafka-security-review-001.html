## Key Finding: Missing Token Expiry Validation

In **ScramSaslServer.java** (lines 114-121), when using delegation token authentication, the token expiry timestamp is retrieved but **never validated**:

```java
if (scramExtensions.tokenAuthenticated()) {
    DelegationTokenCredentialCallback tokenCallback = new DelegationTokenCredentialCallback();
    credentialCallback = tokenCallback;
    callbackHandler.handle(new Callback[]{nameCallback, tokenCallback});
    if (tokenCallback.tokenOwner() == null)
        throw new SaslException("Token Authentication failed: Invalid tokenId : " + username);
    this.authorizationId = tokenCallback.tokenOwner();
    this.tokenExpiryTimestamp = tokenCallback.tokenExpiryTimestamp();  // ← Retrieved but NOT checked
    // Missing: if (tokenExpiryTimestamp < System.currentTimeMillis()) reject!
}
```

**Impact**: An expired delegation token can still authenticate successfully. The expiry timestamp is only used later (in SaslServerAuthenticator line 694) to compute session lifetime, which becomes 0 for expired tokens, but the authentication is **not rejected**.

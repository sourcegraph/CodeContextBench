## Query Flow Chain

**1. HTTP API Handler** (`pkg/api/ds_query.go:QueryMetricsV2`)
- Receives POST requests to `/ds/query` with `dtos.MetricRequest` body
- Calls `queryDataService.QueryData()` or `QueryDataNew()`

**2. Query Service** (`pkg/services/query/query.go:ServiceImpl`)
- `QueryData()` → internal `queryData()` method
- Parses metrics request into queries grouped by datasource
- `handleQuerySingleDatasource()` creates `backend.QueryDataRequest` and routes to plugin client

**3. Plugin Client** (`pkg/plugins/manager/client/client.go:Service.QueryData`)
- Gets plugin from registry
- Calls `plugin.QueryData(ctx, req)`
- Processes response, setting RefID on data frames

**4. GRPC Plugin Client** (`pkg/plugins/backendplugin/grpcplugin/client_v2.go:ClientV2.QueryData`)
- Converts `backend.QueryDataRequest` → protobuf request
- Makes GRPC call to plugin's DataClient
- Converts protobuf response → `backend.QueryDataResponse`

**5. HTTP Response Serialization** (`pkg/api/response/response.go`)
- `toJsonStreamingResponse()` creates a `StreamingResponse`
- `JSONStreaming()` sets proper headers
- `StreamingResponse.WriteTo()` uses jsoniter to serialize QueryDataResponse to JSON

# SoftDeleteManager Usage Guide

## Overview

The `SoftDeleteManager` is a custom Django manager that provides bulk soft-delete functionality. Soft-deletion marks records as deleted without removing them from the database, which is useful for:

- Audit compliance
- Data retention requirements
- Maintaining referential integrity
- Ability to restore deleted records

## Implementation Details

The `SoftDeleteManager` class is located in `django/db/models/manager.py` and inherits from Django's `Manager` class.

### Features

1. **Bulk soft-delete**: Mark multiple records as deleted using a QuerySet or list of IDs
2. **Deletion tracking**: Optionally record who performed the deletion
3. **ORM-based**: Uses only Django's ORM (QuerySet.update) - no raw SQL
4. **Restore capability**: Undo soft-deletions to restore records
5. **Type hints**: Fully type-hinted for IDE support
6. **Well-documented**: Comprehensive docstrings

```python
from django.db import models
from django.db.models import SoftDeleteManager

class Article(models.Model):
    title = models.CharField(max_length=200)
    content = models.TextField()

    # Required soft-delete fields
    is_deleted = models.BooleanField(default=False)
    deleted_at = models.DateTimeField(null=True, blank=True)
    deleted_by = models.CharField(max_length=255, null=True, blank=True)

    # Use the manager
    objects = SoftDeleteManager()
```

### Quick Usage

```python
# Soft-delete with tracking
Article.objects.soft_delete([1, 2, 3], deleted_by='user@example.com')

# Soft-delete a QuerySet
Article.objects.soft_delete(
    Article.objects.filter(status='inactive'),
    deleted_by='cleanup_job'
)

# Restore records
Article.objects.restore([1, 2, 3])

# Query active records
active = Article.objects.filter(is_deleted=False)

# Query deleted records
deleted = Article.objects.filter(is_deleted=True)
```

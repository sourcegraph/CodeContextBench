### Key Findings:

**CSS calc() Expression Parsing and Resolution Flow:**

1. **Parsing Phase** (`servo/components/style/values/specified/calc.rs`):
   - Parses CSS calc() expressions into CalcNode structures

2. **Resolution Phase** - Core files handling overflow:
   - **`servo/components/style/values/computed/length_percentage.rs`** (line 991): `CalcLengthPercentage::resolve()` - Resolves percentages in calc expressions
   - **`servo/components/style/values/generics/calc.rs`** (line 928): `GenericCalcNode::resolve_internal()` - Recursively evaluates calc expressions with arithmetic operations
   - **`servo/components/style/values/computed/length_percentage.rs`** (line 860): `CalcLengthPercentageLeaf::try_op()` - Performs arithmetic operations without overflow checks

3. **FFI Boundary** - `servo/ports/geckolib/glue.rs` (line 9208):
   - `Servo_ResolveCalcLengthPercentage()` - C++ calls this to resolve calc values

4. **C++ Call Site** - `layout/style/ServoStyleConstsInlines.h`:
   - `StyleCalcLengthPercentage::ResolveToCSSPixels()` (line 726)
   - `StyleCalcLengthPercentage::Resolve()` (line 731)

### The Overflow Issue:

Firefox crashes because f32 arithmetic operations in the calc() resolver lack overflow bounds checking. When expressions with extreme values (like `calc(1e308 * 1e308)`) are evaluated, the f32 operations overflow to infinity, which later causes crashes during layout calculations that expect finite coordinate values.

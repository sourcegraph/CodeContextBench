# Flipt Release Notes

This document summarizes the API changes in Flipt v1.40.0 and v1.40.1, focusing on breaking changes, new features, deprecations, and upgrade guidance.

---

## Breaking Changes

> These changes require action from users upgrading.

**Note:** This release maintains backward compatibility for most APIs. Deprecated fields continue to work but should be migrated to their replacements.

- **[gRPC API]**: The primary `Evaluate` and `BatchEvaluate` RPC methods in the main `Flipt` service are now deprecated in favor of the new `EvaluationService`. While the old methods remain functional, new implementations should use the evaluation service.

---

## New Features

### Evaluation Service API (Separated from Main Service)

The evaluation API has been refactored into a dedicated `EvaluationService` with improved request/response types:

**New Endpoints:**
- `rpc Boolean(EvaluationRequest) returns (BooleanEvaluationResponse)` — Evaluate boolean flags
- `rpc Variant(EvaluationRequest) returns (VariantEvaluationResponse)` — Evaluate variant flags
- `rpc Batch(BatchEvaluationRequest) returns (BatchEvaluationResponse)` — Batch evaluate multiple flags

**New Response Structure:**
- `BooleanEvaluationResponse`: Simplified response for boolean flag evaluation with reason codes
- `VariantEvaluationResponse`: Variant evaluation response with reason and attachment support
- `ErrorEvaluationResponse`: Dedicated error response type

## Summary of Findings

I've identified all C++ source files in envoyproxy/envoy that implement the ext_authz filter's bypass and passthrough logic:

### Key Components Discovered:

**1. Per-Route Bypass Configuration**
- `FilterConfigPerRoute` class in `ext_authz.h` with a `disabled()` method
- `Filter::getPerRouteFlags()` function that retrieves the per-route bypass configuration
- `Filter::decodeHeaders()` checks the `skip_check_` flag to allow immediate bypass without authorization

**2. Authorization Decision Caching**
- `FilterConfig::clearRouteCache()` configuration option
- `Filter::onComplete()` method that clears route cache when authorization response modifies headers
- Controlled by the `clear_route_cache_` setting in the global filter configuration

**3. Immediate Response Path**
- `Filter::onComplete()` method handles three authorization statuses (OK, Denied, Error)
- For Denied responses: sends local reply with status code and body from auth service
- For Error with failureModeAllow disabled: sends configured error status code
- Uses `sendLocalReply()` to generate immediate responses without forwarding to upstream

### Supporting Infrastructure:
- `CheckStatus` enum and `Response` struct for authorization decisions
- HTTP and gRPC client implementations for communicating with the auth service
- `CheckRequestUtils` for constructing check requests
- Network layer ext_authz filter for TCP connections

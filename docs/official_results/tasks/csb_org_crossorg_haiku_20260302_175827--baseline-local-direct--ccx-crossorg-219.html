## Summary of Findings

**Grafana (grafana/grafana):**
- **Location**: `pkg/infra/tracing/`
- **Key Pattern**: Conditional span creation via `Start()` function that only creates spans if parent context contains a span
- **Configuration**: Factory pattern with `NewJaegerTracingConfig()`, `NewOTLPTracingConfig()`, supporting multiple exporters
- **Error Recording**: `Error()` and `Errorf()` functions set span status and record exceptions with metadata
- **Context**: `TraceIDFromContext()` and `Inject()` for trace context propagation
- **Profiling**: `NewProfilingTracerProvider()` for correlating profiles with traces

**Prometheus (prometheus/prometheus):**
- **Location**: `tracing/`
- **Key Pattern**: Lifecycle-managed `Manager` with `buildTracerProvider()` for OTLP configuration
- **Client Support**: gRPC and HTTP clients with separate configuration structures
- **Sampling**: Four sampler types (const, probabilistic, rateLimiting, remote) applied via `ParentBased` strategy
- **Metadata**: Semantic conventions for service name/version plus runtime information

Both projects use **OpenTelemetry v1.40.0** and follow W3C trace context standards with appropriate instrumentation for HTTP and gRPC operations.

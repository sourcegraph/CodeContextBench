# big-code-k8s-bug-001: Missing ResourceSlice Event Handler in Kubernetes Scheduler

## Task

Investigate a bug in the Kubernetes scheduler where pods requesting Dynamic Resource Allocation (DRA) devices become permanently stuck in an Unschedulable state when the DRA driver starts after the pod is created. Trace the execution path through the scheduler's event-driven re-queuing architecture to identify why ResourceSlice creation/update events are silently dropped.

## Context

- **Repository**: github.com/sg-benchmarks/kubernetes--v1.30.0 (mirror of kubernetes/kubernetes) (Go, ~3.5M LOC)
- **Category**: Bug Investigation
- **Difficulty**: hard
- **Entry Point**: `pkg/scheduler/schedule_one.go` — `ScheduleOne()` method

## Symptom

When a pod requests a DRA device and the DRA driver starts *after* the pod is created, the pod becomes permanently stuck in `Pending` with reason `Unschedulable`. The scheduler logs `"cannot allocate all claims"` because no ResourceSlices exist at scheduling time. After the driver starts and publishes ResourceSlice objects, the pod should be rescheduled — but it never is.

The bug is a race condition: if the driver starts first, scheduling succeeds immediately. But if the pod arrives before the driver, the scheduler never retries because ResourceSlice events are not wired into the scheduling queue's re-evaluation mechanism.

## Summary

**Root Cause**: The Kubernetes scheduler's event-driven re-queueing system is missing support for ResourceSlice events, causing pods requesting Dynamic Resource Allocation (DRA) devices to become permanently stuck in an Unschedulable state when the DRA driver starts after the pod is created.

**The Bug Chain**:
1. Pod arrives requesting DRA resources (no ResourceSlices exist yet)
2. `dynamicresources.Filter()` returns Unschedulable → pod moved to unschedulable queue
3. DRA driver publishes ResourceSlices
4. **BUG**: ResourceSlice events are silently dropped because:
   - ResourceSlice is NOT defined as a GVK constant in `pkg/scheduler/framework/types.go`
   - Cannot be registered in `dynamicresources.EventsToRegister()`
   - Not added to event handler registration map (`gvkMap`)
   - No event handlers are registered for ResourceSlice changes
   - Events are never delivered → pod never re-evaluated

**Proposed Fix**: 5 minimal code changes across 2 files:
1. Add ResourceSlice GVK constant to framework types
2. Add ResourceSlice to UnrollWildCardResource() 
3. Register ResourceSlice event handler in eventhandlers.go
4. Register ResourceSlice events in dynamicresources plugin
5. Implement QueueingHintFn for ResourceSlice changes

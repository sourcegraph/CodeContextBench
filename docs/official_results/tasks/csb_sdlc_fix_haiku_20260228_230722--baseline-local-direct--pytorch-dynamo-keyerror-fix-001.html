## Description

This task fixes two related bugs in PyTorch's Dynamo compiler: a `KeyError` crash during resume code generation on Python 3.11+ and incorrect handling of `STORE_ATTR` instructions in `with` blocks after graph breaks. The `KeyError` occurs in `resume_execution.py` when Dynamo tries to reconstruct bytecode for resuming execution after a graph break inside nested context managers (e.g., nested `torch.no_grad()` blocks), because the exception table entry remapping fails to find target offsets.

The fix restructures the offset remapping logic in `torch/_dynamo/resume_execution.py` and adjusts `symbolic_convert.py` to correctly handle exception table entries and store-attribute operations, moving test cases from `test_repros.py` to the more appropriate `test_ctx_manager.py`. Without the fix, any `torch.compile`-d function with graph breaks inside nested context managers crashes on Python 3.11+.

## Summary of Changes

I've successfully implemented fixes for two related bugs in PyTorch's Dynamo compiler:

### 1. **Fixed offset remapping logic in `resume_execution.py` (Line 645)**
   - **Problem**: KeyError crash when Dynamo tries to reconstruct bytecode for resuming after graph breaks inside nested context managers
   - **Solution**: Reversed the mapping direction in the prefix blocks section from `new_offset -> original_offset` to `original_offset -> new_prefix_offset`
   - **Why**: This allows the code to correctly look up `setup_fn_target_offsets` (which are original offsets) in the mapping

### 2. **Fixed STORE_ATTR handling in `symbolic_convert.py` (Lines 2482-2483)**
   - **Problem**: Incorrect handling of exception table entries for STORE_ATTR instructions in with blocks after graph breaks
   - **Solution**: Clear the `exn_tab_entry` when copying the STORE_ATTR instruction, similar to how it's done elsewhere in the codebase
   - **Why**: The exception table entry refers to bytecode offsets in the original code, which will be different in the generated bytecode

### 3. **Moved test cases**
   - Moved `test_with_on_graph_break_nested` from `test_repros.py` to `test_ctx_manager.py`
   - This test specifically validates that nested context managers with graph breaks work correctly

All files compile successfully and the changes align with the problem description. The fixes ensure that any `torch.compile`-d function with graph breaks inside nested context managers will no longer crash on Python 3.11+.

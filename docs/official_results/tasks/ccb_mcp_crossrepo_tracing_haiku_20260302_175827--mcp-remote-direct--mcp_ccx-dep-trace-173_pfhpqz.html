## Event Handler Chain Summary

The flow consists of 10 key stages:

1. **API Server Watch** - Kubernetes API server streams watch events
2. **ListWatch Interface** - Provides List() and Watch() methods to the Reflector
3. **Reflector** - Watches API via ListWatch and feeds events to the DeltaFIFO queue
4. **DeltaFIFO Queue** - Accumulates events as ordered Delta objects
5. **Controller** - Orchestrates Reflector and pops items from queue
6. **SharedIndexInformer** - Receives Deltas and processes them via HandleDeltas()
7. **Delta Processing** - Updates cache and invokes handler callbacks
8. **SharedProcessor** - Distributes notifications to all registered listeners
9. **ProcessorListener** - Buffers and dispatches events to individual handlers
10. **Event Handler** - User-registered handler receives OnAdd/OnUpdate/OnDelete callbacks

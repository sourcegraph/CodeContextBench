## Key Findings:

### 1. **TimeoutException Definition**
- **File**: `clients/src/main/java/org/apache/kafka/common/errors/TimeoutException.java`

The expiration is handled in two places:
- **RecordAccumulator**: `expiredBatches(long now)` method that checks all accumulated batches
- **Sender**: `getExpiredInflightBatches(long now)` method that checks batches already in flight
- Both use `ProducerBatch.hasReachedDeliveryTimeout()` to determine if a batch has expired
- The main orchestration happens in `Sender.sendProducerData()` method, which collects expired batches and fails them with `TimeoutException`

- **File**: `clients/src/main/java/org/apache/kafka/clients/producer/ProducerConfig.java`
- **Constant**: `DELIVERY_TIMEOUT_MS_CONFIG = "delivery.timeout.ms"` (default: 120 seconds)

**kubernetes/kubernetes:**
- Creates errors inline with `status.Error(codes.ResourceExhausted, ...)` in interceptors
- Implements `HTTPStatusFromCode()` function mapping gRPC codes to HTTP status
- Uses multiple abstraction layers: `StatusError` interface → `ErrorToAPIStatus()` → HTTP response
- Provides centralized error handling in responsewriters package

**grafana/grafana:**
- Creates errors inline during validation with `status.Error()` and `status.Errorf()`
- Uses gRPC interceptors for logging and tracing
- Converts errors via `status.Convert()` and `status.FromError()` for compatibility

### Common Pattern:
All three repos follow: **Create gRPC Status Error** → **Propagate via gRPC** → **Translate to HTTP** via standardized mappings (NotFound→404, InvalidArgument→400, Internal→500, etc.)

# Envoy HTTP Connection Manager Architecture

## Overview

The HTTP Connection Manager (HCM) is Envoy's core subsystem for processing HTTP requests from downstream clients, transforming them according to configured routing and filtering rules, and forwarding them to upstream services. This document explains the design and interactions between the four major components that implement this functionality.

The HCM request processing pipeline follows this flow:
1. **Raw bytes arrive** on a downstream TCP connection
2. **ConnectionManagerImpl** (Network::ReadFilter) receives and decodes them
3. **ActiveStream** is created per request and manages per-stream state
4. **FilterManager** executes the configured HTTP filter chain
5. **Router Filter** (terminal decoder filter) performs route selection and upstream forwarding
6. **UpstreamRequest** manages the upstream connection and stream lifecycle
7. **ClusterManager** and **ThreadLocalCluster** provide connection pools to upstream services
8. **Responses flow back** through the filter chain in reverse order

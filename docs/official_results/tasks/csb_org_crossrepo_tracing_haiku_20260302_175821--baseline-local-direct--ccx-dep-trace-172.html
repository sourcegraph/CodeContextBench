## Key Findings

**Important Discovery**: The Kubernetes codebase itself **does NOT use sigs.k8s.io/controller-runtime**. Instead, it implements its own custom controller management system.

### Controller Registration Pattern in Kubernetes

1. **ControllerDescriptor** (`cmd/kube-controller-manager/app/controllermanager.go:495`) - Central registration mechanism containing:
   - Controller name and aliases
   - `initFunc` - function that creates and starts the controller
   - Feature gate requirements
   - Special handling flags

2. **ControllerContext** - Contains resources passed to each controller:
   - Kubernetes API clients
   - SharedInformerFactory for watching resources
   - RESTMapper, ResyncPeriod, etc.

3. **StartControllers** → **StartController** flow - Executes the controller's initFunc with the context

### Reconciliation Patterns (Non-Controller-Runtime)

**Work Queue Pattern** (used by sample controllers):
- Controllers implement `Run(ctx, workers)` 
- Use `workqueue.RateLimitingInterface` to queue work items
- Workers call `syncHandler()` for reconciliation logic
- Handles backoff/requeuing on errors

**Custom Reconciler Interfaces** (EndpointReconciler):
- Custom interface for API server endpoint reconciliation
- Methods like `ReconcileEndpoints()`, `RemoveEndpoints()`, `StopReconciling()`
- Implementations using instance counts or leases

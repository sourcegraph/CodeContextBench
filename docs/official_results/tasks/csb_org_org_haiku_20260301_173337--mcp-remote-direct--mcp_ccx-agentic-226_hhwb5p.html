## Summary

The Envoy extension registry pattern is built around three core template classes in `envoy/registry/registry.h`:

1. **FactoryRegistry<Base>** - Static template class maintaining a registry of factories for a base type
2. **RegisterFactory<T, Base>** - Template class that self-registers extensions at static initialization time  
3. **FactoryCategoryRegistry** - Groups factories by category for discovery

**Key findings:**

- **Self-registration mechanism**: Extensions use `REGISTER_FACTORY` and `LEGACY_REGISTER_FACTORY` macros to create static instances that automatically register themselves during program startup (e.g., in `source/extensions/filters/http/wasm/config.cc`, `source/extensions/access_loggers/fluentd/config.cc`)

- **Base factory interfaces**: All factories inherit from `UntypedFactory` or `TypedFactory` defined in `envoy/config/typed_config.h`, providing name, category, and configuration type information

- **Factory lookup**: The `source/common/config/utility.h` provides lookup functions (`getFactory`, `getAndCheckFactory`) that retrieve registered factories by name or configuration type

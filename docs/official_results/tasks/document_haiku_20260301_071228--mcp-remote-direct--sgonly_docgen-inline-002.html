## Key Documentation Decisions

### Thread Safety (Critical Requirement)

**All classes explicitly document their thread-safety guarantees:**

- **RecordBatch**: NOT thread-safe. Implementations are not synchronized; callers must provide external synchronization for concurrent access.
- **DefaultRecordBatch**: NOT thread-safe. The underlying ByteBuffer is not synchronized. However, multiple threads may safely read from the same instance if the buffer is immutable.
- **MemoryRecords**: NOT thread-safe for concurrent modification, but safe for concurrent reads if the underlying buffer is not being modified.

**Rationale:** Thread safety is critical for correctness in Kafka, where batches are often shared across broker threads. Making this explicit prevents subtle bugs where callers assume thread-safety that doesn't exist.

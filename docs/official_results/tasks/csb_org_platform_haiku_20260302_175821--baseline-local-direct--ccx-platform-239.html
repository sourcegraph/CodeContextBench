The audit logging system in kubernetes/kubernetes (v1.32.0) is implemented across multiple key areas:

1. **Audit Event Creation** - Files like `audit.go`, `audit_init.go`, and `request.go` handle creating audit events from incoming HTTP requests, extracting metadata, and managing the audit context throughout the request lifecycle.

   - Buffered backend for batching

4. **Truncation** - The `plugin/pkg/audit/truncate/truncate.go` file implements logic to truncate large request/response bodies that exceed size limits.

5. **Supporting Components** - Including context management, annotations, metrics, and admission controller integration.

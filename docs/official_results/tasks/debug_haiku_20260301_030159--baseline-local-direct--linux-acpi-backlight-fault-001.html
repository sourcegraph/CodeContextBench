## Bug Report

**Bug ID**: 207835
**Summary**: ACPI video backlight brightness control does not work on Acer TravelMate 5735Z by default

On the Intel GM45/GMA 4500MHD laptop with Debian Sid/unstable and Linux 5.6.7 and GNOME Shell 3.36.2, trying to change the brightness of the internal display using the function keys, GNOME shows the OSD, which seems to have five levels (from acer_wmi?), but the actual brightness does *not* change. There is only `/sys/class/backlight/intel_backlight/brightness` though, and writing values to it seems to work.

Booting the system with `acpi_backlight=vendor`, exposes `/sys/devices/platform/acer-wmi/backlight/acer-wmi/brightness`, but the behavior is the same.

Booting the system with `acpi_backlight=native` or `acpi_backlight=native`, changing the brightness with the function keys works, and there now seem to be 15 (or 16) levels.

## Summary

I have identified the source of the ACPI video backlight control failure on the Acer TravelMate 5735Z. Here are my findings:

### **Buggy File**
- **`drivers/acpi/video_detect.c`**

### **Buggy Function/Data Structure**
- **`video_detect_dmi_table`** (lines 112-377) - Missing DMI entry for the Acer TravelMate 5735Z

### **Root Cause**
The Acer TravelMate 5735Z is not in the `video_detect_dmi_table`, which contains device-specific quirks for backlight selection. When the system boots:

1. The auto-detection logic in `acpi_video_get_backlight_type()` falls through to the default case
2. Since the device has ACPI video backlight capability, it defaults to `acpi_backlight_video`
3. However, the ACPI video backlight doesn't work properly on this Intel GM45/GMA 4500MHD hardware
4. The bug report confirms that native backlight (`acpi_backlight=native`) works correctly

### **Why This Happens**
- Device is a 2011 model (pre-Windows 8 era) - fails the Windows 8 BIOS check
- Not in the DMI table - no specific quirk entry for this model
- Falls back to ACPI video backlight, which doesn't work for this hardware

### **The Fix Needed**
Add a DMI entry for Acer TravelMate 5735Z to the `video_detect_dmi_table` with the `video_detect_force_native` callback to force native backlight, which is known to work on this device.

### **Confidence Level**: 0.90
Strong evidence from the bug report and code analysis confirms this localization.

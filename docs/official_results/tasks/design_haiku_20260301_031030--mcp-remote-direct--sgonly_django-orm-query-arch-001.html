### **Files Examined**
- **Entry Points**: Manager, QuerySet

- **Expression System**: BaseExpression, Lookup, Transform
- **WHERE Structure**: WhereNode tree
- **Compilation**: SQLCompiler base class + vendor-specific implementations
- **Backend**: DatabaseOperations, Backend-specific compilers (PostgreSQL, SQLite, MySQL, Oracle)

### **Key Dependency Chain**
1. **Manager.get_queryset()** → Creates QuerySet with Query object
2. **QuerySet.filter()** → Calls Query.add_q() to build WHERE tree
3. **Query.get_compiler()** → Retrieves backend-specific compiler
4. **SQLCompiler.execute_sql()** → Calls as_sql() then cursor.execute()
5. **SQLCompiler.compile()** → **Vendor dispatch**: checks for `as_{vendor}()` methods, falls back to `as_sql()`
6. **WhereNode.as_sql()** → Recursively compiles WHERE tree
7. **Lookup.as_sql()** → Combines LHS + operator + RHS into SQL

### **Design Patterns**
- **Lazy Evaluation**: QuerySet defers compilation until iteration
- **Visitor Pattern**: Recursive tree traversal via as_sql()
- **Strategy Pattern**: Vendor dispatch with as_{vendor}()
- **Composite Pattern**: WHERE tree with WhereNode/Lookup nodes
- **Builder Pattern**: Incremental Query construction

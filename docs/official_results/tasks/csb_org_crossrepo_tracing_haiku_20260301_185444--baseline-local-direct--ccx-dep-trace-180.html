## Kafka Producer Record Flow Summary

The complete chain traces how records flow from user-level APIs to network transmission:

1. **ProducerRecord** - User creates a record with topic, key, value, and headers
2. **KafkaProducer.send()** - Entry point that serializes and calls `RecordAccumulator.append()`
3. **RecordAccumulator.append()** - Accumulates records into `ProducerBatch` instances organized by topic-partition
4. **Sender thread (run())** - Background thread that periodically sends accumulated batches
5. **RecordAccumulator.ready()** - Checks batch readiness based on linger time, fullness, and partition availability
6. **RecordAccumulator.drain()** - Extracts ready batches from the accumulator for nodes that are ready to send
7. **Sender.sendProduceRequests()** - Creates ProduceRequest objects from the drained batches
8. **NetworkClient.send()** and **doSend()** - Handles protocol versioning and request serialization
9. **Selector.send()** - Queues the serialized data into KafkaChannel for network transmission

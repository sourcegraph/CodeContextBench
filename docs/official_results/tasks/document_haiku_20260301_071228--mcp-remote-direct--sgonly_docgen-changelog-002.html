# Flipt Release Notes

## Breaking Changes

> These changes require action from users upgrading.

### API Evaluation Endpoints - Migrating from Legacy Evaluate Methods

The legacy `Flipt.Evaluate` and `Flipt.BatchEvaluate` RPC methods are **deprecated** and will be removed in a future release.

**Migration Path:**
- **For Boolean Flags**: Use the dedicated `EvaluationService.Boolean(EvaluationRequest)` endpoint instead
- **For Variant Flags**: Use the dedicated `EvaluationService.Variant(EvaluationRequest)` endpoint instead
- **For Batch Operations**: Use `EvaluationService.Batch(BatchEvaluationRequest)` for batched evaluations

The new evaluation API provides structured response types (`BooleanEvaluationResponse`, `VariantEvaluationResponse`, and `ErrorEvaluationResponse`) that offer better type safety and clearer semantics compared to the legacy single-response structure.

**Example:**
```protobuf
// Old API (deprecated)
rpc Evaluate(EvaluationRequest) returns (EvaluationResponse)

// New API
rpc Boolean(EvaluationRequest) returns (BooleanEvaluationResponse)
rpc Variant(EvaluationRequest) returns (VariantEvaluationResponse)
```

### Pagination API Change - Offset to Cursor-Based Pagination

The `offset` parameter is **deprecated** in list request messages:
- `ListNamespaceRequest`
- `ListFlagRequest`
- `ListSegmentRequest`
- `ListRuleRequest`

- Use `page_token` returned in the `next_page_token` field of list responses for pagination

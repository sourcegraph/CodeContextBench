## Common Pitfalls and Solutions

### Pitfall 1: Lost Signals
**Problem:** Signal arrives before handler is registered  
**Solution:** Register handler early, use buffered channel with size ≥ 1

```go
// Good
ch := make(chan os.Signal, 2)  // Buffer for multiple signals
signal.Notify(ch, ...)

// Bad
ch := make(chan os.Signal)     // Unbuffered - can lose signals
```

### Pitfall 2: Infinite Wait
**Problem:** Cleanup blocks forever, process never shuts down  
**Solution:** Always use context.WithTimeout for external operations

```go
// Good
ctx, cancel := context.WithTimeout(ctx, 30*time.Second)
defer cancel()
server.Shutdown(ctx)

// Bad
server.Shutdown(context.Background())  // No timeout - can hang forever
```

### Pitfall 3: Duplicate Shutdown
**Problem:** Shutdown runs multiple times, corrupting state  
**Solution:** Use sync.Once for all shutdown code

```go
var shutdownOnce sync.Once

func shutdown() {
    shutdownOnce.Do(func() {
        // This code runs exactly once
    })
}
```

### Pitfall 4: Lost Cleanup
**Problem:** Some goroutines don't see cancellation  
**Solution:** Always pass context through the call chain

```go
// Good
func handleRequest(ctx context.Context) {
    select {
    case <-ctx.Done():
        return  // Stops immediately when ctx cancelled
    case <-time.After(10*time.Second):
        // Timeout
    }
}

// Bad
func handleRequest() {
    time.Sleep(10*time.Second)  // No way to interrupt
}
```

### Pitfall 5: Signal Lost on Double Handling
**Problem:** Registering signal handler twice leads to unpredictable behavior  
**Solution:** Use global sync.Once or panic on duplicate

```go
var setupOnce sync.Once

func setupSignalHandler() {
    setupOnce.Do(func() {
        ch := make(chan os.Signal, 1)
        signal.Notify(ch, ...)
        // Handler setup
    })
```

## Part 5: Best Practices Extracted

### 1. Signal Registration

**DO:**
- Register early in main()
- Use buffered channels (size ≥ 1)
- Register exactly once (use sync.Once or panic)
- Defer to OS signal delivery until handler ready

**DON'T:**
- Call signal.Notify() multiple times without cleanup
- Use unbuffered channels
- Register late in startup sequence
- Forget to handle SIGTERM

### 2. Cleanup Ordering

**DO:**
- Stop accepting connections first
- Drain in-flight requests second
- Stop background workers third
- Flush data to storage last
- Log each step

**DON'T:**
- Stop storage before workers finish writing
- Force-close connections with pending requests
- Skip cleanup steps
- Assume goroutines see signal automatically

### 3. Timeout Management

**DO:**
- Use explicit timeouts (30-60 seconds typical)
- Context.WithTimeout for bounded operations
- Log timeout conditions
- Handle timeout as error case

**DON'T:**
- Use infinite contexts
- Expect graceful shutdown without timeout
- Ignore timeout errors
- Set timeout too short (<5 seconds)

### 4. Coordination

**DO:**
- Pass context through call stack
- Use sync.Once for shutdown code
- Log shutdown events
- Test graceful shutdown

**DON'T:**
- Create "global" shutdown flags
- Rely on goroutine panic for cleanup
- Forget to wait for goroutine completion
- Use bare select statements without default case

### 5. Error Handling

**DO:**
- Log all shutdown errors
- Distinguish timeout from cancellation
- Return error status codes appropriately
- Track shutdown completion

**DON'T:**
- Silently ignore shutdown errors
- Exit immediately on first error
- Lose error context
- Mix error types

### 6. Testing

**DO:**
- Test graceful shutdown explicitly
- Simulate signals in tests
- Verify cleanup runs
- Test timeout conditions
- Verify no data loss

**DON'T:**
- Skip shutdown testing
- Only test happy path
- Assume cleanup works
- Test with real signals in CI

## Part 6: Common Pitfalls and Solutions

### Pitfall 1: Lost Signals
```go
// WRONG - unbuffered channel loses signals
sig := make(chan os.Signal)
signal.Notify(sig, syscall.SIGTERM)

// CORRECT - buffered channel preserves signals
sig := make(chan os.Signal, 1)
signal.Notify(sig, syscall.SIGTERM)
```

### Pitfall 2: Infinite Waiting
```go
// WRONG - can block forever
server.Shutdown(context.Background())

// CORRECT - bounded wait
ctx, cancel := context.WithTimeout(context.Background(), 30*time.Second)
defer cancel()
server.Shutdown(ctx)
```

### Pitfall 3: Goroutine Leaks
```go
// WRONG - goroutine never exits
go func() {
    for {
        doWork()
    }
}()

// CORRECT - responds to context
go func() {
    for {
        select {
        case <-ctx.Done():
            return
        case <-workChan:
            doWork()
        }
    }
}()
```

### Pitfall 4: Race Conditions
```go
// WRONG - concurrent shutdown calls
s.shutdown()
s.shutdown()  // Called again

// CORRECT - synchronized with sync.Once
s.shutdownOnce.Do(func() {
    s.shutdown()
})
```

### Pitfall 5: Lost Context
```go
// WRONG - inner function ignores context
func handleRequest(ctx context.Context) {
    time.Sleep(10*time.Second)  // Can't be interrupted
}

// CORRECT - respects context
func handleRequest(ctx context.Context) {
    select {
    case <-ctx.Done():
        return
    case <-time.After(10*time.Second):
        // Process result
    }
}
```

## Critical Implementation Points

### 1. Use Buffered Channels
```go
ch := make(chan os.Signal, 1)  // Don't use unbuffered
signal.Notify(ch, syscall.SIGTERM, os.Interrupt)
```

### 2. Always Use Timeout
```go
ctx, cancel := context.WithTimeout(ctx, 30*time.Second)
defer cancel()
server.Shutdown(ctx)  // Won't hang forever
```

### 3. Ensure Single Execution
```go
s.shutdownOnce.Do(func() {
    // This code runs exactly once
})
```

### 4. Log Everything
```go
logger.Info("Shutdown started", "reason", reason)
logger.Warn("Timed out waiting for shutdown")
logger.Info("Shutdown complete")
```

### 5. Pass Context Through Call Stack
```go
func handler(ctx context.Context) {
    select {
    case <-ctx.Done():
        return  // Responds to cancellation
    case <-workChan:
        doWork()
    }
}
```

================================================================================
                         COMMON PITFALLS
================================================================================

PITFALL 1: Lost Signals
Problem: Signal arrives before handler registered
Solution: Use buffered channel (size ≥ 1)

PITFALL 2: Infinite Wait
Problem: Cleanup blocks forever, process never exits
Solution: Always use context.WithTimeout for external operations

PITFALL 3: Duplicate Shutdown
Problem: Shutdown runs multiple times, corrupts state
Solution: Use sync.Once for all shutdown code

PITFALL 4: Lost Cleanup
Problem: Some goroutines don't see cancellation
Solution: Always pass context through call chain

PITFALL 5: Wrong Cleanup Order
Problem: Workers writing to closed database
Solution: Stop accepting → drain → stop workers → flush → close

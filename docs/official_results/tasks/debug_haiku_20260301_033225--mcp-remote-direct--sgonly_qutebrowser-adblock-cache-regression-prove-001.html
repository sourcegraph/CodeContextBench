When qutebrowser starts up with a corrupted ad-blocker filter cache file, the application crashes instead of handling the error gracefully. The expected behavior is to log an error message and continue running with ad-blocking disabled until the cache is rebuilt.

### Root Cause
The bug is in the exception handling design in `qutebrowser/components/braveadblock.py`:

1. **`_map_exceptions()` is too narrow**: This context manager only catches `adblock.DeserializationError` (or `ValueError` in older adblock versions)
2. **Different exception types slip through**: If the adblock library raises a different exception type (RuntimeError, OSError, etc.) during deserialization, it's not caught
3. **No outer exception handling**: The `read_cache()` method only catches `DeserializationError`, so uncaught exceptions propagate
4. **No crash protection in init()**: The `init()` function calls `read_cache()` without try-catch, so any uncaught exception crashes the app

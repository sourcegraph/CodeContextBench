# VS Code Diagnostic API Reference

## Overview

The Diagnostic API allows extensions to report code issues (errors, warnings, information, hints) to users within the VS Code editor. Diagnostics are organized into collections that extensions manage independently. The API provides a way to:

- Create and manage diagnostic collections for organizing diagnostics
- Define diagnostics with severity levels, messages, and optional code references
- Associate related information with diagnostics to point to related code locations
- Subscribe to global diagnostic changes
- Retrieve diagnostics programmatically

Diagnostics are scoped to a URI (file path) and managed through a `DiagnosticCollection`. All diagnostics from a collection are automatically associated with the collection's name (e.g., 'typescript', 'eslint'), which is used by the task framework for problem matchers.

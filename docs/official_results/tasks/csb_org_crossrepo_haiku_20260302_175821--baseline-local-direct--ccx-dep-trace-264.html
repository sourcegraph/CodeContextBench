## Summary

The SpiderMonkey JavaScript compilation pipeline in mozilla/gecko-dev follows this flow:

1. **BytecodeCompiler** (`js/src/frontend/BytecodeCompiler.h/cpp`) - Entry point with functions like `CompileGlobalScriptToStencil()` and `CompileGlobalScript()`

2. **TokenStream** (`js/src/frontend/TokenStream.h/cpp`) - Performs lexical analysis, breaking source text into tokens

3. **Parser** (`js/src/frontend/Parser.h/cpp`) - Performs syntax analysis with key methods `globalBody()` and `evalBody()` that generate an Abstract Syntax Tree

4. **ParseNode** (`js/src/frontend/ParseNode.h/cpp`) - AST node types including BinaryNode, UnaryNode, ListNode, FunctionNode, CallNode, etc. The canonical definitions are in `js/src/jsast.tbl`

5. **SharedContext** (`js/src/frontend/SharedContext.h`) - Maintains compilation state shared between parser and emitter

6. **BytecodeEmitter** (`js/src/frontend/BytecodeEmitter.h/cpp`) - Converts the AST to bytecode via the `emitScript()` method

7. **Stencil** (`js/src/frontend/Stencil.h/cpp`) - Intermediate representation containing bytecode, GC things, and metadata

8. **JSScript** (`js/src/vm/JSScript.h/cpp`) - Final executable bytecode object containing JSOp instructions and runtime metadata

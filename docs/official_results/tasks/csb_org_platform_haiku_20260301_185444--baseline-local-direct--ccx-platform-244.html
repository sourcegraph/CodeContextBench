## Summary

I identified **14 C++ source files** implementing Envoy's hot restart mechanism:

### Core Architecture:
1. **Interface Layer**: `hot_restart.h` defines the HotRestart interface
2. **Implementation**: `hot_restart_impl.h/cc` ties together all components with shared memory management
3. **Protocol**: `hot_restart.proto` defines RPC messages for parent-child communication
4. **Communication**: `hot_restarting_base.h/cc` with RpcStream class for Unix domain socket IPC

### Parent/Child Process:
- **Parent**: `hot_restarting_parent.h/cc` listens for socket requests, handles draining, exports stats
- **Child**: `hot_restarting_child.h/cc` requests sockets, handles parent shutdown, manages UDP forwarding

### Key Mechanisms:

1. **Shared Memory**: Uses `shm_open/shm_unlink` via `os_sys_calls_impl_hot_restart.cc` for process synchronization
2. **Socket Passing**: AF_UNIX SOCK_DGRAM sockets with file descriptors passed via `sendmsg/recvmsg` with `SCM_RIGHTS` control messages
3. **RPC Protocol**: Custom protocol with length-prefixed serialized protobuf messages supporting up to 4096 bytes per datagram
4. **Drain Sequence**: Parent drains listeners gracefully while tracking `parent_drained_` status
5. **Child Readiness**: Validates parent connection on startup and registers UDP listeners for QUIC packet forwarding
6. **Stats Merging**: Transfers counter deltas, gauge values, and dynamic stat information to child process

The mechanism enables zero-connection-loss restarts through elegant parent-child handshaking and seamless socket/state transfer.

# Prometheus TSDB Compaction Operational Runbook

## 1. Overview

### What Compaction Does

Compaction in Prometheus TSDB merges multiple time-series data blocks into larger, more efficient blocks. This process:

- **Merges overlapping blocks**: Combines blocks with overlapping time ranges into a single block to improve query performance and consistency
- **Removes deleted series**: Eliminates tombstoned (deleted) series from blocks, reducing disk space usage
- **Deduplicates samples**: Consolidates duplicate samples from overlapping blocks
- **Optimizes index and chunk layout**: Reorganizes data for better I/O efficiency and memory locality

### Compaction Levels and Block Structure

Prometheus uses **exponential compaction levels**:

- **Level 1**: Freshly persisted head blocks (2-hour default duration)
- **Level 2**: First-level compaction (4-hour range)
- **Level 3**: Second-level compaction (8-hour range)
- **Level N**: Progressive doubling of time ranges

Each level is defined by:
- **MinTime (mint)**: Start timestamp (milliseconds, inclusive)
- **MaxTime (maxt)**: End timestamp (milliseconds, exclusive)
- **ULID**: Unique identifier for the block
- **Compaction.Level**: Current compaction level
- **Compaction.Parents**: Source blocks that were compacted
- **Compaction.Sources**: Original blocks the compact traces back to

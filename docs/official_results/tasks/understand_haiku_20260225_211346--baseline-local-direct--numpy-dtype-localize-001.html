# NumPy Dtype Compatibility Issue: Pandas Nullable Integers → Scikit-Learn Preprocessing

## Summary

When a pandas DataFrame with nullable integer types (e.g., `pd.Int64Dtype`) is passed to scikit-learn's preprocessing functions like `StandardScaler` or `MinMaxScaler`, a dtype compatibility issue occurs. The root cause lies in how nullable integer arrays are converted to NumPy arrays when missing values are present, combined with sklearn's input validation expecting specific float dtypes.

## Issue Root Cause

**Primary Issue**: Implicit dtype conversion of nullable integers to object arrays instead of expected float arrays, causing downstream NumPy operations to fail or produce unexpected results.

# Flipt Release Notes

## Overview

These release notes document the API and configuration changes in Flipt since v1.40.1, including new features, improvements, and important information for upgrading.

## Breaking Changes

No breaking changes identified in this release. All API changes are backward compatible.

## New Features

### Configuration: Remote Blob Storage Support

**Added in:** Unreleased (after v1.40.1)

Flipt now supports loading configuration files from remote blob storage backends. This enables storing your Flipt configuration in cloud object storage (S3, Azure Blob Storage, Google Cloud Storage) instead of local filesystem.

#### Supported Schemes
- **S3:** `s3://bucket-name/path/to/config.yml`
- **Azure Blob Storage:** `azblob://container/path/to/config.yml`
- **Google Cloud Storage:** `gs://bucket-name/path/to/config.yml`

#### Configuration Example
```bash
# Load configuration from S3
FLIPT_CONFIG_PATH=s3://my-bucket/flipt-config.yml

# Load configuration from Azure Blob Storage
FLIPT_CONFIG_PATH=azblob://mycontainer/flipt-config.yml

# Load configuration from Google Cloud Storage
FLIPT_CONFIG_PATH=gs://my-bucket/flipt-config.yml
```

#### API Details
- Configuration is loaded via the `getConfigFile()` function in `internal/config/config.go`
- The function supports URLs with schemes mapped to object storage backends
- Falls back to local filesystem if no recognized remote scheme is detected
- Requires appropriate cloud provider credentials/authentication configured in your environment

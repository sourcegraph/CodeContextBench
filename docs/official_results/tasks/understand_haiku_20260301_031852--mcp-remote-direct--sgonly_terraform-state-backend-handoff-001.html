CLI lock wrapper with output\n\n### State Storage Failures\n\n**Network Connectivity**:\n- S3 bucket unreachable\n- Database server down\n- HTTP endpoint timeout\n- **Handling**: Returned as errors from `RefreshState()` or `PersistState()`\n- **Impact**: Operations fail early, state not modified\n\n**Concurrent Modifications**:\n**Mechanism** (S3 example):\n```go\n// S3 detects concurrent writes via:\n// 1. ETag comparison (MD5 of object)\n// 2. Serial number in state file\n// 3. Lineage matching\n```\n**File**: `internal/states/statefile/` - State version tracking\n**Behavior**: `PersistState()` detects mismatch, returns error\n**CLI Behavior**: User must resolve conflict (merge/retry)\n\n**Permissions Issues**:\n- Write denied to state file\n- DynamoDB table unreadable\n- KMS key unavailable (for encrypted state)\n- **Impact**: Operations fail, user needs to fix permissions\n\n### State Corruption\n\n**Corrupted State File**:\n- Invalid JSON\n- Missing required fields\n- **Detection**: Happens during `RefreshState()` unmarshaling\n- **Prevention**: Backup files created before writes\n- **Recovery**: Use `.backup` file or restore from version control\n\n**Inconsistent Lineage**:\n- State has different lineage than expected\n- Indicates state was modified outside Terraform\n- **Detection**: Lineage mismatch during refresh\n- **User Action**: Investigate, possibly use `-refresh=false` to preserve state\n\n### Workspace Errors\n\n**Default Workspace Missing**:\n```\nError: Cannot find default state\nThe \"default\" workspace must exist and is required.\n```\n**Code**: `internal/backend/backend.go:28-29` -

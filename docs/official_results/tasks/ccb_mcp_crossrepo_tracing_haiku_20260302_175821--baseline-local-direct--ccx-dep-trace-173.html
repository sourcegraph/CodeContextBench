## Summary

The analysis identifies the complete event handler chain from API server watch events to handler notifications:

### **Key Components:**

1. **ListWatch** (`listwatch.go`) - Establishes watch connections to the API server
2. **Reflector** (`reflector.go`) - Continuously pumps watch events into the queue
3. **DeltaFIFO** (`delta_fifo.go`) - Producer-consumer queue that accumulates object changes as Deltas
4. **Controller** (`controller.go`) - Orchestrates reflector and processes queued deltas
5. **sharedIndexInformer** (`shared_informer.go`) - Main informer implementation that:
   - Updates the cache store with received changes
   - Dispatches notifications via the processor
6. **sharedProcessor** (`shared_informer.go`) - Distributes notifications to multiple registered listeners
7. **processorListener** (`shared_informer.go`) - Per-handler buffering and delivery mechanism
8. **Store/ThreadSafeStore** (`store.go`, `thread_safe_store.go`) - Thread-safe object cache

### **Event Flow Path:**
```
API Server → ListWatch.Watch() → Reflector → DeltaFIFO → Controller.Run() → 
processDeltas() → Store Update + Handler Call → sharedProcessor.distribute() → 
processorListener.pop() → processorListener.run() → ResourceEventHandler Callbacks
```

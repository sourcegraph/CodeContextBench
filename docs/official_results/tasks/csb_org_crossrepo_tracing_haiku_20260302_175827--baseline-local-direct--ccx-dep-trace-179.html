## Summary

The Grafana unified alerting system implements a complete pipeline with these key components:

**1. Rule Scheduler** (`schedule/schedule.go`)
- The `schedule` struct orchestrates periodic evaluation of alert rules
- `Run()` method executes the main event loop, with `processTick()` determining which rules to evaluate each interval

**2. Alert Rule Evaluation** (`schedule/alert_rule.go`)
- Individual `alertRule` objects represent scheduled rules
- `evaluate()` builds an evaluator, executes the rule's condition queries, and processes results through the state manager

**3. Evaluation Engine** (`eval/eval.go`)
- `conditionEvaluator` executes rule conditions through the expression service
- `EvaluateAlert()` converts query responses into alert states

**4. State Management** (`state/manager.go`, `state/state.go`)
- `Manager.ProcessEvalResults()` processes evaluation results and manages state transitions
- The `transition()` method applies state logic (alerting→normal, error→alerting, etc.)
- `State` tracks individual alert instances with labels, annotations, and state history

**5. Alert Routing** (`sender/router.go`)
- `AlertsRouter.Send()` routes alerts to internal or external alertmanagers based on configuration

**6. Internal Alertmanager** (`notifier/multiorg_alertmanager.go`, `notifier/alertmanager.go`)
- `MultiOrgAlertmanager` manages per-organization alertmanagers
- `PutAlerts()` accepts evaluated alerts and routes them to configured receivers

**7. Notification Receivers** (`notifier/receiver_svc.go`)
- `ReceiverService` manages notification endpoints (email, webhook, etc.)

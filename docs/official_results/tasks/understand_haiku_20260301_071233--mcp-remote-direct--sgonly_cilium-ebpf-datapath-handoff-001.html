Failure Modes\n\n### eBPF Program Loading Failures\n\n#### Verifier Failures\n\n**Problem**: The eBPF verifier rejects program loads if they violate kernel safety rules.\n\n**Common Causes**:\n- **Unbounded loops**: eBPF can't have unbounded loops (kernel prevents DoS)\n- **Out-of-bounds memory access**: Accessing memory beyond packet or stack bounds\n- **Invalid register usage**: Using uninitialized registers\n- **Complexity limit**: Program exceeds instruction limit (~100k instructions per kernel version)\n\n**Detection**:\n```go\n// pkg/datapath/loader/netlink.go:loadDatapath()\n// Returns error with verifier log:\n// \"invalid memory access for read from context addr=offset+size\"\n```\n\n**Recovery**:\n1. Check kernel logs: `dmesg | grep -i \"ebpf\\|bpf\"`\n2. Enable verifier logging (rebuilds with debug flags)\n3. Simplify program logic or split into tail calls\n\n#### Map Incompatibility Failures\n\n**Problem**: Maps can't be loaded if their structure (key/value size, type, max entries) differs from pinned versions.\n\n**Common Causes**:\n- Changing endpoint ID size (uint8 \u2192 uint16)\n- Changing key/value structures in policy maps\n- Max entries mismatch between runs\n\n**Detection** (in `pkg/bpf/collection.go:256-260`):\n```go\n// MapSpecs that differ (type/key/value/max/flags) from their pinned versions\n// will result in an ebpf.ErrMapIncompatible\n```\n\n**Recovery**:\n```bash\n# Unpin incompatible maps\nsudo rm /sys/fs/bpf/cilium/tc/globals/cilium_policy_*\n# Restart agent\nsystemctl restart cilium\n```\n\n#### Permission Failures\n\n**Problem**: eBPF requires CAP_BPF and CAP_PERFMON (or CAP_SYS_ADMIN on older kernels).\n\n**Common Causes**:\n- Running Cilium in unprivileged container\n- SELinux policies blocking eBPF syscalls\n- AppArmor restrictions\n\n**Detection**:\n```\nerror: \"failed loading eBPF collection: permission denied\"\n```\n\n**Recovery**:\n1. Run with `--cap-add=CAP_BPF --cap-add=CAP_PERFMON`\n2. Or grant `--cap-add=CAP_SYS_ADMIN` (broader)\n3. Disable SELinux or update policies\n\n### eBPF Map Failures\n\n#### Map Full Failures\n\n**Problem**: Connection tracking or policy maps reach max_entries limit.\n\n**Impact**:\n- New connections drop silently\n- Packets matching new identities are dropped\n- Service load balancing fails for new backends\n\n**Detection**:\n```bash\n# Check map pressure\ncilium-dbg bpf map list\n# Look for pressure > 0.1 (threshold in policymap.go)\n```\n\n**Monitoring** (metrics):\n```go\n// pkg/metrics/metrics.go:\nBpfMapPressure // Map utilization as percentage\nBpfMapSize     // Current entries vs max entries\n```\n\n**Recovery**:\n1. Increase `--bpf-map-dynamic-size-ratio` (for dynamic sizing)\n2. Increase max entries in map definitions\n3. Restart agent to recreate maps with new size\n\n#### Map Update Failures\n\n**Problem**: Updating policy or NAT entries in kernel maps fails.\n\n**Common Causes**:\n- Map locked by multiple processes\n- Memory exhaustion in kernel\n- Incompatible data format\n\n**Detection**:\n```go\n// pkg/endpoint/bpf.go:\n// \"cilium_call_policy: delete: key does not exist\"\n```\n\n**Impact**:\n- Policies don't apply to traffic\n- Old policies persist even after update\n- Service routing becomes inconsistent\n\n### Kernel Compatibility Issues\n\n#### Feature Detection Failures\n\nCilium probes for kernel features at startup (`pkg/datapath/linux/probes/probes.go`):\n\n```go\ntype SystemConfig struct {\n    ConfigBpf          KernelParam  // CONFIG_BPF\n    ConfigBpfSyscall   KernelParam  // CONFIG_BPF_SYSCALL\n    ConfigCgroupBpf    KernelParam  // CONFIG_CGROUP_BPF\n    // ... 30+ kernel config checks\n}\n```\n\n**If critical features missing:**\n```\nerror: \"BPF requires CONFIG_BPF_SYSCALL (see docs/)\"\n```\n\n**Recovery**:\n- Check `/boot/config-$(uname -r)` for missing CONFIGs\n- Recompile kernel with required options\n- Or use system with adequate kernel support\n\n#### Helper Function Unavailable\n\n**Problem**: eBPF program calls a kernel helper not available in this kernel version.\n\n**Example**:\n```\nerror: \"XDP program requires bpf_xdp_load_bytes (kernel 5.8+)\"\n```\n\n**Detection** (in `pkg/datapath/linux/probes/probes.go`):\n```go\nProgramHelpers map[ProgramHelper]bool  // Each helper is probed\n// XDP: { bpf_xdp_load_bytes, bpf_xdp_store_bytes, bpf_xdp_get_buff_len }\n// TC: { bpf_skb_load_bytes, bpf_skb_store_bytes, ... }\n```\n\n**Recovery**:\n1. Disable features requiring newer helpers: `--bpf-skip-feature=xyz`\n2. Upgrade kernel\n\n### Configuration Errors\n\n#### Invalid Node Configuration\n\n**Problem**: Node configuration header generation fails.\n\n**Detected in**: `pkg/datapath/loader/template.go`\n\n**Common Causes**:\n- Invalid CIDR ranges in node configuration\n- Conflicting endpoint ID ranges\n- Invalid network device specification\n\n**Detection**:\n```\nerror: \"invalid node configuration: invalid CIDR\"\n```\n\n#### Endpoint-Specific Failures\n\n**Problem**: Specific endpoint policy can't be loaded.\n\n**Examples**:\n- Port range overflow\n- Policy identity conflict\n- Invalid protocol specifications\n\n**Detection** (in `pkg/endpoint/bpf.go:Regenerate()`):\n```\nerror: \"failed to generate endpoint: invalid policy entry\"\n```\n\n**Recovery**:\n1. Check endpoint logs: `kubectl logs -c agent <pod> | grep endpoint-id`\n2. Verify endpoint configuration: `cilium endpoint get <id>`\n3. Look for conflicting policies\n\n### Recovery and Mitigation\n\n#### Automatic Recovery Mechanisms\n\n1. **Template Cache** (`pkg/datapath/loader/cache.go`):\n   - Pre-compiles datapath templates\n   - Falls back to cached version on compilation failure\n   - Prevents complete datapath loss during compilation errors\n\n2. **Map Pinning** (`pkg/bpf/`:\n   - Maps persist at `/sys/fs/bpf/cilium/` across restarts\n   - Avoids losing connection tracking state\n   - Can be cleared with `cilium bpf reset` if needed\n\n3. **Graceful Degradation**:\n   - If XDP fails, falls back to TC\n   - If TCX fails, falls back to legacy TC\n   - If policy loading fails, endpoint isolation applies\n\n#### Manual Recovery\n\n```bash\n# Full BPF reset (dangerous - drops all state)\ncilium bpf reset\n\n# Reload datapath for specific interface\ncilium-dbg reload-datapath\n\n# Check loader status\ncilium-dbg endpoint status | grep datapath\n\n# View verifier logs (if debug enabled)\ncat /sys/kernel/debug/tracing/trace_pipe | grep BPF\n```\n\n---\n\n## 5.

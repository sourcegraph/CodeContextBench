# VS Code Diagnostic API Reference

## Overview

The Diagnostic API provides a way for VS Code extensions to report code issues (errors, warnings, information, hints) to users within the editor. The API allows extensions to:

- Create diagnostic collections to organize diagnostics by source (e.g., 'typescript', 'eslint')
- Report diagnostics for specific files or documents
- Track and manage diagnostics across the workspace
- Subscribe to diagnostic changes globally
- Provide detailed diagnostic information with related locations and additional metadata

Diagnostics are displayed in the editor gutter, problems panel, and inline as squiggles. They can be associated with quick fixes through the Code Actions API.

# Bug Investigation: HSV Color Percentage Parsing Produces Wrong Hue Values

**Repository:** qutebrowser/qutebrowser
**Task Type:** Find and Prove (write a regression test)

## Reported Issue

When users specify colors in the qutebrowser configuration using HSV (Hue-Saturation-Value) notation with percentage values, the hue component is calculated incorrectly.

The hue channel in HSV color space has a valid range of 0-359 degrees. However, the color parser incorrectly scales hue percentages using a maximum of 255 instead of 359. This means:

- `hsv(100%, 100%, 100%)` is parsed as `(255, 255, 255)` instead of the correct `(359, 255, 255)`
- `hsv(50%, 50%, 50%)` produces hue 127 instead of the correct hue 179

The saturation and value channels (which correctly use 0-255 range) are not affected — only the hue channel has this scaling error.

The same bug affects HSVA notation (HSV with alpha channel). For example, `hsva(100%, 100%, 100%, 100%)` also produces the wrong hue value.

## Investigation Results

**Root Cause Found:** In `/app/qutebrowser/config/configtypes.py` (lines 1004-1018), the `_parse_value()` method uses a fixed multiplier of `255.0/100` for all percentage values:

```python
def _parse_value(self, val: str) -> int:
    mult = 255.0
    if val.endswith('%'):
        val = val[:-1]
        mult = 255.0 / 100  # BUG: Wrong for HSV hue!
    
    return int(float(val) * mult)
```

**The Bug:** This multiplier is correct for RGB (0-255 range) and for HSV saturation/value (0-255 range), but **incorrect for HSV hue** which has a range of 0-359 degrees.

**Impact:**
- `hsv(100%, 100%, 100%)` produces hue=254 instead of 359
- `hsv(50%, 50%, 50%)` produces hue=127 instead of 179
- `hsv(10%, 10%, 10%)` produces hue=25 instead of 35
- Same bug affects HSVA notation

## Regression Test Created

**File:** `/workspace/regression_test.py`

The test contains **15 comprehensive test cases** organized into two test classes:

### TestHSVPercentageBug (11 tests)
- Tests HSV and HSVA percentage parsing with various values
- Verifies the buggy hue scaling behavior
- Tests that integer values work correctly
- Confirms RGB percentages are unaffected
- Cross-references existing tests from the codebase

### TestHSVBugMathematicalProof (4 tests)
- Provides mathematical proof of the bug
- Shows conversion tables comparing buggy vs. correct values
- Verifies saturation and value channels work correctly

**All 15 tests pass** and demonstrate the bug exists on the current (buggy) code.

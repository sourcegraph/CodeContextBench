# Envoy HTTP Connection Manager Architecture

## Overview

The HTTP Connection Manager (HCM) is Envoy's core subsystem for processing HTTP requests from downstream clients through to upstream services. This document explains how the major components interact to handle HTTP request/response lifecycle: from receiving raw bytes on a downstream connection, through per-stream filtering and routing, to dispatching upstream requests and returning responses downstream.

The architecture consists of four tightly-integrated layers:

1. **ConnectionManagerImpl** — Network I/O and connection lifecycle management
2. **FilterManager** — Per-stream filter chain execution engine
3. **Router Filter** — Terminal filter that performs route selection and upstream dispatch
4. **Cluster Manager** — Upstream connection pool allocation and host selection

---

## 1. ConnectionManagerImpl: Network Read Filter and Connection Lifecycle

**Source files:**
- `source/common/http/conn_manager_impl.h` (lines 60-220)
- `source/common/http/conn_manager_impl.cc` (lines 118-300)

### Responsibilities

ConnectionManagerImpl is a `Network::ReadFilter` that owns the HTTP connection lifecycle. It:

- **Receives raw downstream bytes** via the `Network::ReadFilter` interface's `onData()` method
- **Instantiates and manages the HTTP codec** (H1/H2/H3) lazily based on the first bytes received
- **Creates and tracks `ActiveStream` objects** — one per HTTP request/response pair
- **Manages connection-level concerns**:
  - Drain decisions (graceful shutdown)
  - Overload management (rejecting new streams under load)
  - Write buffer watermarks (pausing/resuming downstream read when buffers exceed limits)
  - Connection lifetime limits (max requests, timeouts)
  - Tracing and access logging coordination

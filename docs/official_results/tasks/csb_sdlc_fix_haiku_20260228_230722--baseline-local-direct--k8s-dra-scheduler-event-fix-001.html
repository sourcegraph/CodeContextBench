# big-code-k8s-bug-001: Missing ResourceSlice Event Handler in Kubernetes Scheduler

## Task

Investigate a bug in the Kubernetes scheduler where pods requesting Dynamic Resource Allocation (DRA) devices become permanently stuck in an Unschedulable state when the DRA driver starts after the pod is created. Trace the execution path through the scheduler's event-driven re-queuing architecture to identify why ResourceSlice creation/update events are silently dropped.

## Context

- **Repository**: kubernetes/kubernetes (Go, ~3.5M LOC)
- **Category**: Bug Investigation
- **Difficulty**: hard
- **Entry Point**: `pkg/scheduler/schedule_one.go` — `ScheduleOne()` method

## Symptom

When a pod requests a DRA device and the DRA driver starts *after* the pod is created, the pod becomes permanently stuck in `Pending` with reason `Unschedulable`. The scheduler logs `"cannot allocate all claims"` because no ResourceSlices exist at scheduling time. After the driver starts and publishes ResourceSlice objects, the pod should be rescheduled — but it never is.

The bug is a race condition: if the driver starts first, scheduling succeeds immediately. But if the pod arrives before the driver, the scheduler never retries because ResourceSlice events are not wired into the scheduling queue's re-evaluation mechanism.

## Investigation Summary

I've traced the bug from symptom to root cause and documented the complete analysis in `/logs/agent/solution.md`. Here's what I found:

### **The Bug**
Pods requesting Dynamic Resource Allocation (DRA) devices become permanently stuck in `Unschedulable` state when the DRA driver (which creates ResourceSlice objects) starts AFTER the pod is created.

### **Root Cause** 
ResourceSlice events are **never registered** in the scheduler's event handling pipeline. This happens because:

1. **Missing GVK Definition** in `pkg/scheduler/framework/types.go` (lines 92-96)
   - ResourceSlice is not defined as a GVK constant
   - Other DRA resources (ResourceClaim, ResourceClass, ResourceClaimParameters, etc.) ARE defined, but ResourceSlice is missing

2. **Missing Event Registration** in `pkg/scheduler/framework/plugins/dynamicresources/dynamicresources.go` (lines 381-413)
   - The `EventsToRegister()` method declares handlers for ResourceClaim, PodSchedulingContext, ResourceClass, etc.
   - But ResourceSlice is NOT in this list

### **Why This Causes the Bug**
The scheduler uses a plugin-driven event system:
1. Plugins declare which events should trigger pod re-evaluation via `EventsToRegister()`
2. The scheduler aggregates these into a `gvkMap`
3. Only resources in `gvkMap` get event handlers registered
4. When ResourceSlice is missing from both, ResourceSlice creation/update events are silently dropped
5. Pods in the unschedulable queue are never re-queued, even though resources are now available

**Impact**: Critical - pods with DRA claims can become permanently unschedulable if timing is unfavorable

**Fix Complexity**: Minimal - requires adding ResourceSlice as a GVK constant and registering it in EventsToRegister()

**Total Changes**: 2 files, 3 lines of code

   - Core files: `schedule/schedule.go`, `alert_rule.go`, `registry.go`, `fetcher.go`
   - Periodically fetches rules from the store and evaluates them
   - Maintains active rule evaluation routines in a registry
   - Data flows: rules → evaluations → state manager

3. **State Manager**
   - Core files: `state/manager.go`, `state.go`, `cache.go`, `persist.go`
   - Tracks alert instance states in memory and persists to database

   - Core files: `state/historian/core.go`, `backend.go`, `annotation.go`, `loki.go`, `prometheus.go`, `multiple.go`

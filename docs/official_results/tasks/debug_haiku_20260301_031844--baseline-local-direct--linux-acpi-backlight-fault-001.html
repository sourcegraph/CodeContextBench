## Bug Report

**Bug ID**: 207835
**Summary**: ACPI video backlight brightness control does not work on Acer TravelMate 5735Z by default

On the Intel GM45/GMA 4500MHD laptop with Debian Sid/unstable and Linux 5.6.7 and GNOME Shell 3.36.2, trying to change the brightness of the internal display using the function keys, GNOME shows the OSD, which seems to have five levels (from acer_wmi?), but the actual brightness does *not* change. There is only `/sys/class/backlight/intel_backlight/brightness` though, and writing values to it seems to work.

Booting the system with `acpi_backlight=vendor`, exposes `/sys/devices/platform/acer-wmi/backlight/acer-wmi/brightness`, but the behavior is the same.

Booting the system with `acpi_backlight=native` or `acpi_backlight=native`, changing the brightness with the function keys works, and there now seem to be 15 (or 16) levels.

## Analysis Summary

### The Bug
On the Acer TravelMate 5735Z laptop with Intel GM45/GMA 4500MHD graphics, the ACPI video backlight brightness control doesn't work by default. The system only works correctly with the `acpi_backlight=native` kernel parameter.

### Root Cause
The problem is in **`drivers/acpi/video_detect.c`** in the `video_detect_dmi_table` data structure. The Acer TravelMate 5735Z is missing from this DMI quirks table.

### How the Bug Manifests
1. **Default behavior**: The kernel's `acpi_video_get_backlight_type()` function determines which backlight driver to use:
   - Checks for command-line overrides
   - Checks DMI quirks table (this is where the bug is)
   - Checks ACPI capabilities and Windows 8 compatibility
   - Falls back to `acpi_backlight_video`

2. **For the TravelMate 5735Z**, since there's no DMI entry:
   - The system has ACPI video backlight capability (_BCL and _BCM methods)
   - The system is not Windows 8-ready or lacks a BACKLIGHT_RAW device
   - It defaults to `acpi_backlight_video`
   - BUT the ACPI video backlight methods don't actually control the hardware

3. **The native Intel GMA backlight driver IS what works**, but it's not being used by default.

### The Fix
The Acer TravelMate 5735Z should be added to the `video_detect_dmi_table` with a callback to `video_detect_force_native` to force the system to use the native Intel backlight driver instead of the broken ACPI video backlight methods.

# Kubernetes API Priority and Fairness (APF) QueueSet Deep-Dive

## Subsystem Overview and Purpose

The API Priority and Fairness (APF) QueueSet subsystem is the core request dispatching mechanism that implements fair queuing for server requests in Kubernetes apiserver. It prevents resource starvation by ensuring that workloads at lower priority levels cannot block higher priority workloads from making progress, and that different flow schemas within the same priority level receive fair access to execution capacity.

### Subsystem Purpose

The QueueSet subsystem solves a fundamental problem in multi-tenant API servers: how to allocate finite execution concurrency (CPU, memory, I/O) fairly across competing request flows while respecting priority levels. Without it, a single noisy tenant making expensive list requests could starve critical system workloads like node heartbeats or cluster autoscaler operations.

The subsystem addresses this by:
- **Queuing incoming requests** when the system is at capacity
- **Fair dispatching** using a modified Weighted Fair Queuing (WFQ) algorithm adapted for server requests
- **Virtual time progression** based on actual work done (seat-seconds), not just wall-clock time
- **Concurrency allocation** to ensure priority levels cannot steal execution capacity from each other
- **Shuffle sharding** to prevent thundering herd problems within a priority level

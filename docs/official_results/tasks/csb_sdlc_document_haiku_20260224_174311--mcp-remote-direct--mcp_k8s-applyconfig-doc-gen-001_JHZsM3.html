## 1. Subsystem Purpose and Scope

The **Server-Side Apply (SSA)** and **Apply Configurations** subsystem represents Kubernetes' declarative, intent-based patch mechanism. It provides:

1. **Field Ownership Tracking**: Server maintains `managedFields` metadata to track which "manager" (client/controller identity) owns each field
2. **Conflict Detection**: Automatic detection when multiple managers attempt to modify the same field
3. **Type-Safe Apply Configuration API**: Generated Go types that represent Kubernetes objects with all fields optional, enabling safe declarative management
4. **Structured Merge Semantics**: Deterministic three-way merge behavior based on structured field path tracking

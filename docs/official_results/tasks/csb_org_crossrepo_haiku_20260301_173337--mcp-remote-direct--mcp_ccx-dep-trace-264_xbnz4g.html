## Key Findings

The SpiderMonkey compilation pipeline consists of these main stages:

1. **Compilation Entry Point** (`BytecodeCompiler.cpp`):
   - `CompileGlobalScriptToStencilWithInput()` - Main entry point
   - `ScriptCompiler::compile()` - Orchestrates parsing and bytecode emission

2. **Parser** (`Parser.cpp`):
   - `GeneralParser::parse()` - Top-level script parsing entry point
   - `Parser::globalBody()` - Parses global script body
   - Generates an Abstract Syntax Tree (AST) of ParseNode objects

3. **AST Node Types** (`ParseNode.h`):
   - `FOR_EACH_PARSE_NODE_KIND` macro defines all 100+ node types
   - Includes statements, expressions, literals, declarations
   - All nodes allocated in LifoAlloc for efficient management

4. **Bytecode Emitter** (`BytecodeEmitter.cpp`):
   - `BytecodeEmitter::emitScript()` - Traverses AST and generates bytecode
   - Emits instructions for declarations, expressions, control flow, etc.
   - Stores metadata including source notes and GC references

5. **Bytecode Output** (`JSScript.h`):
   - `JSScript` class is the runtime representation
   - Contains bytecode instructions, atoms, GC things, scope info
   - Used by interpreter and JIT compiler

6. **Bytecode Instructions** (`Opcodes.h`):
   - Defines all ~150 JSOp instructions with operand layouts
   - Instructions like Add, Call, GetLocal, IfEq, etc.

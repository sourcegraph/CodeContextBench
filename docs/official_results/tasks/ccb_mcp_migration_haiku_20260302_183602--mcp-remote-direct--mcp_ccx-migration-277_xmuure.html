## Key Findings:

**Core Components:**
1. **Provider** (`java/security/Provider.java`) - Base abstract class with `putService()` method for registering cryptographic services
2. **ProviderConfig** (`sun/security/jca/ProviderConfig.java`) - Lazy-loading mechanism that maps provider names to classes (SUN, SunJCE, SunJSSE, SunEC, etc.)
3. **Sun** (`sun/security/provider/Sun.java`) - Core provider for basic algorithms (DSA, SHA, MD5, X.509, PKIX, etc.)
4. **SunEntries** (`sun/security/provider/SunEntries.java`) - Defines all services provided by the Sun provider
5. **SunJCE** (`com/sun/crypto/provider/SunJCE.java`) - Cryptography provider (RSA, DES, AES, Diffie-Hellman, HMAC, etc.)
6. **SunJSSE** (`sun/security/ssl/SunJSSE.java`) - SSL/TLS provider with SSLContext factories and key/trust managers
7. **SunEC** (`sun/security/ec/SunEC.java`) - Elliptic curve provider (ECDSA, EdDSA, ECDH)

**How it works:**
- ProviderConfig implements lazy-loading via the `getProvider()` method, which instantiates providers only when needed
- Each provider registers services via `putService()` calls, which store Service objects in a serviceMap
- Services are looked up by type and algorithm name with support for aliases
- The framework allows both programmatic registration and legacy property-based registration

- HTTP translation via generated grpc-gateway files

**kubernetes/kubernetes** uses distributed error handling:
- No centralized error mapping
- Rate limiting interceptor creates `status.Error(codes.ResourceExhausted)`
- Health probe checks error codes at inspection points (Unimplemented, DeadlineExceeded)
- CRI error utilities use `status.FromError()` to check codes
- CSI client handles gRPC status errors for storage operations

**grafana/grafana** uses interceptor-based and plugin-focused error handling:
- Authentication interceptor creates `status.Error(codes.Unauthenticated)`
- Request status tracking checks for `codes.Canceled` and `codes.DeadlineExceeded`
- Authorization service uses `status.Errorf(codes.InvalidArgument)`
- Plugin architecture integrates gRPC error handling
- Resource layer maps errors to HTTP status codes

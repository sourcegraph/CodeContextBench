## Bug Report

**Bug ID**: 207835
**Summary**: ACPI video backlight brightness control does not work on Acer TravelMate 5735Z by default

On the Intel GM45/GMA 4500MHD laptop with Debian Sid/unstable and Linux 5.6.7 and GNOME Shell 3.36.2, trying to change the brightness of the internal display using the function keys, GNOME shows the OSD, which seems to have five levels (from acer_wmi?), but the actual brightness does *not* change. There is only `/sys/class/backlight/intel_backlight/brightness` though, and writing values to it seems to work.

Booting the system with `acpi_backlight=vendor`, exposes `/sys/devices/platform/acer-wmi/backlight/acer-wmi/brightness`, but the behavior is the same.

Booting the system with `acpi_backlight=native` or `acpi_backlight=native`, changing the brightness with the function keys works, and there now seem to be 15 (or 16) levels.

## Summary of Fault Localization

**Buggy File**: `drivers/acpi/video_detect.c`

**Buggy Data Structure**: `video_detect_dmi_table` (lines 112-377)

**Root Cause Analysis**:

The Acer TravelMate 5735Z experiences ACPI video backlight brightness control failure due to incorrect backlight interface selection:

1. **Detection Logic Flow**:
   - `acpi_video_get_backlight_type()` determines which backlight driver to use
   - For the Acer system, the ACPI video device lacks the required `_DOD` or `_DOS` methods
   - This prevents `acpi_is_video_device()` from detecting `_BCL`/`_BCM` methods
   - Without `ACPI_VIDEO_BACKLIGHT` capability, the system defaults to vendor backlight

2. **The Problem**:
   - The acer-wmi vendor backlight interface doesn't respond to function key events
   - The system should instead use the native backlight (through ACPI's native interface)
   - Native backlight works fine when explicitly set via `acpi_backlight=native`

3. **The Fix**:
   - Add a DMI quirk entry for the Acer TravelMate 5735Z to the `video_detect_dmi_table`
   - Match on `DMI_SYS_VENDOR="Acer"` and `DMI_PRODUCT_NAME="TravelMate 5735Z"`
   - Use the `video_detect_force_native` callback to force native backlight
   - This allows the system to skip the vendor backlight default and use native backlight instead

The fault localization result has been written to `/workspace/fault_localization_result.json`.

# CompositFieldValidator Usage Examples

The `CompositFieldValidator` is a form validator that allows you to validate relationships between multiple form fields simultaneously. This is useful for cross-field validation where you need to ensure that values across multiple fields meet certain conditions.

## Basic Usage

```python
from django import forms
from django.forms.validators import CompositFieldValidator

class DateRangeForm(forms.Form):
    start_date = forms.DateField()
    end_date = forms.DateField()

    # Validator to ensure end_date is after start_date
    date_range_check = CompositFieldValidator(
        validation_func=lambda start, end: start <= end,
        field_names=['start_date', 'end_date'],
        message="End date must be after or equal to start date."
    )
```

## How It Works

1. **Create a validator instance** as a class attribute on your form
2. **Specify the validation function** - a callable that receives the field values as arguments
3. **List the field names** to validate together (in order)
4. **Provide an error message** that will be displayed if validation fails

## Validation Function

The `validation_func` parameter should be a callable that:
- Receives values for each field in the order specified in `field_names`
- Returns `True` if validation passes, `False` if it fails
- Can raise an exception if needed (will be caught and treated as a validation error)

## Notification Pipeline Overview

The alertmanager implements a sophisticated pipeline that processes firing alerts from ingestion to final delivery:

### **1. Alert Ingestion** (`api/v2/api.go`)
- The `postAlertsHandler()` receives alerts via HTTP POST
- Validates and stores them in memory via `provider/mem/mem.go`

### **2. Routing Tree Traversal** (`dispatch/route.go`, `dispatch/dispatch.go`)
- `Dispatcher.Run()` monitors incoming alerts
- `routeAlert()` invokes `route.Match()` which traverses the routing tree depth-first
- Returns all matching routes based on label matchers

### **3. Alert Grouping** (`dispatch/dispatch.go`)
- `groupAlert()` creates aggregation groups per matched route
- Groups alerts for batch notification delivery

### **4. Inhibition Check** (`inhibit/inhibit.go`)
- `Inhibitor.Mutes()` evaluates inhibition rules
- Prevents notifications for alerts suppressed by higher-level alerts
- Applied via `MuteStage` in the notification pipeline

### **5. Silence Check** (`silence/silence.go`)
- `Silencer.Mutes()` queries active silences
- Filters alerts that match silence matchers
- Also applied via `MuteStage` in the pipeline

### **6. Notification Pipeline & Dispatch** (`notify/notify.go`, `notify/mute.go`)
- `PipelineBuilder.New()` constructs a multi-stage pipeline
- Includes gossip settling, muting stages, time interval checks
- `Integration.Notify()` sends notifications to receivers (email, Slack, etc.)

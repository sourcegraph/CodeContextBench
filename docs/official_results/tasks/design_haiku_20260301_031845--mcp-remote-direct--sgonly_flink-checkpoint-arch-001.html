### **Dependency Chain** (4 phases traced)
1. **Phase 1 (Trigger):** `CheckpointCoordinator` → `triggerTasks()` → RPC to `TaskExecutor` → `Task.triggerCheckpointBarrier()`
2. **Phase 2 (Barrier Propagation):** Source task injects `CheckpointBarrier` → flows through network channels → downstream `CheckpointedInputGate` intercepts
3. **Phase 3 (Snapshot):** `SingleCheckpointBarrierHandler` coordinates alignment → triggers operator `snapshotState()` → collects state handles
4. **Phase 4 (Acknowledgment):** Task sends `AcknowledgeCheckpoint` via `RpcCheckpointResponder` → `CheckpointCoordinator.receiveAcknowledgeMessage()` → `completePendingCheckpoint()`

### **Key Architectural Insights**
- **Two-phase commit protocol** for atomicity
- **State machine pattern** in `SingleCheckpointBarrierHandler` (aligned vs. unaligned modes)
- **Future-driven async coordination** with non-blocking RPC
- **Acknowledgment-counting protocol** ensuring all-or-nothing completion
- **Aligned checkpoints:** block inputs, wait for all barriers, guarantee exactly-once
- **Unaligned checkpoints:** immediate trigger, buffer in-flight data, lower latency

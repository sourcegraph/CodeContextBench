## Overview

Terraform's plan/apply pipeline is a sophisticated system that transforms infrastructure configurations into a series of ordered operations. The pipeline uses a directed acyclic graph (DAG) representation to model resources, their dependencies, and the operations needed to reach the desired infrastructure state.

### High-Level Architecture

```
User Command (plan/apply)
         ↓
Command Package
         ↓
Backend (local or remote)
         ↓
Context (main orchestrator)
         ↓
Graph Builder (constructs DAG)
         ↓
Graph Walk (evaluates nodes)
         ↓
State Output
```

### Key Architectural Principles

1. **DAG-based Execution**: Resources and other objects are represented as vertices in a directed acyclic graph, with edges representing "must happen after" dependencies.

2. **Lazy Evaluation**: Expressions in configuration are not evaluated until graph walk time, allowing dependencies between resources to be properly resolved.

3. **Concurrent Evaluation**: Graph nodes are evaluated in parallel where possible, respecting dependency constraints.

4. **Immutable State Snapshots**: State is copied and wrapped in thread-safe structures (SyncState) to enable safe concurrent access.

5. **Plugin-Based Providers**: Resource operations are delegated to provider plugins via a well-defined interface.

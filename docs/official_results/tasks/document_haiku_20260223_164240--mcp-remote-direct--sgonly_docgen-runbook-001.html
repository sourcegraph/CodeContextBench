# Prometheus TSDB Compaction Operational Runbook

## 1. Overview

### What is TSDB Compaction?

TSDB (Time Series Database) compaction in Prometheus is the process of merging and optimizing disk-based time series blocks. As Prometheus ingests data, the in-memory "Head" block accumulates samples. When the Head reaches a configured time threshold (default: 2 hours), it is persisted to disk as a new block. Compaction then:

1. **Merges overlapping blocks** into larger, more efficient blocks
2. **Removes tombstoned (deleted) series** from blocks
3. **Reorganizes data** to improve query performance
4. **Manages disk space** by consolidating multiple small blocks into fewer larger ones

### Block Structure and Levels

Prometheus uses a **leveled compaction strategy**:

- **Level 0**: Head blocks persisted from in-memory data (time range: typically 2 hours)
- **Level 1+**: Progressively larger blocks created by merging smaller blocks

Block ranges grow exponentially (e.g., 2h → 10h → 50h → 250h) to maintain balanced query performance and retention efficiency.

### When Compaction Is Triggered

Compaction is initiated in two ways:

#### 1. Time-Based (Periodic, Primary)
- **Trigger**: Every 1 minute, the background compaction loop (`tsdb/db.go:995`) checks if the Head is compactable
- **Condition**: Head block age ≥ half of configured `MinBlockDuration` (default: 1 hour)
- **Code reference**: `tsdb/db.go:1162-1188`

#### 2. Sample-Based (Append-Time, Secondary)
- **Trigger**: When appenders commit samples after Head becomes "compactable"
- **Condition**: Head block time range exceeds threshold
- **Code reference**: `tsdb/db.go:1115-1127` (dbAppender.Commit)

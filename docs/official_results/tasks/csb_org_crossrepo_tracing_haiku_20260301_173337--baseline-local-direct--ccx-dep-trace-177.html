## Key Components:

**1. Factory Interface** (`NamedHttpFilterConfigFactory` in `envoy/server/filter_config.h`)
- Defines the interface all HTTP filter factories must implement
- Key method: `createFilterFactoryFromProto()` returns a `FilterFactoryCb`

**2. Factory Registry** (`FactoryRegistry` in `envoy/registry/registry.h`)
- Global static registry storing factories by name
- Uses `RegisterFactory` template with `REGISTER_FACTORY` macros for registration

**3. Filter Chain Factory** (`FilterChainFactory` in `envoy/http/filter_factory.h`)
- Interface implemented by HTTP connection manager
- `createFilterChain()` method called when new streams arrive

**4. HTTP Connection Manager** (`HttpConnectionManagerConfig` in `source/extensions/filters/network/http_connection_manager/config.h`)
- Implements `FilterChainFactory`
- Creates and stores `FilterFactoryCb` callbacks during initialization

**5. Per-Request Instantiation** (The Critical Flow):
- `FilterChainManager::applyFilterFactoryCb()` executes stored callbacks per-request
- `FilterChainFactoryCallbacksImpl` creates actual filter instances via `addStreamDecoderFilter()`
- Filters are wrapped in `ActiveStreamDecoderFilter`/`ActiveStreamEncoderFilter` and added to stream

**6. Example** (Router filter in `source/extensions/filters/http/router/`)
- Shows the complete pattern: factory creates a lambda callback that instantiates `Router::ProdFilter`
- Registered with `LEGACY_REGISTER_FACTORY` macro

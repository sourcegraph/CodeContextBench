## Review Summary

I identified **2 critical security defects** introduced in the recent changes to curl:

### 1. **Integer Overflow in Base64 Encoding** (CRITICAL)
- **File**: `lib/base64.c`, line 202
- **Issue**: A crucial integer overflow check was removed that prevented heap buffer overflow on 32-bit systems. The removed code was:
  ```c
  #if SIZEOF_SIZE_T == 4
    if(insize > UINT_MAX/4)
      return CURLE_OUT_OF_MEMORY;
  #endif
  ```
- **Impact**: Without this check, an attacker can provide a large input size that causes integer overflow in the allocation calculation `(insize + 2) / 3 * 4 + 1`, resulting in a small heap buffer being allocated. Subsequent writes cause heap overflow and potential code execution.

### 2. **Cookie Domain Matching Bypass** (HIGH)
- **File**: `lib/cookie.c`, line 147
- **Issue**: The pointer arithmetic for subdomain validation was changed from `hostname_len - cookie_domain_len - 1` to `hostname_len - cookie_domain_len`, removing the critical `-1` offset.
- **Impact**: This causes the function to check the wrong character position, allowing cookies set for domain "example.com" to be incorrectly matched against "notexample.com", bypassing RFC 6265 cookie scope validation.

Both defects have been documented in `/workspace/review.json` with detailed descriptions and proposed unified diff patches that restore the removed security checks.

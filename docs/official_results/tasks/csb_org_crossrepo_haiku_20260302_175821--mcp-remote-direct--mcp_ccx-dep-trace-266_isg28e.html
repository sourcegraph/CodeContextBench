## Key Findings

The Firefox event dispatch system consists of several critical components:

1. **Event Creation** (`EventDispatcher::DispatchDOMEvent` & `::Dispatch`): Initiates event processing and validates security context

2. **Event Target Chain Building** (EventDispatcher.cpp lines 1008-1178): Constructs an ancestor chain by repeatedly calling `GetEventTargetParent()` on each target up the DOM tree

3. **Capture/Bubble/Target Phase Traversal** (`EventTargetChainItem::HandleEventTargetChain`):
   - **Capture Phase**: Iterates backward through chain (root to target)
   - **Target Phase**: Processes the original target
   - **Bubble Phase**: Iterates forward through chain (target to root)
   - Each phase can be stopped via `stopPropagation()`

4. **Event Listener Invocation** (`EventListenerManager::HandleEventInternal` & `HandleEventWithListenerArray`):
   - Retrieves registered listeners by event type
   - Filters listeners by phase (capture vs bubble)
   - Invokes each matching listener callback
   - Handles passive listeners and `once` listeners

5. **Default Action Execution** (`PostHandleEvent` methods):
   - Executed after dispatch if not prevented
   - Implemented in specific event target classes (e.g., HTMLButtonElement)

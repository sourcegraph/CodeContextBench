# big-code-k8s-bug-001: Missing ResourceSlice Event Handler in Kubernetes Scheduler

## Task

Investigate a bug in the Kubernetes scheduler where pods requesting Dynamic Resource Allocation (DRA) devices become permanently stuck in an Unschedulable state when the DRA driver starts after the pod is created. Trace the execution path through the scheduler's event-driven re-queuing architecture to identify why ResourceSlice creation/update events are silently dropped.

## Context

- **Repository**: kubernetes/kubernetes (Go, ~3.5M LOC)
- **Category**: Bug Investigation
- **Difficulty**: hard
- **Entry Point**: `pkg/scheduler/schedule_one.go` — `ScheduleOne()` method

## Symptom

When a pod requests a DRA device and the DRA driver starts *after* the pod is created, the pod becomes permanently stuck in `Pending` with reason `Unschedulable`. The scheduler logs `"cannot allocate all claims"` because no ResourceSlices exist at scheduling time. After the driver starts and publishes ResourceSlice objects, the pod should be rescheduled — but it never is.

The bug is a race condition: if the driver starts first, scheduling succeeds immediately. But if the pod arrives before the driver, the scheduler never retries because ResourceSlice events are not wired into the scheduling queue's re-evaluation mechanism.

## Summary

I successfully traced the Kubernetes scheduler bug where pods requesting Dynamic Resource Allocation (DRA) become permanently stuck in an `Unschedulable` state when the DRA driver starts after the pod is created.

### Root Cause Identified

The bug occurs in **`pkg/scheduler/framework/plugins/dynamicresources/dynamicresources.go`** in the `EventsToRegister()` method (line 381). **ResourceSlice events are not registered**, breaking the event-driven pod re-queuing mechanism.

### Call Chain

1. **Entry Point** (`pkg/scheduler/schedule_one.go:66`): `ScheduleOne()` attempts to schedule a pod requesting DRA resources
2. **Scheduling Fails** (`pkg/scheduler/schedule_one.go:139`): `schedulingCycle()` fails → `handleSchedulingFailure()` adds pod to unschedulable queue
3. **Pod Should Be Re-evaluated** (`pkg/scheduler/internal/queue/scheduling_queue.go:1120`): When ResourceSlice events occur, `MoveAllToActiveOrBackoffQueue()` should move waiting pods back to active queue
4. **Event Check Fails** (`pkg/scheduler/internal/queue/scheduling_queue.go:414`): `isEventOfInterest()` checks the `queueingHintMap` for ResourceSlice events
5. **BUG**: ResourceSlice is missing from the queueing hint map → `isEventOfInterest()` returns false → unschedulable pods never re-evaluated

### Missing Components

1. **Missing GVK Constant**: No `ResourceSlice` defined in `pkg/scheduler/framework/types.go` (while ResourceClaim, ResourceClass, etc. are defined)
2. **Missing Event Registration**: DRA plugin doesn't declare interest in ResourceSlice events
3. **Missing Event Handler**: No case for ResourceSlice in the switch statement in `pkg/scheduler/eventhandlers.go:396`

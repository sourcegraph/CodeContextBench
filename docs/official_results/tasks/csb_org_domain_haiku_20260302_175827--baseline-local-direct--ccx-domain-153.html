## Key Findings

### **Route Matching Architecture**

1. **Virtual Host Selection** (`RouteMatcher::findVirtualHost`):
   - Exact hostname matching using hash maps
   - Wildcard suffix matching (e.g., `*.example.com`)
   - Wildcard prefix matching
   - Falls back to default virtual host
   - Handles port sensitivity and case-insensitivity

2. **Route Matching Algorithm** (`RouteMatcher::route`, `VirtualHostImpl::getRouteFromRoutes`):
   - Different matching strategies by route type:
     - `PrefixRouteEntryImpl` - prefix matching
     - `PathRouteEntryImpl` - exact path matching
     - `RegexRouteEntryImpl` - regex matching
     - `PathSeparatedPrefixRouteEntryImpl` - path-separated prefix
     - `UriTemplateMatcherRouteEntryImpl` - URI templates
     - `ConnectRouteEntryImpl` - HTTP CONNECT handling

3. **Route Action Dispatch** (`RouteEntryImplBase`):
   - Determines action type (direct response, redirect, or forward to cluster)
   - Handles header modifications, retries, rate limiting, and other policies

### **Integration Points**

The route matching is integrated through:
- HTTP Connection Manager filter instantiation
- Request processing in `ConnectionManagerImpl::ActiveStream::decideRoute()`
- Route selection and caching
- Router filter dispatch based on matched route

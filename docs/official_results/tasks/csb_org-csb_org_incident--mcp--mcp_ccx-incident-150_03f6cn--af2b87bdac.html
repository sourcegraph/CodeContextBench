The error occurs when a node tries to catch up via snapshot during rejoin, but the snapshot is either:
- Too old (filtered out by `ValidSnapshotEntries` because its index exceeds the committed hardstate)
- Missing from disk (returned as `ErrNoDBSnapshot`)
- Being prepared (returned as `ErrSnapshotTemporarilyUnavailable`)

## Context

The comment likes feature spans three backend files:

1. **`ghost/core/core/server/services/comments/comments-service.js`** — Service layer: `getCommentLikes()` method that queries the database for likes on a given comment.
2. **`ghost/core/core/server/services/comments/comments-controller.js`** — Controller layer: `getCommentLikes()` method that extracts the comment ID from the request frame and delegates to the service.

## Review Summary

I've completed the code review and identified **3 defects** in the comment likes feature:

- **File**: `comments-controller.js`, line 370
- **Issue**: Uses `frame.data.id` instead of `frame.options.id`
- **Impact**: Comment ID will be undefined, breaking the query

- **Issue**: Uses `withRelated: []` instead of `withRelated: ['member']`
- **Impact**: API response lacks member information, making the likes list incomplete and unusable
- **Root Cause**: Inconsistent with parallel method `getCommentReporters` (line 246) which correctly includes `withRelated: ['member']`

### **DEFECT 3 (MEDIUM)**: Unused Comment Fetch
- **File**: `comments-service.js`, line 261  
- **Issue**: Comment is fetched but never validated or used
- **Impact**: Wasteful database query; inconsistent error handling compared to methods like `getCommentByID`
- **Root Cause**: Appears to be incomplete implementation or dead code

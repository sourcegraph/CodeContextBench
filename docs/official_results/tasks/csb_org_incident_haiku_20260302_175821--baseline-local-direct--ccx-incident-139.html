## Summary

I've identified the key C++ source files implementing LibreOffice Calc's formula engine:

### **1. Compiler Phase** (`sc/source/core/tool/compiler.cxx`)
- Implements `ScCompiler` class that tokenizes and compiles formula strings into bytecode
- Parses strings like `=SUM(A1:A10)` into token arrays

### **2. Interpreter Phase** (`sc/source/core/tool/interpr1.cxx` through `interpr6.cxx`)
- Implements `ScInterpreter` class (declared in `sc/source/core/inc/interpre.hxx`)
- Executes formula bytecode and evaluates functions/references
- Split across 6 files due to large code size

### **3. Formula Cell** (`sc/inc/formulacell.hxx`)
- Defines `ScFormulaCell` - the cell type holding formula and result
- Contains `Interpret()` method that triggers evaluation
- Tracks recalculation state with `bDirty` flag

### **4. Recalculation Scheduling** 
- `include/formula/tokenarray.hxx` - defines `ScRecalcMode` enum (ALWAYS, ONLOAD_MUST, NORMAL, etc.)
- `sc/source/core/data/documen7.cxx` - implements `ScDocument::CalcFormulaTree()`
- `sc/source/core/data/document.cxx` - implements `ScDocument::CalcAll()`

### **Supporting Files**
- `sc/inc/tokenarray.hxx` / `sc/source/core/tool/token.cxx` - token array and token definitions
- `sc/inc/compiler.hxx` - compiler class definition

The complete evaluation chain flows: **Parser → Tokenizer → Token Array → Formula Cell → Interpreter → Result**

## Summary

I've identified the Java source files in apache/kafka that define the heartbeat session timeout threshold and handle the timeout during the consumer polling loop:

**Key Files Found:**

1. **CommonClientConfigs.java** - Defines the `SESSION_TIMEOUT_MS_CONFIG` configuration constant ("session.timeout.ms")

2. **GroupRebalanceConfig.java** - Extracts and stores the `sessionTimeoutMs` value from the configuration

3. **Heartbeat.java** - Creates a `sessionTimer` using the `sessionTimeoutMs` value and provides the `sessionTimeoutExpired(long now)` method to check if the timeout has expired

4. **AbstractCoordinator.java** - Implements the heartbeat thread that checks `heartbeat.sessionTimeoutExpired(now)` during the consumer polling loop. When the session timeout expires, it logs "session timed out without receiving a heartbeat response" and calls `markCoordinatorUnknown()`, which triggers the consumer's removal from the group.

**Flow:**
The session timeout value flows from configuration → GroupRebalanceConfig → Heartbeat class → checked in AbstractCoordinator's HeartbeatThread during polling → triggers group membership loss when exceeded.

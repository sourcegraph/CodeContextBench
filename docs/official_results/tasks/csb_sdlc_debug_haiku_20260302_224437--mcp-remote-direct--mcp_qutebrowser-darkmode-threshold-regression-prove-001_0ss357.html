### What the Test Does

The test verifies that the `colors.webpage.darkmode.threshold.text` setting is correctly mapped to `ForegroundBrightnessThreshold` for Qt 6.4 (not `TextBrightnessThreshold` which was used in earlier Qt versions).

### Four Test Cases

1. **`test_qt64_threshold_text_is_renamed()`** - Core test that would **FAIL** if the bug exists
   - Verifies Qt 6.4 uses `ForegroundBrightnessThreshold`
   - Verifies Qt 5.15.3 uses `TextBrightnessThreshold` (for comparison)

2. **`test_qt64_other_settings_unchanged()`** - Confirms the fix doesn't break other settings
   - All non-threshold.text settings remain identical between Qt 6.3 and Qt 6.4

3. **`test_qt64_background_threshold_unchanged()`** - Verifies only text threshold is affected
   - Background threshold correctly remains `BackgroundBrightnessThreshold`

4. **`test_qt64_has_increase_text_contrast()`** - Ensures inheritance chain is intact
   - Qt 6.4 properly inherits `IncreaseTextContrast` from Qt 6.3

### The Bug Explained

In Qt 6.4, Chromium renamed the dark mode setting from `TextBrightnessThreshold` to `ForegroundBrightnessThreshold`. If qutebrowser doesn't use the new name for Qt 6.4, Chromium silently ignores the old setting, causing users to report that the text threshold setting has no effect on dark mode rendering.

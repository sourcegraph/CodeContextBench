# Cilium eBPF Map API Reference

## Overview

The eBPF map API provides Go wrappers for user-space interaction with kernel eBPF maps. These maps serve as the primary data structure for sharing information between eBPF programs running in the kernel and user-space applications. This documentation covers the complete API contracts and behavioral semantics needed to effectively use eBPF maps in Cilium.

### Map Type

The `Map` struct represents an eBPF map and is the primary interface for map operations.

```go
type Map struct {
    Logger *slog.Logger  // Logger for debug/error output
```

#### Key Methods

**Type() ebpf.MapType**
- Returns the map type (Hash, LRUHash, Array, PerCPUHash, etc.)
- Returns `UnspecifiedMap` if the map has not been created yet

**KeySize() uint32**
- Returns the size in bytes of keys for this map

**ValueSize() uint32**
- Returns the size in bytes of values for this map

**MaxEntries() uint32**
- Returns the maximum number of entries the map can hold

**Flags() uint32**
- Returns the map flags

**FD() int**
- Returns the file descriptor of the underlying eBPF map
- Only valid after the map has been opened/created

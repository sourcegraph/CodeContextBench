# Cilium eBPF Map API Reference

## Overview

The Cilium eBPF map package provides a comprehensive Go wrapper API for user-space interaction with kernel eBPF maps. It abstracts the complexity of eBPF syscalls while providing high-level operations for managing maps, inserting data, iterating entries, and maintaining map state.

## Core Concepts

### Map Lifecycle

Maps follow a clear lifecycle:
1. **Creation**: Create a Map instance using `NewMap()` or `NewMapWithInnerSpec()`
2. **Opening/Creating**: Call `OpenOrCreate()` to open an existing map or create a new one
3. **Operations**: Perform CRUD operations while the map is open
4. **Closing**: Call `Close()` to close the map (optional if you want to keep it open)

### MapKey and MapValue Interfaces

All key and value types used with maps must implement these interfaces:

```go
type MapKey interface {
    fmt.Stringer
    // New must return a pointer to a new instance of the key type
    New() MapKey
}

type MapValue interface {
    fmt.Stringer
    // New must return a pointer to a new instance of the value type
    New() MapValue
}
```

For per-CPU maps, values must additionally implement:

```go
type MapPerCPUValue interface {
    MapValue
    // NewSlice returns a pointer to a slice of values, one per CPU
    NewSlice() any
}
```

**Example Implementation**:

```go
type EndpointKey struct {
    IP        [16]byte // IPv6 or IPv4 in lowest 4 bytes
    Family    uint8
    Key       uint8
    ClusterID uint16
}

func (k *EndpointKey) String() string {
    // Return human-readable representation
    return fmt.Sprintf("endpoint:%s", k.IP)
}

func (k *EndpointKey) New() MapKey {
    return &EndpointKey{}
}
```

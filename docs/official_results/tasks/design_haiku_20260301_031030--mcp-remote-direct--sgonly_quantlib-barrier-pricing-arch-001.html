## Summary

I've traced the complete pricing chain for barrier options in QuantLib, documenting:

### **Key Findings:**

1. **Entry Point**: `BarrierOption.NPV()` → `Instrument.NPV()` → `calculate()` triggers lazy evaluation

2. **Lazy Object Pattern**: The `LazyObject` framework defers computation until needed, caches results, and uses Observer pattern for change notification

3. **Dual Pricing Routes**:
   - **Analytic**: Direct formula application (AnalyticBarrierEngine) using current market data
   - **Monte Carlo**: Path simulation loop (MCBarrierEngine) generating thousands of paths

4. **Monte Carlo Architecture**:
   - `McSimulation` orchestrates the tolerance loop
   - `MonteCarloModel` executes core sample generation and pricing
   - `PathGenerator` creates asset paths using `GeneralizedBlackScholesProcess`
   - `PathPricer` evaluates barrier conditions and payoffs

5. **Stochastic Process Integration**:
   - `GeneralizedBlackScholesProcess` encapsulates market dynamics
   - Queries three term structures at each time step:
     - `YieldTermStructure` (risk-free rates & dividends)
     - `BlackVolTermStructure` (volatility surface)
   - Uses Euler discretization for continuous SDE evolution

6. **Design Patterns**: The architecture elegantly combines Lazy Evaluation, Strategy, Template Policy, Handle/Repository, and Factory patterns to achieve flexibility and separation of concerns.

The analysis document includes detailed dependency chains, component responsibilities, data flow diagrams, and interface contracts between all major components.

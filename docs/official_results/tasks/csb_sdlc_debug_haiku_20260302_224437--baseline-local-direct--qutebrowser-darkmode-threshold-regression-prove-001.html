## Investigation Complete ✅

### Root Cause Found
The dark mode text threshold setting has no effect on Qt 6.4 because:

1. **Missing Qt 6.4 Variant**: The `Variant` enum in `/app/qutebrowser/browser/webengine/darkmode.py` only defines Qt 5.15.2, 5.15.3, and 6.3 variants - **Qt 6.4 is missing**

2. **Setting Name Changed**: Qt 6.4's Chromium (v99.0.4785.0+) renamed the setting from `TextBrightnessThreshold` to `ForegroundBrightnessThreshold`

3. **Fallback Bug**: When Qt 6.4 is detected, the code falls back to the Qt 6.3 variant, which still uses the old setting name. Chromium ignores this setting, making the user's configuration ineffective.

# Bug Investigation: Dark Mode Text Threshold Setting Has No Effect on Qt 6.4

## Summary

The dark mode text brightness threshold setting (`colors.webpage.darkmode.threshold.text`) has no effect on Qt 6.4 because the codebase is missing Qt 6.4 variant support. When running on Qt 6.4, the code falls back to using the Qt 6.3 variant, which still uses the old Chromium setting name `TextBrightnessThreshold`. However, Qt 6.4's Chromium (v99.0.4785.0 and later) renamed this setting to `ForegroundBrightnessThreshold`, so the old setting name is ignored.

## Root Cause

**Location:** `/app/qutebrowser/browser/webengine/darkmode.py`

The `Variant` enum (line 106-112) only defines three variants:
- `qt_515_2`
- `qt_515_3`
- `qt_63`

It's missing a `qt_64` variant. The `_variant()` function (line 309-328) uses version detection to select which variant to use. For Qt 6.4, it doesn't find a `qt_64` variant, so it falls through and returns `qt_63` (line 318-319).

```python
if versions.webengine >= utils.VersionNumber(6, 3):
    return Variant.qt_63
```

## The Fix (Already Exists in Commit 50efac08f)

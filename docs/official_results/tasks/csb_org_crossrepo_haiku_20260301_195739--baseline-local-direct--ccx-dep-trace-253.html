## LLVM Pass Manager Architecture

I've identified and documented the complete registration and execution chain for LLVM optimization passes:

### **Core Components:**

1. **PassManager Template** (`llvm/include/llvm/IR/PassManager.h`): The templated container that manages pass execution, parameterized by IR unit type (Module, Function, etc.)

2. **PassConcept & PassModel** (`llvm/include/llvm/IR/PassManagerInternal.h`): Type-erased polymorphic pattern where PassModel wraps any pass class implementing the PassConcept interface

3. **PassInfoMixin** (`llvm/include/llvm/IR/PassManager.h`): CRTP mixin providing pass naming and pipeline printing

4. **PassRegistry** (`llvm/include/llvm/PassRegistry.h`): Manages global pass registration and lookup

### **Registration & Parsing:**

- **PassRegistry.def** (`llvm/lib/Passes/PassRegistry.def`): Macro-based registry mapping string names to pass constructors
- **PassBuilder** (`llvm/lib/Passes/PassBuilder.cpp`): Parses pass pipeline strings and instantiates passes

### **O2/O3 Optimization Pipeline:**

The optimization pipeline hierarchy in `PassBuilderPipelines.cpp`:
- `buildPerModuleDefaultPipeline()` → Main entry point
- `buildModuleSimplificationPipeline()` → Early canonicalization passes
- `buildModuleOptimizationPipeline()` → Main optimizations (with O3-specific passes like ControlHeightReduction)

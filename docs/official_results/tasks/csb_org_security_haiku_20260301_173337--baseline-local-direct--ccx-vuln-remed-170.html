## Key Findings

I've identified three main files in the Prometheus repository that implement web UI authentication:

### 1. **Admin API Access Control** (`web/api/v1/api.go`)
- The `API` struct contains an `enableAdmin` boolean field (line 241)
- Three admin functions check this flag:
  - `deleteSeries()` (line 2026-2029)
  - `snapshot()` (line 2059-2062)
  - `cleanTombstones()` (line 2093-2096)
- Each function returns an error if admin APIs are disabled

### 2. **TLS and BasicAuth Integration** (`web/web.go`)
- The `Handler.Run()` method (lines 722-770) integrates with the external exporter-toolkit
- Web configuration is passed to `toolkit_web.ServeMultiple()` which handles:
  - TLS certificate validation
  - BasicAuth middleware
  - Client certificate authentication

### 3. **Web Configuration** (`cmd/prometheus/main.go`)
- The `--web.config.file` flag (lines 401-404) specifies the path to a YAML configuration file
- This file enables TLS and authentication settings
- Example configuration shown in `documentation/examples/web-config.yml` demonstrates:
  - TLS certificate and key file paths
  - BasicAuth user credentials with bcrypt hashing

**Important Note**: While Prometheus integrates with TLS and BasicAuth middleware, the actual implementation of these features is delegated to the external `github.com/prometheus/exporter-toolkit` library (v0.15.1). Prometheus itself directly implements only the admin API access control through the `enableAdmin` flag checks.

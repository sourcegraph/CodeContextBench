## Key Findings:

**Kubernetes** implements the most sophisticated approach with:

- Platform-specific handling (POSIX and Windows variants)

**Prometheus** uses a simpler but effective pattern:
- Direct `signal.Notify()` in `cmd/prometheus/main.go` for SIGTERM/SIGINT  
- `oklog/run.Group` for coordinating component shutdown order
- Web service integration via `webHandler.Quit()` for HTTP-triggered shutdown
- Automatic log message: "Received an OS signal, exiting gracefully..."

**Grafana** employs context-based coordination:
- `Shutdown()` method with `sync.Once` to ensure single execution
- Context deadlines for timeout enforcement
- HTTP server graceful shutdown via `http.Server.Shutdown()`
- Service manager delegation for cleanup

All implementations follow the pattern: **register signal handlers → drain connections → stop background services → exit cleanly**

## Log Compaction Architecture in Apache Kafka

### **Cleaner Thread** (`LogCleaner.java`)
- Main cleaner thread management class implementing `BrokerReconfigurable`
- Contains inner class `CleanerThread` extending `ShutdownableThread`
- Manages a pool of cleaning threads that continuously work through logs

### **Cleaning Logic** (`Cleaner.java`)
- Implements the actual log compaction logic
- Key methods:
  - `doClean()` - orchestrates the cleaning process
  - `buildOffsetMap()` - builds key=>offset mapping for deduplication
  - `cleanSegments()` - cleans groups of segments
  - `shouldRetainRecord()` - key deduplication logic
  - `groupSegmentsBySize()` - segment selection/grouping policy

### **Segment Selection Policy** (`LogCleanerManager.java`)
- Selects which logs to clean based on "dirtiness" ratio
- `grabFilthiestCompactedLog()` - selects partition with highest dirty-to-total bytes ratio
- `cleanableOffsets()` - calculates which offsets are cleanable
- `calculateCleanableBytes()` - determines bytes eligible for cleaning

### **Key-Based Deduplication** 
- **OffsetMap.java** - Interface for key=>offset mapping
- **SkimpyOffsetMap.java** - Hash-table implementation using MD5 hashing, memory-efficient deduplication

### **Supporting Files**
- `LogToClean.java` - Encapsulates a log's cleaning state
- `CleanedTransactionMetadata.java` - Handles transactional records during cleaning
- `CleanerConfig.java` - Configuration parameters
- `CleanerStats.java` - Cleaning metrics tracking

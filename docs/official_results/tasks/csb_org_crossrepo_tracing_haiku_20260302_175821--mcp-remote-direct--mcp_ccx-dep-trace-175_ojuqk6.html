## Pipeline Flow

The alertmanager notification pipeline in Grafana Mimir follows this sequence:

1. **Alert Ingestion**: `Dispatcher.Run()` subscribes to alerts from the memory provider and processes them continuously

2. **Routing Tree Traversal**: `Route.Match()` performs a depth-first search through the routing configuration tree to find all matching routes for each alert

3. **Alert Processing**: `processAlert()` determines which aggregation group each alert belongs to and schedules notification dispatch

4. **Notification Pipeline**: The `PipelineBuilder.New()` creates a sequence of stages:
   - **GossipSettleStage** - Waits for cluster gossip to settle
   - **MuteStage (Inhibitor)** - Checks inhibition rules via `Inhibitor.Mutes()`
   - **TimeActiveStage** - Checks active time intervals
   - **TimeMuteStage** - Checks mute time intervals
   - **MuteStage (Silencer)** - Checks silence rules via `Silencer.Mutes()`
   - **Receiver stages** - Deduplication, retry, and dispatch

5. **Inhibition Check**: The `Inhibitor` maintains a cache of source alerts and checks if target alerts should be suppressed

6. **Silence Check**: The `Silencer` queries the silence store to check if alerts match active silences

7. **Final Dispatch**: `RetryStage.Exec()` sends notifications with exponential backoff retry logic

## Key Components Identified

1. **FilterChainFactory Interface** (`envoy/http/filter_factory.h`): Defines the contract for creating filter chains. It has two key methods:
   - `createFilterChain()` - Creates normal HTTP filter chains
   - `createUpgradeFilterChain()` - Handles protocol upgrades (WebSocket, etc.)

2. **FilterChainHelper** (`source/common/http/filter_chain_helper.h/cc`): A template class that processes the protobuf configuration during server initialization. It:
   - Parses each filter in the `http_filters` configuration
   - Instantiates filter factories from the registry
   - Creates a list of filter factories for runtime use

3. **FilterManager** (`source/common/http/filter_manager.h/cc`): Implements both `FilterChainManager` and manages filter execution at runtime. It:
   - Creates filter chain instances for each new HTTP stream
   - Manages both decoder and encoder filter chains
   - Coordinates filter iteration through `ActiveStreamFilterBase`

4. **HttpConnectionManager Proto**: Defines the configuration schema with:
   - `repeated HttpFilter http_filters` field at line 433
   - Each `HttpFilter` message contains name, config type, and optional/disabled flags

5. **HttpConnectionManagerConfig**: Implements the `FilterChainFactory` interface and orchestrates the entire registration flow during bootstrap

## Registration and Execution Flow

The complete flow is:
- **Configuration Phase**: Proto config → FilterChainHelper → Filter factories list → stored in HttpConnectionManagerConfig
- **Execution Phase**: New stream created → FilterManager::createFilterChain() → HttpConnectionManagerConfig::createFilterChain() → FilterChain

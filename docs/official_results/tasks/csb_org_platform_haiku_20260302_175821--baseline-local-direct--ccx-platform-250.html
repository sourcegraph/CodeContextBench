The Apache Kafka log segment and index architecture is implemented across the storage module with the following key components:

### Core Components:
1. **LogSegment** - Represents a single log segment containing records and managing persistent storage
2. **OffsetIndex** - Sparse index mapping offsets to physical file locations using relative offsets
3. **TimeIndex** - Index mapping timestamps to logical offsets with 12-byte entries
4. **AbstractIndex** - Base class providing memory-mapped file operations and binary search lookups

### Configuration & Validation:
5. **LogConfig** - Validates log configuration parameters including retention policies
6. **LogValidator** - Validates incoming records during append operations

### Log Recovery on Startup:
7. **LogLoader** - Orchestrates log recovery from checkpoint files
8. **OffsetCheckpointFile** - Persists topic partition offsets
9. **LeaderEpochCheckpointFile** - Maintains leader epoch information
10. **CheckpointFileWithFailureHandler** - Provides atomic checkpoint operations

### Log Management:
11. **LocalLog** - Lower-level local log management
12. **UnifiedLog** - Main log abstraction for appends, reads, and segment lifecycle
13. **LogSegments** - Manages the collection of log segments
14. **LogManager** - Manages multiple log instances across partitions

### Supporting Infrastructure:
15. **TransactionIndex** - Maintains transaction metadata
16. **ProducerStateManager** - Persists producer state for idempotent writes

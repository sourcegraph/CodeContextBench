## CSS Style Change → Layout → Paint Chain

The complete flow in Firefox/Gecko involves these key stages:

1. **Style Change Notification**: `RestyleManager::PostRestyleEvent` queues elements for restyle
2. **Flush Orchestration**: `PresShell::DoFlushPendingNotifications` initiates the style→layout→paint sequence
3. **Style Recomputation**: `RestyleManager::ProcessPendingRestyles` uses Servo to recalculate styles
4. **Frame Hook**: `nsIFrame::DidSetComputedStyle` is called for affected frames
5. **Dirty Frame Marking**: `PresShell::FrameNeedsReflow` marks frames needing reflow and adds them to dirty roots
6. **Reflow Processing**: `PresShell::ProcessReflowCommands` → `DoReflow` recalculates frame geometry
7. **Paint Scheduling**: `SchedulePaint` is called during reflow to invalidate regions
8. **Display List Building**: `nsLayoutUtils::PaintFrame` → `BuildDisplayListForStackingContext` constructs the display list from the frame tree
9. **Paint Execution**: `nsDisplayList::PaintRoot` renders the display list
10. **WebRender Submission**: `WebRenderLayerManager::EndTransactionWithoutLayer` sends the display list to WebRender for GPU rendering

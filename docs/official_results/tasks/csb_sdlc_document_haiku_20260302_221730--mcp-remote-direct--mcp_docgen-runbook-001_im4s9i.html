# Prometheus TSDB Compaction Operational Runbook

## 1. Overview

### What is TSDB Compaction?

TSDB compaction is the process of merging multiple time series data blocks into larger, more efficient blocks while removing deleted series and optimizing storage layout. Compaction maintains query performance and manages disk space in Prometheus's time series database.

**Key objectives:**
- Merge overlapping time blocks to prevent duplicate data in queries
- Remove tombstoned (deleted) series to reclaim disk space
- Consolidate smaller blocks into larger ones following an exponential leveling strategy
- Optimize chunk sizes and index structure for better query performance

### Block Structure

Blocks are immutable, self-contained units of time series data. Each block:
- Contains data for a specific time range `[MinTime, MaxTime)`
- Is stored as a directory with `meta.json`, chunks, index, and tombstones files
- Is identified by a ULID (Universally Unique Lexicographically Sortable Identifier)
- Has a compaction level indicating how many times it has been compacted
- Tracks parent blocks and source ULIDs for debugging and recovery

**Block organization:**
```
tsdb_data/
├── 01ARZ3NDEKTSV4RRFFQ69G5FAV/  # Block ULID
│   ├── meta.json                 # Metadata: time range, stats, compaction info
│   ├── index                     # Index of all series and labels
│   ├── chunks/                   # Chunk segment files
│   └── tombstones                # Deletion markers
├── 01ARZ3NDEKTSV4RRFFQ70G5FAW/
...
├── wal/                          # Write-ahead log (active head)
└── wbl/                          # Write-ahead log for out-of-order samples
```

# Test Coverage Gap Analysis: Envoy HTTP Connection Manager

## Summary

The HTTP Connection Manager (HCM) is a critical component in Envoy's HTTP processing pipeline, responsible for routing, filter execution, stream lifecycle management, and response handling. Analysis of `source/common/http/conn_manager_impl.cc` against `test/common/http/conn_manager_impl_test.cc` reveals **5 significant untested code paths** that handle edge cases, error conditions, and complex state transitions. ## Coverage Gaps

### Gap 1: Zombie Stream Lifecycle and Codec Completion Race Condition

- **File**: source/common/http/conn_manager_impl.cc
- **Lines**: ~2137-2180 (onCodecEncodeComplete, onCodecLowLevelReset), ~348-352 (canDestroyStream check in doDeferredStreamDestroy)
- **Risk**: HIGH
- **Description**:
  When a stream's response encoding completes, the HCM checks if the stream is a "zombie" (marked for destruction but waiting for codec completion signal). The zombie stream path handles two codec completion scenarios: `onCodecEncodeComplete()` and `onCodecLowLevelReset()`. The code defers stream destruction until the codec signals completion, but the interaction between these states and the deferred close logic (`checkForDeferredClose()` with `close_connection_on_zombie_stream_complete_` flag) is never directly tested.

- **Proposed Test**: `ZombieStreamOnCodecEncodeComplete`
  - Setup: Create a stream, trigger `doEndStream()` before codec completes encoding (via response encoder still present but conditions met), verify stream becomes zombie
  - Action: Simulate codec completion via `onCodecEncodeComplete()`

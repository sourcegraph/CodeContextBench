## Key Findings

### Controller Registration Pattern in Kubernetes

Controllers in Kubernetes register with a central Manager through a **descriptor-based pattern**:

1. **ControllerDescriptor** - Each controller is wrapped in a descriptor containing:
   - Name and aliases
   - Initialization function (initFunc)
   - Required feature gates
   - Metadata about special handling

2. **Central Registry** - `NewControllerDescriptors()` in `controllermanager.go` is the registration point where all controllers (Deployment, ReplicaSet, StatefulSet, DaemonSet, Job, etc.) are registered via a `register()` function.

3. **Manager Orchestration** - `StartControllers()` and `StartController()` functions manage the lifecycle:
   - Iterate through registered descriptors
   - Invoke each controller's initFunc
   - Provide a `ControllerContext` with necessary dependencies

4. **Dependency Injection** - `ControllerContext` provides:
   - ClientBuilder for API clients
   - InformerFactory for watching Kubernetes resources
   - RESTMapper for API resource discovery
   - Shared metrics and configuration

5. **Controller Interfaces** - Controllers implement:
   - `controller.Interface` (required, with `Name()` method)
   - `Debuggable` (optional, for debugging handlers)
   - `HealthCheckable` (optional, for health checks)

### Key Files Identified

- **controllermanager.go** - Main manager orchestration and descriptor definitions
- **interfaces.go** - Controller interface definitions
- **controllercontext.go** - Dependency context structure
- **apps.go** - Concrete controller registrations

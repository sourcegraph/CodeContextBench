that may be concurrently modified, with tracking of iteration progress and recovery from concurrent deletions.\n\n```go\nfunc (m *Map) DumpReliablyWithCallback(\n    cb DumpCallback,\n    stats *DumpStats,\n) error\n```\n\n**Parameters:**\n- `cb`: Callback invoked for each entry\n- `stats`: Statistics object (initialized via `NewDumpStats()`) tracking iteration state\n\n**Semantics:**\n- limit is exceeded\n- Callback must not invoke map operations that require the write lock\n- Holds the write lock during iteration (to allow deletions within callbacks)\n- Populates `stats` with iteration statistics (lookups, failures, etc.)\n- Returns `nil` and `stats.Completed == true` when fully iterated\n- Returns `ErrMaxLookup` if iteration limit exceeded without completing\n\n**Statistics Tracking:**\n```go\ntype DumpStats struct {\n    Started          time.Time  // Iteration start time\n    Finished         time.Time  // Iteration finish time\n    Lookup capacity\n    Interrupted      uint32     // Iteration restarts\n    Completed        bool       // Whether fully iterated\n}\n```\n\n**Example:**\n```go\nstats := NewDumpStats(m)\nerr := m.DumpReliablyWithCallback(func(key MapKey, value MapValue) {\n    // Process entry\n}, stats)\nif err != nil {\n    log.Printf(\"Iteration error: %v\", err)\n}\nlog.Printf(\"Completed: %v, Lookups: Batch Iterator\n\n`BatchIterator` provides a typed, convenient interface for iterating maps using the BPF batch API.\n\n```go\ntype BatchIterator[KT, VT any, KP KeyPointer[KT], VP ValuePointer[VT]] struct {\n    // ... internal fields\n}\n```\n\n**Creating an Iterator:**\n```go\nfunc NewBatchIterator[KT any, VT any, KP KeyPointer[KT], VP ValuePointer[VT]](\n    m *Map,\n) *BatchIterator[KT, VT, **ErrMapNotOpened**: Returned when operations are attempted on a map that hasn't been opened\n- **ErrKeyNotExist**: Returned from Lookup when key doesn't exist (wrapped in ebpf errors)\n- **ErrMaxLookup**: Returned from DumpReliablyWithCallback when iteration limit exceeded\n- **ErrMapIncompatible**: Returned when LoadCollection encounters incompatible pinned maps\n\n### Synchronization and Error Recovery\n\nWhen a map has caching enabled via `WithCache()`, failed Update or Delete operations are tracked:\n\n1.

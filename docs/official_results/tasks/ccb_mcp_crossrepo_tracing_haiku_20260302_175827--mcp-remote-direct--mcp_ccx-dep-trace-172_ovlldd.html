## Summary

**Key Findings:**

1. **Reconciler Interface** - The core abstraction defined in `vendor/sigs.k8s.io/controller-runtime/pkg/reconcile/reconcile.go`:
   - Single method: `Reconcile(context.Context, Request) (Result, error)`
   - Controllers implement this to define their business logic

2. **Builder Pattern** - Controllers wire themselves via `vendor/sigs.k8s.io/controller-runtime/pkg/builder/controller.go`:
   - `ControllerManagedBy(manager)` - Binds controller to a Manager
   - `.For()`, `.Owns()`, `.Watches()` - Configure watch sources
   - `.Complete(reconciler)` or `.Build()` - Finalizes setup

3. **SetupWithManager Pattern** - Standard implementation pattern:
   ```go
   func (r *Reconciler) SetupWithManager(mgr ctrl.Manager) error {
       return ctrl.NewControllerManagedBy(mgr).
           For(&MyResource{}).
           Owns(&Pod{}).
           Complete(r)
   }
   ```

4. **Kubernetes Integration** - The Kubernetes kube-controller-manager uses its own pattern:
   - `ControllerDescriptor` with `InitFunc` callbacks
   - Base `Interface` with `Name()` method
   - Different but complementary to controller-runtime pattern

5. **Reconciliation Loop** - Orchestrated by Manager:
   - Event sources trigger enqueuing
   - Manager calls `Reconciler.Reconcile()` with Request
   - Result determines requeue behavior

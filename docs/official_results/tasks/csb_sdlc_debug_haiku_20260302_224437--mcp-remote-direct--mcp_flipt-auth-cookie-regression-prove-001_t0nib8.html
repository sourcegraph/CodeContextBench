## Reported Issue

The authentication system in Flipt only accepts client tokens via the `Authorization` header using Bearer format. This prevents browser-based sessions from working, since browsers typically store authentication tokens in HTTP cookies.

Specifically:

1. **Cookie tokens rejected**: When a valid client token is stored in an HTTP cookie (under the key `flipt_client_token`), the authentication middleware does not extract or validate it. ## Summary of Findings

### **Bug 1: Cookie-Based Token Extraction Failure**
**Location:** `internal/server/authn/method/http.go`, lines 45-54  
**Issue:** The `ForwardCookies()` function has a critical bug on line 49 where it always writes to the same metadata key regardless of which cookie is being processed:

```go
for _, key := range []string{stateCookieKey, tokenCookieKey} {
    if cookie, err := req.Cookie(key); err == nil {
        md[stateCookieKey] = []string{cookie.Value}  // BUG: Always uses stateCookieKey!
    }
}
```

This causes the `flipt_client_token` cookie value to overwrite the `flipt_client_state` value in metadata, and the token is never properly forwarded to the gRPC middleware.

**Impact:** Browser-based sessions fail because:
- Valid tokens stored in cookies are not extracted
- Cookies aren't forwarded to the gRPC middleware in the expected format
- The middleware cannot validate cookie-based authentication

---

### **Bug 2: Server-Level Authentication Skip Not Configured**
**Location:** `internal/cmd/authn.go`, function `authenticationGRPC()` (lines 88-277)  
**Issue:** While public servers (like `public.Server`) implement the `SkipsAuthenticationServer` interface, they're never registered with the `WithServerSkipsAuthentication()` option during middleware setup.

**Impact:** Public authentication endpoints incorrectly require authentication:
- `ListAuthenticationMethods` should be public but requires auth
- OIDC callbacks need unauthenticated access but are protected
- Delegated authentication flows that need server-to-server communication fail

## Codebase Patterns
- Investigation tasks: benchmarks/ccb_investigation/{task_id}/ — existing suite, expand from 4 to 12 tasks
- Existing tasks: inv-debug-001, inv-impact-001, inv-migration-001, inv-regression-001 (all score 0.9+)
- New tasks must be SIGNIFICANTLY harder: target baseline 0.2-0.5 (not 0.9+)
- Three new subtypes: deep_causal_chain (4-5 hops), multi_component_interaction (3+ components), regression_hunt (use commit history)
- Output format: /logs/agent/investigation.md (same as existing)
- Scoring: weighted checklist via ground_truth.json (same scorer as existing — test.sh is shared across suite)
- ground_truth.json for harder tasks: 8-12 required_findings (vs 5-8), 4-5 hop causal_chains (vs 2-3), 2+ negative_checks (vs 1)
- Deep causal: symptom in package A → root cause in package D/E, with 3+ intermediate packages
- Multi-component: bug emerges from interaction between 3+ independent components under specific timing
- Regression hunt: agent must find the COMMIT that introduced the regression (commit_search + diff_search essential)
- Regression task naming: use inv-regression-001b (or inv-regression-002) to avoid conflict with existing inv-regression-001
- Dockerfile: pin to commit where bug exists (pre-fix for investigations, post-regression for regression hunts)
- MCP advantage: find_references for cross-package tracing, commit_search for regression hunts, diff_search for code changes
- Config: investigation_2config.sh — add new task IDs to existing TASKS array

## Codebase Patterns
- Benchmark tasks: benchmarks/ccb_{suite}/{task_id}/ with task.toml, instruction.md, environment/Dockerfile, tests/test.sh, tests/ground_truth.json
- Investigation test.sh pattern (reuse for analysis tasks): bash wrapper → embedded Python reads ground_truth.json, scores via regex pattern matching
- ground_truth.json format: {weights, required_findings[], file_references[], causal_chain[], negative_checks[]}
- task.toml fields: [metadata] name, description, license; [task] id, repo, category, language, difficulty, time_limit_sec; [verification] type=test, command="bash /tests/test.sh"; [environment] build_timeout_sec, setup_scripts.mcp_config
- MCP config in task.toml: copy from benchmarks/ccb_investigation/inv-debug-001/task.toml [environment.setup_scripts] section
- Dockerfile pattern: FROM ubuntu:22.04, apt-get install git, git clone at pinned SHA, WORKDIR /workspace
- instruction.md must have ZERO mentions of Sourcegraph, MCP, or specific tool names (MCP guidance injected at runtime)
- selected_benchmark_tasks.json entries: {benchmark, task_id, repo, language, difficulty, mcp_benefit_score}
- Config script TASKS array: add task_id to configs/nlqa_2config.sh
- Shared scorer template at benchmarks/templates/weighted_checklist_scorer.sh (created by infra Ralph)
- Use Sourcegraph MCP tools (keyword_search, find_references, list_files, read_file) to research target repos and build ground truth
